# SSE란?

## 개념 정의

SSE(Security Service Edge, 보안 서비스 엣지)는 가트너가 정의한 개념으로, **사용자가 어디서 접속하든 웹·SaaS·사내 애플리케이션에 대한 접속을 클라우드 기반 보안 서비스로 보호하는 아키텍처**이다.  
SASE에서 **네트워크(SD‑WAN) 기능을 제외하고, 보안 기능만을 따로 떼어낸 보안 전용 영역**이라고 보면 됨.  

***

## 핵심 개념 요약

- SASE = SD‑WAN(네트워크) + SSE(보안 서비스)  
- SSE는 보안 웹 게이트웨이(SWG), CASB, ZTNA, FWaaS 등 **클라우드 제공 보안 기능**을 통합한 것.  
- 목적: “사용자·장치 위치에 상관없이, 인터넷·SaaS·프라이빗 앱에 대한 **제로 트러스트 기반 안전한 접근**을 제공”하는 것.  

***

## SSE를 구성하는 주요 기술

**1) SWG (Secure Web Gateway)**  
- 웹 트래픽 프록시·필터링.  
- 악성 사이트 차단, 카테고리 기반 웹 차단, 다운로드 검사, DLP 정책 적용 등.  

**2) CASB (Cloud Access Security Broker)**  
- SaaS(클라우드 앱) 사용 가시성·통제.  
- 그림자 IT 탐지, 민감 데이터 과다 공유 탐지, SaaS 정책 위반 제어 등.  

**3) ZTNA (Zero Trust Network Access)**  
- “네트워크 단위 접근”이 아닌 “애플리케이션 단위 접근”을 허용.  
- 사용자·디바이스 신원·상태 기반으로 사내/프라이빗 앱에 최소 권한 접근만 허용.  
- 기존 VPN을 대체/보완하는 제로 트러스트 원격접속 방식.  

**4) FWaaS (Firewall as a Service) 및 기타**  
- 클라우드 기반 L3–L7 방화벽, DNS 보호, DLP, 샌드박싱 등 보안 기능을 엣지에서 제공.  

***

## SSE와 SASE 차이

| 항목 | SSE | SASE |
|------|-----|------|
| 초점 | 보안 서비스에만 집중 | 네트워크(SD‑WAN) + 보안을 통합 |
| 주요 구성 | SWG, CASB, ZTNA, FWaaS, DLP 등 | SD‑WAN + SSE (위의 보안 기능들) |
| 목표 | 사용자–앱 간 안전한 접속, 웹·SaaS·프라이빗 앱 보호 | 지점·사용자–클라우드 간 연결 최적화 + 보안 일원화 |
| 도입 대상 | 기존 WAN/네트워크는 유지하고, 보안만 클라우드로 이전하려는 조직 | 네트워크와 보안을 함께 재설계하려는 조직 |

정리하면, SSE는 “보안만 클라우드 엣지로 빼서 통합 관리하는 보안 계층”이고, 거기에 SD‑WAN까지 더해 네트워크까지 함께 재구성한 것이 SASE이다.
