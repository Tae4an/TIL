# 어플라이언스란?

## 1. 개념 요약

어플라이언스(Appliance)는 **특정 기능이나 용도에 최적화된 일체형 하드웨어 장치**를 말한다.  
**OS, 소프트웨어가 미리 설치·설정되어 있어 전원만 꽂으면 바로 사용할 수 있는 "즉시 사용형 장비"** 다.

***

## 2. 특징

- **즉시 사용 가능**  
  - 복잡한 설치·설정 불필요, 박스 열고 전원 연결 → 바로 운영  
  - 하드웨어+OS+애플리케이션이 사전 최적화  

- **특화 설계**  
  - 범용 서버와 달리 특정 업무(보안, 스토리지 등)에만 초점  
  - 성능 최적화, 안정성 보장  

- **관리 단순화**  
  - 벤더가 업데이트·패치 제공  
  - 확장성 제한적이나 운영 부담 최소  

- **폐쇄형 구조**  
  - 사용자 직접 접근 제한(보안 강화)  

***

## 3. 어디에 쓰이는가

- **네트워크 보안**  
  - 방화벽(Fortinet, Palo Alto), WAF, IPS/IDS 어플라이언스  

- **스토리지/백업**  
  - NAS 어플라이언스, 백업 어플라이언스(Dell PowerVault 등)  

- **가상화/클라우드**  
  - vShield, NSX 같은 가상 어플라이언스(VM 형태)  

- **애플리케이션 전용**  
  - DW(Data Warehouse) 어플라이언스, 로드밸런서  

***

## 4. 일반 서버와의 차이

| 항목 | 일반 서버 | 어플라이언스 |
|------|-----------|-------------|
| **설치** | OS·소프트웨어 직접 설치·설정 | 미리 설치, 바로 사용 |
| **최적화** | 범용, 사용자별 튜닝 필요 | 특정 기능에 특화 최적화 |
| **관리** | 사용자 직접 OS·패치 관리 | 벤더 통합 관리 |
| **확장성** | 자유로운 하드웨어 추가 | 제한적(벤더 정책) |
| **비용** | 초기 저렴, 운영 복잡 | 초기 비쌈, 운영 단순 |

어플라이언스는 **빠른 도입·안정 운영**이 필요한 환경(보안, 미션크리티컬)에서 선호된다.

***

## 5. 한 줄 정리

어플라이언스는 **특정 기능에 최적화된 즉시 사용형 하드웨어**로, 설치 없이 바로 쓸 수 있어 네트워크 보안·스토리지 등 분야에서 운영 효율을 극대화하는 "턴키 솔루션"이다.
