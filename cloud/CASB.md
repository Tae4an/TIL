# CASB(Cloud Access Security Broker)란?
## 개념 정의
CASB(Cloud Access Security Broker)는 **조직의 사용자와 클라우드 서비스(SaaS 등) 사이 중간에 위치해서, 클라우드 앱 사용에 대한 보안 정책을 통합적으로 적용·감시·제어하는 보안 게이트웨이**이다.  
섀도 IT를 포함한 모든 클라우드 서비스 사용을 가시화하고, 데이터 보안(DLP), 접근 제어, 위협 탐지, 규정 준수 정책을 클라우드에까지 확장하는 플랫폼이라고 볼 수 있음.  

***
## CASB가 하는 일 한 줄로
“직원들이 어떤 SaaS(예: Microsoft 365, Google Workspace, Salesforce, Box 등)를 어떻게 쓰는지 모두 **눈에 보이게 만들고**, 그 안에서의 데이터 업로드·공유·다운로드·협업을 정책으로 통제하면서, 이상 행위와 위협까지 탐지하는 클라우드 전용 보안 관제·제어 지점”이라고 이해하면 된다.  

***
## 핵심 기능
### 1. 클라우드 사용 가시성 (섀도 IT 탐지)
- 프록시/로그/API 연동을 통해 조직 내에서 사용 중인 모든 클라우드 서비스(승인/비승인 포함)를 식별.  
- 각 서비스의 위험도 평가(보안 수준, 위치, 인증 방식 등)와 사용량·사용자 현황을 제공.
### 2. 접근 제어 및 정책 적용
- 사용자·그룹·디바이스(관리/비관리), 위치, 리스크 수준에 따라 클라우드 앱 접근 허용/차단/제한.  
- 예: 사내망+관리 디바이스는 풀 액세스, 외부망+비관리 디바이스는 읽기 전용, 업로드 차단 등 세분화 정책.
### 3. 데이터 보안·DLP
- 클라우드 내 데이터(파일, 메일, 채팅 등)와 전송 중 데이터 모두에 대해 민감 정보 탐지 및 정책 적용.  
- 신용카드 번호, 주민번호, 고객 식별 정보, 기밀 문서 등이 외부 공유/퍼블릭 링크로 노출되지 않도록 차단·암호화·토큰화.
### 4. 위협 방지·이상 행위 탐지
- 비정상 로그인(위치·시간대·디바이스), 과도한 다운로드, 대량 공유 등 이상 행위를 행동 분석으로 탐지.  
- 랜섬웨어 활동, 계정 탈취(Compromised Account), 악성 OAuth 앱 연동 등의 위협을 식별하고 자동/반자동 대응.
### 5. 규정 준수·감사
- 누가 어떤 클라우드 서비스에서 어떤 데이터를 어떻게 다루는지 로그·감사 기록 유지.  
- PCI DSS, GDPR, HIPAA 등 규제 준수 요구사항에 맞춘 보고·정책 템플릿 제공.  

***
## 동작 방식
CASB는 구현 방식에 따라 여러 모드를 제공한다.

- 프록시 기반(Forward/Reverse Proxy)  
  - 사용자의 클라우드 트래픽을 CASB 프록시를 경유시켜 실시간으로 통제.  
- API 기반(App-to-App)  
  - SaaS API 연동으로, 클라우드 내에 존재하는 데이터·설정·활동을 비동기적으로 검사·제어.  
- 로그 기반(Discovery)  
  - 프록시/방화벽/게이트웨이 로그를 분석해 어떤 클라우드 서비스를 사용하는지 “발견(Discovery)” 중심으로 활용.  

실제 서비스들은 보통 이 세 가지를 조합해,  
“실시간 제어 + 사후 스캔 + 사용량/섀도 IT 가시화”를 동시에 구현한다.  

***
## CASB vs SWG 관점
- SWG(Secure Web Gateway)  
  - 대상: 인터넷/웹 트래픽 전반  
  - 역할: URL 필터링, 악성 사이트 차단, 웹 콘텐츠·파일 검사, 안전한 웹 브라우징 보장  

- CASB  
  - 대상: 클라우드 서비스(SaaS) 및 그 안의 데이터·행위  
  - 역할: 클라우드 앱 사용 가시성, 세밀한 앱 내 권한/행위 제어, 데이터 보안·규정 준수·위협 탐지  

요약하면,  
SWG = “어디로 나가느냐(웹 목적지) 중심의 보안”,  
CASB = “클라우드 서비스 안에서 무엇을 하느냐(데이터·행위) 중심의 보안”이라고 구분하면 이해가 쉽다.  

***
## SSE/SASE에서 CASB의 위치
- SSE(Security Service Edge)의 핵심 구성 요소 중 하나가 CASB이며,  
  SWG, ZTNA와 함께 클라우드 기반으로 제공되는 보안 서비스 축을 이룬다.  
- SASE에서는 SD‑WAN(네트워크) + SSE(보안)를 합친 아키텍처 안에서,  
  “클라우드/SaaS 보안·데이터 보호·섀도 IT 통제” 역할을 CASB가 담당한다고 볼 수 있다.
