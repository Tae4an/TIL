# CNAPP란?

## 개념 정의
CNAPP(Cloud-Native Application Protection Platform, 클라우드 네이티브 애플리케이션 보호 플랫폼)는 클라우드 환경의 애플리케이션·워크로드·데이터를 개발부터 운영까지 전 수명주기에 걸쳐 보호하는 **통합 보안 플랫폼**이다. 2021년 Gartner가 처음 정의한 개념으로, 기존에 분산되어 있던 여러 클라우드 보안 도구를 하나의 플랫폼으로 결합해 일관되고 포괄적인 보안 접근 방식을 제공한다.

쉽게 말해 "클라우드 네이티브 환경(컨테이너, 쿠버네티스, 서버리스, 멀티클라우드 등)에서 코드 작성→빌드→배포→런타임까지 모든 단계의 보안을 하나의 플랫폼에서 관리하는 올인원 보안 솔루션"이다.

## 핵심 구성 요소

CNAPP는 다음과 같은 기존 독립 보안 솔루션들을 통합한다:

**CSPM (Cloud Security Posture Management)**
- 클라우드 인프라 구성 오류(misconfiguration) 탐지 및 컴플라이언스 검증
- 보안 태세 지속 모니터링, 규정 준수 자동 검사

**CWPP (Cloud Workload Protection Platform)**
- VM, 컨테이너, 서버리스 등 워크로드 런타임 보호
- 실시간 위협 탐지, 멀웨어·침입 방지, 행위 분석

**CIEM (Cloud Infrastructure Entitlement Management)**
- 클라우드 ID·권한 관리, 과도한 권한 탐지
- 최소 권한 원칙 적용, IAM 정책 최적화

**KSPM (Kubernetes Security Posture Management)**
- 쿠버네티스·컨테이너 환경의 구성 오류 및 취약점 검사
- 컨테이너 이미지 스캔, 런타임 보안

**IaC Security (Infrastructure as Code Scanning)**
- Terraform, CloudFormation 등 IaC 템플릿의 보안 취약점 사전 스캔
- 배포 전 보안 문제 조기 발견

**Vulnerability Management & Scanning**
- 애플리케이션·라이브러리·이미지·OS 취약점 스캔
- 위험도 우선순위화(Risk Prioritization), 패치 관리

**CDR (Cloud Detection and Response)**
- 실시간 위협 탐지·대응, 이상 행위 분석
- SIEM/SOAR 연동, 자동화된 보안 대응

**Data Security Posture Management (DSPM)**
- 클라우드 내 민감 데이터 위치·접근 권한·유출 위험 관리
- 데이터 분류, 암호화, 접근 제어

## 주요 특징

**통합 플랫폼**
- 여러 보안 도구를 따로 운영하지 않고 단일 플랫폼에서 모든 클라우드 보안 관리
- 통합 대시보드로 전체 보안 상태 가시성 확보

**사전 예방(Prevention) + 사후 탐지(Detection)**
- 코드·설계 단계에서 취약점·오류 조기 발견(Shift Left)
- 런타임에서 실시간 위협 탐지·대응

**자동화 및 AI/ML 기반 분석**
- 위험 자동 상관 분석, 그래프 기반 공격 경로(Attack Path) 시각화
- 머신러닝으로 이상 행위 탐지, 우선순위 자동 지정

**에이전트리스(Agentless) 접근**
- CSP(Cloud Service Provider) API 활용해 에이전트 없이도 전체 환경 스캔
- 사각지대 제거, 구성·유지보수 부담 최소화

**멀티클라우드·하이브리드 지원**
- AWS, Azure, GCP, 프라이빗 클라우드 등 모든 환경을 일관되게 보호
- 단일 콘솔에서 여러 클라우드 통합 관리

**DevSecOps 통합**
- CI/CD 파이프라인에 보안 검사 자동 삽입
- 개발·보안·운영 팀 간 협업 강화, 보안을 코드화(Security as Code)

## 기존 보안 도구와의 차이

| 특징 | 기존 분산 도구 | CNAPP |
|------|---------------|-------|
| 구조 | CSPM, CWPP, CIEM 등 개별 도구 | 단일 통합 플랫폼 |
| 가시성 | 도구별 분산·사일로화 | 전체 환경 통합 가시성 |
| 위험 상관분석 | 수동 연계 필요 | 자동 상관분석·우선순위화 |
| 배포·관리 | 각 도구별 설정·관리 | 통합 대시보드·정책 관리 |
| DevOps 통합 | 복잡·부분적 | CI/CD 네이티브 통합 |

## 주요 활용 사례

- 컨테이너·쿠버네티스·서버리스 등 클라우드 네이티브 애플리케이션 전체 수명주기 보호
- 멀티클라우드·하이브리드 환경의 일관된 보안 정책 적용 및 컴플라이언스 관리
- DevSecOps 파이프라인에서 개발·빌드·배포·런타임 각 단계 자동 보안 검증
- 공격 경로 분석을 통한 고위험 취약점 우선 대응
- 규제 준수(PCI-DSS, ISO 27001, SOC 2, HIPAA, GDPR) 자동화 및 감사 간소화

## 도입 효과

- 보안 도구 통합으로 운영 복잡도·비용 감소
- 개발 속도 유지하면서 보안 강화(Shift Left + 런타임 보호)
- 실시간 가시성·자동화로 보안 사고 대응 시간 단축
- 팀 간 협업 강화, 보안을 개발 프로세스에 자연스럽게 통합
- 멀티클라우드 환경에서도 일관된 보안 거버넌스 유지

## 누가 사용하나?

- 클라우드 네이티브 애플리케이션(컨테이너, 마이크로서비스, 서버리스)을 개발·운영하는 조직
- DevOps·DevSecOps 팀: 보안을 CI/CD에 통합, 취약점 조기 발견
- 보안·컴플라이언스 팀: 전사 클라우드 보안 태세 관리, 규정 준수 자동화
- 멀티클라우드 전략을 운영하는 엔터프라이즈

CNAPP는 클라우드 네이티브 시대의 복잡하고 동적인 보안 요구사항을 해결하기 위해 등장한 차세대 통합 보안 플랫폼으로, 현대적인 클라우드 인프라와 애플리케이션 보호의 필수 솔루션이다.
