# Zscaler란?

## 개념 정의

Zscaler(지스케일러)는 **클라우드 네이티브 SSE/SASE 플랫폼을 제공하는 보안 회사**로, 사용자를 네트워크가 아니라 애플리케이션에 직접 연결하는 제로 트러스트 프록시 아키텍처를 제공한다.  
전 세계에 분산된 클라우드 보안 플랫폼(Zero Trust Exchange)을 통해 SWG, CASB, ZTNA, 클라우드 방화벽, DLP 등을 서비스형으로 제공하는 것이 특징이다.  

***

## Zscaler가 하는 일 한 줄로

“회사 밖·안 어디에 있든 **모든 사용자 트래픽을 Zscaler 클라우드로 먼저 보내서** 거기서 검사·정책 적용을 한 뒤, 안전한 트래픽만 인터넷·SaaS·사설 앱으로 전달하는 **클라우드형 보안 게이트웨이/제로 트러스트 플랫폼**”이라고 보면 된다.  

***

## 핵심 플랫폼: Zero Trust Exchange

- 제로 트러스트 원칙 기반  
  - 사용자·기기·위치·애플리케이션·콘텐츠·리스크를 세션마다 평가해, 최소 권한으로 1:1 연결을 브로커.  
  - 네트워크에 붙여놓고 “안에 들어오면 어느 정도 신뢰”하는 방식이 아니라, 매 세션마다 다시 검증.  
- 프록시 기반 아키텍처  
  - 풀 프록시(HTTP/HTTPS, 기타 애플리케이션 프로토콜) 구조로 TLS/SSL 트래픽을 대규모로 복호화·검사 가능.  
  - 백홀링 없이 사용자와 가장 가까운 Zscaler PoP에서 검사 후, 목적지로 직접 연결.  
- 전 세계 분산 인프라  
  - 150개 이상 데이터센터에 분산된 클라우드 보안 PoP로 구성된 글로벌 인라인 플랫폼으로 소개된다.  

***

## 주요 서비스 구성

### 1) ZIA (Zscaler Internet Access)

인터넷·SaaS로 나가는 트래픽을 보호하는 **클라우드 SWG/SSE 서비스**.

- SWG(웹 프록시, URL 필터링, 웹 위협 방지)  
- 클라우드 방화벽 / IPS  
- DLP(데이터 유출 방지)·CASB(클라우드 앱 통제)·샌드박스·RBI(원격 브라우저 격리) 등 데이터·위협 보호 기능  
- 모든 TLS/SSL 트래픽 인라인 검사 및 AI 기반 위협 탐지 엔진 제공  

즉, 인터넷 출구에 두던 웹 프록시·방화벽·DLP·샌드박스 등을 클라우드로 올려서 “ZIA 하나로 대체”하는 개념이다.  

### 2) ZPA (Zscaler Private Access)

VPN을 대체하는 **ZTNA(Zero Trust Network Access)** 서비스.

- 사용자와 사설 애플리케이션(데이터센터·클라우드 내 내부 서비스)을 직접 1:1로 연결.  
- 사용자는 Zscaler 클라이언트(Z-App)로 Zscaler 클라우드에 접속,  
  Zscaler는 고객 환경에 배치된 Connector를 통해 해당 앱에만 세션을 열어준다.  
- 네트워크 자체는 외부에 노출하지 않고, 앱 단위로만 보이는 구조라 공격 표면을 크게 줄일 수 있음.  

***

## 아키텍처 관점 특징

- 사용자 중심 / 앱 중심 모델  
  - “사용자 → 네트워크 → 앱”이 아니라 “사용자 → Zscaler → 앱” 구조로, 네트워크 레벨 접근을 최소화.  
- 제로 Attack Surface  
  - 내부 앱을 인터넷에서 스캔하거나 포트 스캐닝할 수 없도록 숨기고,  
    인증·정책을 통과한 세션에만 임시로 경로를 열어주는 방식.  
- 풀 SSL/TLS 검사 전제  
  - 모든 트래픽(포트 80/443 포함)을 프록시로 수용해, 암호화된 트래픽도 기본적으로 검사한다는 전략.  

***

## 어디에 쓰는지 (주요 활용)

- 기존 온프레미스 웹 프록시·VPN·방화벽을 **클라우드 기반 SSE/SASE 구조로 전환**하려는 기업  
- 재택·모바일·지사 사용자들이 SaaS·인터넷·사설 앱에 접근할 때,  
  위치와 상관없이 **일관된 제로 트러스트 보안 정책**을 적용하고 싶은 조직  
- TLS 암호화 트래픽이 대부분인 환경에서,  
  인라인 SSL 복호화·검사를 대규모로 수행해 위협·데이터 유출을 막고자 하는 경우  

정리하면, Zscaler는 ZIA·ZPA 등을 통해 **“클라우드에 있는 거대한 보안 관문(Zero Trust Exchange)”을 제공하고, 그 관문을 통해 사용자–앱 간 연결을 제로 트러스트 방식으로 중계하는 SSE/SASE 대표 벤더**라고 보면 된다.
