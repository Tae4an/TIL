# Illumio Enforcement란

## 1. 개념 요약

Illumio Enforcement는 **워크로드(서버·VM·컨테이너)에 정의한 세그멘테이션 정책을 실제로 적용·강제하는 모드**다. 정책을 만드는 것과 실제로 트래픽을 차단하는 것은 별개이며, Illumio는 **단계적으로 안전하게 정책을 적용할 수 있도록 세 가지 Enforcement 모드**를 제공한다.

***

## 2. 세 가지 Enforcement 모드

- **Visibility Only (가시성 전용)**
  - 트래픽을 **차단하지 않고 관찰만** 한다.
  - 모든 통신 흐름을 로그·맵으로 시각화해 현황 파악과 정책 설계의 기반 데이터를 수집한다.
  - Zero Trust 도입의 **첫 번째 단계**로, 서비스 영향 없이 안전하게 시작할 수 있다.

- **Selective Enforcement (선택적 강제)**
  - 기본적으로 **모든 트래픽을 허용**하고, 명시적으로 지정한 특정 트래픽·포트만 차단한다.
  - **Deny 규칙(Enforcement Boundary)** 을 사용하며, 예를 들어 SMB(445)·RDP(3389) 같은 고위험 포트만 먼저 막는 방식이다.
  - Allow 정책을 완벽히 준비하기 전에도 빠르게 부분적 차단을 실현할 수 있어, **랜섬웨어 긴급 대응**에 유용하다.

- **Full Enforcement (완전 강제)**
  - 기본적으로 **모든 트래픽을 차단**하고, Allow 규칙에 명시된 통신만 허용한다. (화이트리스트 방식)
  - 가장 강력한 제로 트러스트 정책 모델이며, **링펜싱의 실질적 강제 적용 단계**다.
  - Allow 정책이 완비되지 않은 상태에서 적용하면 정상 통신도 차단될 수 있으므로, Visibility → Selective → Full 순서로 단계적 전환이 권장된다.

***

## 3. Enforcement Boundary란

Selective Enforcement 모드에서 사용하는 **Deny 규칙 집합**을 Enforcement Boundary라고 한다. Full Enforcement처럼 모든 것을 막는 게 아니라, **"이 트래픽만큼은 반드시 차단한다"는 최소한의 보안 경계를 신속하게 설치하는 개념**이다.

주요 활용 사례는 다음과 같다.

- **랜섬웨어 긴급 차단**: SMB·RDP 포트를 전사 워크로드에 즉시 차단
- **환경 분리**: 운영(Prod)과 개발(Dev) 환경 간 트래픽 원천 차단
- **컴플라이언스 격리**: PCI-DSS 카드 데이터 환경(CDE)을 일반 망과 분리
- **M&A 인수사 격리**: 신규 편입 법인의 네트워크를 즉시 경계로 분리
- **미관리 워크로드 보호**: VEN이 없는 레거시·메인프레임 시스템으로의 무단 접근 차단

***

## 4. Enforcement 모드 비교

| 항목 | Visibility Only | Selective | Full |
|---|---|---|---|
| 기본 동작 | 전체 허용(차단 없음) | 전체 허용 | 전체 차단 |
| 규칙 방식 | 없음 | Deny 규칙 | Allow 규칙 |
| 목적 | 현황 파악·정책 설계 | 빠른 부분 차단 | 완전한 제로 트러스트 |
| 정책 준비 필요도 | 없음 | 낮음 | 높음 |
| 서비스 중단 위험 | 없음 | 낮음 | 설정 실수 시 있음 |

***

## 5. 한 줄 정리

Illumio Enforcement는 **Visibility Only → Selective → Full의 3단계 모드로 서비스 중단 없이 단계적으로 제로 트러스트 세그멘테이션 정책을 강제 적용하는 체계**이며, Enforcement Boundary를 통해 Full Enforcement 전환 전에도 즉각적인 위협 차단이 가능하다.
