# Illumio VEN이란

VEN(Virtual Enforcement Node)은 **Illumio Core가 각 워크로드(서버·VM)에 설치하는 경량 에이전트**로, PCE(Policy Compute Engine)의 중앙 정책을 받아 해당 호스트의 OS 내장 방화벽을 자동으로 제어하는 실행 주체다.

***

## 2. 핵심 역할

- **트래픽 가시성 수집**
  - 해당 워크로드에서 발생하는 모든 인바운드·아웃바운드 네트워크 트래픽 흐름(연결 출발지·목적지·포트·프로세스)을 수집해 PCE로 전송한다.
  - 이 데이터가 Illumio Map의 실시간 애플리케이션 의존성 맵의 원천이 된다.

- **정책 시행(Enforcement)**
  - PCE로부터 받은 세그멘테이션 정책을 Linux의 iptables/nftables, Windows Firewall 등 OS 기본 방화벽 규칙으로 변환·적용한다.
  - 별도 네트워크 장비 없이 **워크로드 자체에서 트래픽을 허용/차단**한다.

- **PCE와 지속 연결**
  - VEN은 PCE와 지속적으로 통신하며 정책 변경사항을 실시간으로 반영한다.
  - 네트워크가 일시 단절되어도 마지막으로 받은 정책이 로컬에 유지되어 보호가 끊기지 않는다.

***

## 3. 동작 모드

| 모드 | 설명 |
|---|---|
| **Idle** | VEN이 설치되었지만 정책 시행 비활성 상태. 트래픽 수집만 수행 |
| **Illuminated** | 트래픽 흐름을 수집하고 PCE에 보고하지만, 아직 차단 정책은 적용 안 함(가시성 단계) |
| **Enforced** | 정책에 따라 실제 허용·차단 규칙을 OS 방화벽에 적용(완전 시행 단계) |
| **Selective** | 특정 서비스·포트에 대해서만 선택적으로 정책 시행 |

일반적으로 **Illuminated → 트래픽 흐름 분석 → 정책 설계 → Enforced** 순으로 단계적으로 전환한다.

***

## 4. 특징

- **경량 에이전트**
  - VEN은 실제 패킷 처리를 직접 하지 않고 OS 방화벽에 규칙을 위임하므로, 호스트 성능에 미치는 영향이 최소화된다.

- **다양한 OS 지원**
  - RHEL, CentOS, Ubuntu, Debian, Windows Server, AIX, Solaris 등 다양한 OS를 지원한다.

- **컨테이너 환경 지원**
  - Kubernetes 환경에서는 각 노드에 VEN을 배포하거나, Illumio의 컨테이너 전용 방식으로 Pod 단위 정책 시행도 가능하다.

***

## 5. PCE·VEN·워크로드 관계 요약

```
[PCE] ── 정책 배포 ──▶ [VEN (에이전트)]
                             │
                   OS 방화벽 규칙 자동 적용
                             │
                     [워크로드 (서버·VM)]
                             │
              트래픽 흐름 데이터 수집 → PCE로 전송
```

***

## 6. 한 줄 정리

VEN은 **Illumio Core가 각 워크로드에 설치하는 경량 에이전트로, PCE의 중앙 정책을 OS 방화벽 규칙으로 변환·시행하고 트래픽 흐름을 수집해 돌려보내는 "정책 실행기"** 다.
