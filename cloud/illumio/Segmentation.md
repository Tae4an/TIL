# Illumio Segmentation이란

## 1. 개념 요약

Illumio Segmentation(제로 트러스트 세그멘테이션, ZTS)은 **데이터센터·클라우드·엔드포인트 전반의 워크로드 간 통신을 가시화하고, 필요한 통신만 허용하는 최소 권한 정책을 적용해 공격자의 횡방향 이동(Lateral Movement)을 차단하는 마이크로세그멘테이션 플랫폼**이다. "절대 신뢰하지 말고 항상 확인하라"는 제로 트러스트 원칙을 네트워크 내부 워크로드 간 통신 수준에서 실현한다.

***

## 2. 왜 Segmentation이 필요한가

기존 경계 보안(방화벽·VPN)은 외부 침입은 막지만 **일단 내부에 침투하면 공격자가 내부 서버 간 자유롭게 이동하는 것을 막지 못한다**. 랜섬웨어가 1대 서버를 감염시킨 뒤 수백 대로 확산하는 구조가 바로 이 때문이다. Illumio ZTS는 이 내부 이동 경로 자체를 세그멘테이션 정책으로 차단해, **침해가 발생해도 피해 범위를 특정 세그먼트 안에 봉쇄**한다.

***

## 3. 핵심 동작 원리

- **에이전트(VEN) 설치**
  - 각 워크로드(서버·VM·컨테이너)에 경량 에이전트 VEN(Virtual Enforcement Node)을 설치한다.
  - VEN은 커널 모드에서 동작하지 않고, 호스트 OS의 **기존 방화벽(iptables·Windows Firewall 등)을 활용**해 정책을 적용한다.
  - 에이전트가 장애를 일으켜도 통신 자체가 중단되지 않는 안전한 구조다.

- **PCE(Policy Compute Engine) — 중앙 정책 엔진**
  - 모든 VEN의 트래픽 흐름 데이터를 수집·분석하고 정책을 계산한다.
  - 정책이 변경되면 PCE가 변환된 방화벽 규칙을 각 VEN에 배포한다.

- **레이블 기반 정책 모델**
  - IP·서브넷이 아닌 **애플리케이션·환경·역할·위치 레이블**로 워크로드를 정의한다. (예: `App=웹`, `Env=Prod`, `Role=DB`)
  - 정책은 "Prod 환경의 Web이 Prod 환경의 DB의 3306 포트에 접근 허용" 형태로 선언적으로 작성된다.
  - IP가 바뀌거나 클라우드 환경이 바뀌어도 레이블만 맞으면 정책이 자동 적용된다.

- **선언적 정책 모델(Declarative Policy)**
  - 관리자가 "원하는 상태(Policy)"를 선언하면, PCE가 이를 실제 방화벽 규칙(ACL)으로 자동 변환한다.
  - 관리자가 직접 수천 개의 방화벽 규칙을 작성할 필요가 없다.

***

## 4. ZTS 도입 3단계 여정

| 단계 | 이름 | 내용 |
|---|---|---|
| 1단계 | **Visualize (가시화)** | Visibility Only 모드로 전체 워크로드 간 트래픽 흐름 파악, Illumination Map 생성 |
| 2단계 | **Protect (링펜싱)** | Enforcement Boundary(Selective)로 고위험 포트 차단, 핵심 자산 주변 링펜싱 적용 |
| 3단계 | **Enforce (완전 적용)** | Full Enforcement로 화이트리스트 정책 완전 강제, 최소 권한 마이크로세그멘테이션 완성 |

***

## 5. Illumio 제품 라인

- **Illumio Core**: 데이터센터·프라이빗 클라우드·하이브리드 환경의 서버·VM 워크로드 세그멘테이션 (주력 제품)
- **Illumio Cloud**: AWS·Azure·GCP 등 퍼블릭 클라우드 네이티브 워크로드 세그멘테이션
- **Illumio Endpoint**: 랩탑·데스크탑 등 엔드포인트까지 동일한 정책으로 확장
- **Illumio for OT/IoT**: 산업제어시스템·IoT 디바이스 세그멘테이션

***

## 6. 주요 기능 요약

- **Illumination Map**: 전체 워크로드 간 트래픽 흐름을 실시간 그래프로 시각화
- **링펜싱**: 핵심 앱·자산 주변에 보안 울타리를 쳐서 빠른 초기 보호 실현
- **Enforcement Boundary**: Full Enforcement 전, Deny 규칙으로 고위험 포트·환경 간 트래픽 즉시 차단
- **Full Enforcement**: 화이트리스트 방식으로 허용된 통신만 허용, 나머지 전부 차단
- **AI 정책 추천**: 실시간 트래픽 데이터와 AI를 결합해 최적 세그멘테이션 정책 자동 추천
- **컴플라이언스 리포팅**: PCI-DSS·HIPAA·NIST ZTA 요건 충족 여부를 자동 검증·보고

***

## 7. Illumio ZTS vs 전통 네트워크 세그멘테이션 비교

| 항목 | 전통 VLAN/방화벽 | Illumio ZTS |
|---|---|---|
| 정책 기준 | IP·서브넷·포트 | 레이블(앱·환경·역할) |
| 정책 변경 영향 | IP 변경 시 전체 재작업 | 레이블 자동 추적, 재작업 불필요 |
| 가시성 | 제한적 | 프로세스·포트·앱 수준 실시간 맵 |
| 클라우드 지원 | 어려움 | 멀티 클라우드 동일 정책 적용 |
| 에이전트 의존 | 없음 | VEN 에이전트 필요 |
| 장애 시 통신 | 정상 | VEN 장애 시도 통신 유지 |

***

## 8. 한 줄 정리

Illumio Segmentation은 **레이블 기반 선언적 정책과 에이전트(VEN)·PCE 아키텍처로 데이터센터·클라우드·엔드포인트 전반의 워크로드 간 불필요한 통신을 차단하여 랜섬웨어 확산과 공격자의 횡방향 이동을 봉쇄하는 제로 트러스트 마이크로세그멘테이션 플랫폼**이다.
