# 일루미오(Illumio)란?

## 개념 정의

Illumio(일루미오)는 **제로 트러스트 세그멘테이션(Zero Trust Segmentation, ZTS)을 전문으로 하는 사이버 보안 회사**로, 공격이 네트워크 안으로 들어왔을 때 측면 이동(Lateral Movement)을 차단하는 데 특화된 마이크로세그멘테이션 플랫폼을 제공한다.  
Forrester가 "최초의 마이크로세그멘테이션 전문 기업"으로 평가할 만큼, 이 분야의 선구자·선도 기업으로 인식되고 있음.  

***

## 일루미오가 하는 일 한 줄로

"이미 해커가 내부에 들어왔다고 가정하고, **감염된 워크로드에서 다른 시스템으로 번져나가지 못하도록 워크로드 단위로 통신을 통제·격리**해서, 침해의 폭발 반경을 최소화하는 제로 트러스트 세그멘테이션 플랫폼"이라고 보면 된다.  

***

## 핵심 개념: 마이크로세그멘테이션

기존 네트워크 보안은 "경계(Perimeter)"를 막는 데 집중했다.  
한번 경계를 뚫으면 내부는 상대적으로 자유롭게 이동할 수 있는 구조였기 때문에, 랜섬웨어나 APT 공격이 내부에서 빠르게 번지는 문제가 있었음.  

마이크로세그멘테이션은 이를 보완하기 위해  
- 네트워크를 더 작은 보안 구역으로 나누고  
- 각 워크로드(서버, VM, 컨테이너, 엔드포인트 등) 단위로 최소 권한 통신만 허용하는 정책을 적용.  
- 즉, "A 서버는 B 서버와만 특정 포트로 통신 허용, 나머지는 기본 차단"하는 방식.  

이렇게 하면 공격자가 하나의 워크로드를 침해하더라도, 다른 시스템으로 확산(Lateral Movement)하기 어려워짐.  

***

## 일루미오 주요 제품·기능

**Illumio Segmentation (마이크로세그멘테이션)**

- 에이전트(VEN, Virtual Enforcement Node)를 워크로드에 설치해, 워크로드 간 트래픽 흐름을 실시간 수집·가시화.  
- 수집한 트래픽 맵을 기반으로 보안 정책 자동 추천·적용.  
- 정책은 IP/포트뿐 아니라 라벨(애플리케이션 종류, 환경, 역할 등)로 정의하므로, 인프라 변화에도 정책 자동 추적.  
- 데이터센터, 클라우드(AWS/Azure/GCP), 컨테이너(쿠버네티스), 엔드포인트 전반을 커버.  

**Illumio Insight (가시성·위험 분석)**

- 보안팀이 어떤 워크로드가 어떤 포트/프로토콜로 통신하는지 실시간 파악.  
- 비정상 트래픽, 허가되지 않은 측면 이동 경로, 랜섬웨어 확산 가능 경로를 시각화해 우선순위 결정 지원.  

**침해 시 격리(Quarantine)**

- 랜섬웨어 등 위협이 감지되면, 감염된 워크로드에 "격리됨" 라벨 적용 → 해당 워크로드를 즉시 나머지 환경에서 분리.  
- 전체 시스템 중단 없이 외과적 격리로 피해 범위를 최소화하는 원클릭 대응 지원.  

***

## 어디에 쓰는지 (주요 활용)

- 랜섬웨어 차단  
  - 침해된 워크로드에서 다른 서버·시스템으로 확산되는 것을 마이크로세그멘테이션으로 차단.  
- 제로 트러스트 아키텍처 구현  
  - "내부망이라도 기본 신뢰 없이, 필요한 통신만 허용" 원칙을 전체 하이브리드 환경에 적용.  
- 규정 준수·컴플라이언스  
  - PCI DSS, HIPAA 등 규제에서 요구하는 네트워크 세그멘테이션 증거 확보.  
- 클라우드 마이그레이션 보안  
  - 멀티 클라우드·하이브리드 환경 전반에서 일관된 세그멘테이션 정책 유지.  

***

## ZTNA와의 관계

- ZTNA(Zero Trust Network Access): 사용자/기기 → 애플리케이션 간 **접근 제어(Access)** 에 초점.  
- 일루미오 ZTS: 이미 접근한 뒤, 네트워크 내 **워크로드 간 통신·확산 차단(Containment)** 에 초점.  

두 기술은 서로 다른 레이어를 커버하기 때문에 상호 보완적이고, ZTNA + 일루미오 ZTS 조합으로 "입구 통제 + 내부 확산 차단"을 동시에 구현하는 아키텍처가 많이 언급된다.
