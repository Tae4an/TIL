# SOAR이란?

## 1. 개념 요약

SOAR(Security Orchestration, Automation and Response)은 **보안 오케스트레이션·자동화·대응** 플랫폼이다.  
**다양한 보안 도구를 통합하고 반복 작업을 자동화해 SOC(Security Operations Center)의 인시던트 대응을 빠르고 효율적으로 만드는 "보안 자동화 허브"** 다.

***

## 2. 특징

- **오케스트레이션(Orchestration)**  
  - SIEM, EDR, 방화벽 등 보안 도구 간 워크플로우 조율  
  - 중앙 대시보드에서 모든 보안 이벤트를 통합 관리  

- **자동화(Automation)**  
  - 경고 분류, 이벤트 보강, 플레이북 자동 실행  
  - 반복 작업(호스트 격리, 티켓 생성 등) 자동화로 MTTR 단축  

- **대응(Response)**  
  - 위협 인텔리전스 연동, 자동 격리·차단 액션  
  - 분석가 경험 수준 상관없이 일관된 대응 보장  

- **플레이북 기반**  
  - 사전 정의된 대응 시나리오로 표준화된 프로세스 운영  

***

## 3. 어디에 쓰이는가

- **SOC 효율화**  
  - 경보 피로 감소, 분석가 생산성 향상  

- **인시던트 대응 자동화**  
  - SIEM 경고 → 자동 조사 → 격리 → 보고까지 플레이북 실행  

- **다중 도구 통합**  
  - Splunk+EDR+방화벽을 SOAR 하나로 연결 운영  

- **규정 준수 지원**  
  - 대응 기록 자동화, 감사 추적 용이  

***

## 4. SIEM과의 차이

| 항목 | SIEM | SOAR |
|------|------|------|
| **주요 역할** | 로그 수집·분석·경고 | 자동화 대응·워크플로우 실행 |
| **자동화 수준** | 낮음(경고 생성 위주) | 높음(액션 자동 실행) |
| **통합 범위** | 로그 중심 | 모든 보안 도구+IT 시스템 |
| **사용자** | 분석가(조사 위주) | 운영자(자동 대응 위주) |
| **결과** | MTTD 단축 | MTTR 단축 |

SIEM은 "탐지"에 강하고 SOAR은 "대응"에 특화 → 보통 함께 사용.

***

## 5. 한 줄 정리

SOAR은 **보안 도구를 오케스트레이션하고 자동 대응을 실행**해 SOC의 인시던트 처리 속도와 효율을 극대화하는 "현대 보안 운영 자동화 플랫폼"이다.
