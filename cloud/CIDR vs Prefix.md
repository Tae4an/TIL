# CIDR vs Prefix란?

## 1. 개념 요약

CIDR은 **IP 주소를 유연하게 할당하고 라우팅하는 방식**이고, Prefix는 그 CIDR을 표현할 때 쓰는 **네트워크 비트 길이 표기**다.  
쉽게 말해, **CIDR은 개념/방식이고 Prefix는 그걸 적는 숫자(/24, /16 같은 표기)**다.

***

## 2. 차이점

- **CIDR**
  - Classless Inter-Domain Routing의 약자
  - 클래스 A/B/C처럼 고정된 네트워크 구분 대신, 필요한 만큼 IP 범위를 나누는 방식
  - 라우팅 효율을 높이고 IP 낭비를 줄이는 데 목적이 있다

- **Prefix**
  - IP 주소에서 네트워크 부분이 몇 비트인지 나타내는 값
  - 예: `/24`는 앞의 24비트가 네트워크라는 뜻
  - 서브넷 마스크를 짧게 표현한 것과 같다

***

## 3. 예시

- `192.168.1.0/24`
  - CIDR 표기 전체
  - Prefix는 `/24`
  - 총 256개 주소 범위 중 네트워크 주소와 브로드캐스트 주소를 제외하고 호스트가 사용한다

- `10.0.0.0/16`
  - CIDR 표기 전체
  - Prefix는 `/16`
  - `/24`보다 더 큰 네트워크 범위를 의미한다

***

## 4. 실무에서의 의미

- **AWS VPC 설정**
  - VPC CIDR 블록을 정할 때 `10.0.0.0/16`처럼 Prefix를 함께 쓴다
- **라우팅**
  - 더 긴 Prefix가 더 구체적인 경로를 뜻한다
  - 예: `/24`가 `/16`보다 우선 매칭될 수 있다
- **서브넷 설계**
  - 원하는 호스트 수에 맞게 Prefix 길이를 조정한다

***

## 5. 한 줄 정리

CIDR은 **IP 주소를 나누는 방식**이고, Prefix는 그 CIDR을 나타내는 **`/24`, `/16` 같은 네트워크 비트 길이 표기**다.
