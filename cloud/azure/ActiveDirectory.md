# Azure Active Directory란?

## 개념 정의
Azure Active Directory(Azure AD)는 Microsoft에서 제공하는 **클라우드 기반 ID 및 액세스 관리 서비스**다. 기업/조직의 사용자, 그룹, 애플리케이션, 디바이스 등 모든 리소스의 로그인, 권한, 보안 정책을 일원화해 관리할 수 있다.

쉽게 말하면 “클라우드 환경용 통합 권한/인증 플랫폼”으로, 온프레미스 Active Directory와는 달리 인터넷을 통한 SaaS/클라우드·멀티 디바이스 환경에서 안전한 사용자 인증과 권한 관리, SSO(싱글 사인온), MFA(다중 인증) 제공이 특징이다.

## 핵심 특징

**통합 인증 및 SSO**
- 하나의 계정으로 Office 365, Azure 플랫폼, 수많은 SaaS 애플리케이션(예: Salesforce, Dropbox, ServiceNow 등)에 모두 로그인 가능
- 사용자별·그룹별 SSO 정책, 자동화된 권한 부여

**클라우드 & 온프레미스 통합 관리**
- 로컬(온프레미스) AD와 클라우드 Azure AD 연동(AD Connect 또는 Entra ID Connect)
- 하이브리드 시나리오 지원: 기존 AD 계정으로 클라우드 리소스 접근 및 관리

**고급 보안 및 정책 제어**
- MFA(다중 인증), 조건부 액세스(Conditional Access), 비밀번호 관리 및 정책 설정
- 보안 위험 탐지(Identity Protection), 계정/로그인 시도 실시간 모니터링
- 권한 위임, 관리자 역할 및 리소스별 액세스 세분화

**애플리케이션 및 API 연동**
- OAuth2.0, OpenID Connect 등 최신 인증 프로토콜 지원
- 기업 내 커스텀 앱·API에 쉽게 통합, 외부 협력사 및 게스트 사용자 관리

**디바이스 등록 및 관리**
- 모바일, PC, IoT 등 디바이스 기반 정책 제어(Domain Join, MDM 등)
- 조건 충족 시에만 접근 허용 등 세밀한 정책 적용 가능

## 주요 구성 요소

- **사용자(User):** 조직 내부 사용자/관리자/외부 게스트, 그룹별 정책 부여 가능
- **그룹(Group):** 부서, 프로젝트 단위 등 사용자 묶음별 권한 관리
- **애플리케이션(Application):** SaaS/커스텀 앱 모두 Azure AD 인증 기반 접근 제어
- **디바이스(Device):** 기업의 모든 단말기를 Azure AD에 등록·관리
- **도메인(Domain):** 여러 도메인/서브도메인 인증 일원화
- **관리자 역할(Role):** 각종 관리 권한(예: 보안 관리자, 애플리케이션 관리자 등) 별 분리

## Active Directory(온프레미스) vs Azure AD 비교

|           | 온프레미스 AD        | Azure Active Directory     |
|-----------|---------------------|---------------------------|
| 구축 위치 | 자체 서버, 로컬      | 클라우드, 글로벌 인프라    |
| 접근 방식 | LAN 내부            | 인터넷 및 클라우드         |
| 인증 방식 | Kerberos, NTLM 등   | OAuth2, SAML, OpenID 등    |
| SSO       | 제한적              | 클라우드·SaaS 폭넓게 제공  |
| MFA       | 별도 솔루션 필요    | 내장 기능 제공             |
| 관리 범위 | PC/서버, 도메인     | 사용자·그룹·앱·디바이스    |
| 연동      | 클라우드 연동 어려움 | 온프레미스/클라우드 모두 지원|

## 주요 기능

- SSO(싱글 사인온): 수많은 앱/서비스에 한 번 로그인으로 통합 접근
- 조건부 액세스(Conditional Access): 위치, 디바이스, 위험도 등 조건별 인증 정책
- 계정·비밀번호 연동·동기화: 온프레미스 AD 계정과 자동 동기화
- 외부 사용자(게스트) 관리: 파트너·협력사·고객의 계정 초대·권한 부여
- 인증 로그 분석·보안 알림: 관리자에게 실시간 모니터링·이상 징후 경고
- API/애플리케이션 통합: OAuth2·SAML 기반 인증 직접 구현

## 주요 활용 사례

- 기업·조직 내 Office 365·Azure·SaaS 서비스 통합 로그인 및 권한 관리
- 하이브리드 클라우드 환경의 사용자 계정 자동 연동 및 관리
- 외부 협력사 맞춤 게스트 계정 발급/관리
- 커스텀 앱·API에 Azure AD 기반 인증 적용(기업 내부 솔루션 연동)
- 심층 보안 정책·다중 인증(MFA)·자동화 규칙을 통한 보안 강화

## 기본 워크플로

### 1. Azure AD 테넌트 생성 및 계정 등록
- Azure Portal → Azure Active Directory → 테넌트 및 사용자 추가
- 도메인 연결, 사용자·그룹 등록, 관리자 역할 설정

### 2. Office 365/SaaS 앱 SSO 통합
- Azure AD 대시보드에서 애플리케이션 등록, SSO 설정

### 3. MFA/조건부 액세스 정책 적용
- Azure AD 콘솔 → 보안 → 조건부 액세스 및 MFA 정책 생성

### 4. 온프레미스 AD와 동기화
- Azure AD Connect 설치/설정 → 사용자·그룹·비밀번호 동기화 활성화

## 비용 구조

- Free: 기본 사용자·그룹·앱 관리, SSO 기능 무료
- Premium P1/P2: 고급 기능(조건부 액세스, MFA, 위험기반 인증, Identity Protection 등)은 라이선스 추가 구매 필요
- 사용량·기능·테넌트 규모에 따라 월별 과금

## 보안 모범 사례

- 최소 권한 원칙 적용, 역할별 세밀한 액세스 분리
- 모든 관리자 계정에 MFA 적용
- 조건부 액세스 정책·보안 인텔리전스 실시간 활성화
- 외부 사용자·기기 접근 철저 모니터링 및 알림 설정
- 온프레미스 AD와 클라우드 계정 정기적 연동·감사

Azure Active Directory는 클라우드·하이브리드 환경에서 모든 계정과 접근 권한의 중심축으로, 조직의 생산성·보안·운영 효율성을 혁신적으로 개선하는 핵심 서비스다.
