# Azure Resource Group이란?

## 개념 정의
Azure Resource Group(리소스 그룹)은 Microsoft Azure에서 **관련 리소스들을 논리적으로 묶어 관리하는 단위**다. VM, DB, 네트워크, 스토리지 등 같은 프로젝트, 같은 환경, 같은 서비스에 속하는 리소스들을 하나의 그룹으로 모아 운영·배포·권한·비용·삭제 등을 일괄적으로 관리할 수 있다.

쉽게 말해 “Azure에서 프로젝트별 인프라를 한 곳에 모아 관리하는 컨테이너”이며, 배포·삭제·권한 관리·비용 분석의 중심 축 역할을 한다.

## 핵심 특징

**논리적 묶음**
- 같은 리소스(서버, 네트워크, DB, 스토리지 등)를 목적/프로젝트/환경별로 그룹화
- 리소스 그룹 단위로 생성·배포·삭제 처리 가능
- 배포 단순화, 관리 효율성 강화

**권한·정책 관리**
- 그룹 단위로 RBAC(역할 기반 액세스 제어) 적용 가능  
- 한 번의 권한 설정으로 그룹 내 모든 리소스 관리자·개발자 접근 제어 가능
- 정책이나 태그도 그룹 단위로 적용

**비용·청구관리**
- 리소스 그룹별 비용 분석 및 예산 할당
- 그룹별 태그 결합으로 부서·서비스별 비용 분류 가능

**리소스 종속성 및 일괄 배포**
- ARM(리소스 매니저) 템플릿을 통해 리소스 그룹 전체를 코드로 자동 배포·업데이트
- 환경별(dev/stage/prod) 분리 배포·삭제 효과적

**지역 지정 및 확장성**
- 리소스 그룹의 위치는 그룹 메타데이터 저장용(실제 리소스 위치와 무관)
- 동일 그룹 내 리소스가 여러 리전에 분산될 수 있음

## 주요 활용 사례

- 하나의 서비스(웹앱, DB, 스토리지, 네트워크) 전체 리소스를 한 그룹에 묶어 관리 및 일괄 삭제
- 개발·테스트·운영 환경별로 그룹화하여 분리 관리
- 프로젝트/부서/클라이언트 별로 그룹을 구분해 업무와 비용 원천 분명히 구분
- RBAC과 태그 및 정책을 그룹단위로 적용하여 일관된 보안·운영 기준 확보
- ARM 템플릿으로 새 환경(리소스 그룹 단위) 코드 배포·복구·삭제

## 기본 워크플로 및 관리 예시

### 1. 리소스 그룹 생성
```bash
az group create --name myproject-rg --location koreacentral
```
- 이후 모든 리소스(VM, DB, Storage 등)를 해당 그룹에 생성·등록

### 2. RBAC 및 정책 적용
- Azure Portal 또는 CLI에서 그룹 단위로 관리자·개발자 권한/정책 부여
- 태그도 그룹에 일괄 적용 가능

### 3. 비용 및 태그 관리
- 비용 관리·분석에서 그룹별 예산/비용 수집·보고

### 4. 일괄 삭제/정리
```bash
az group delete --name myproject-rg
```
- 그룹 삭제 시, 그룹 내 모든 리소스도 함께 삭제

## 보안 및 운영 모범 사례

- 환경별(개발/운영 등) 그룹 분리, 권한·정책 효과적으로 적용
- 태그 활용으로 비용 분석·관리 자동화
- 리소스 그룹별 ARM 템플릿 및 IaC로 코드 기반 관리
- 실제 리소스 위치와 그룹 위치 구분, 중요 데이터 리소스 지역 주의

Azure Resource Group은 클라우드 인프라 관리와 배포의 핵심 시작점이자, 지속적 운영 자동화와 비용·권한 통제의 중심적인 역할을 하는 필수 개념이다.
