# SASE(Secure Access Service Edge)란?

## 개념 정의

SASE(보안 액세스 서비스 엣지, Secure Access Service Edge)는 **네트워크(SD‑WAN)와 보안(SWG, CASB, ZTNA, FWaaS 등) 기능을 하나의 클라우드 기반 서비스로 통합한 아키텍처**이다.  
사용자·지사·디바이스·앱이 어디에 있든, 가까운 클라우드 엣지에서 네트워크 연결과 제로 트러스트 보안을 동시에 제공하는 “클라우드형 네트워크+보안 통합 모델”이라고 보면 됨.  

***

## SASE가 하는 일 한 줄로

“지사·재택·모바일·클라우드가 뒤섞인 환경에서, **모든 트래픽을 클라우드 엣지로 모아서** 거기서 네트워크 경로 최적화(SD‑WAN)와 보안 검사(SWG, CASB, ZTNA, FWaaS)를 한 번에 수행하게 해주는 아키텍처”라고 이해하면 된다.  

***

## SASE의 핵심 구성요소

### 1) SD‑WAN (Software‑Defined WAN)

- 지사·데이터센터·클라우드 간 연결을 인터넷/전용선 등 여러 회선을 묶어 소프트웨어로 제어.  
- 애플리케이션별 최적 경로 선택, 대역폭 집계, 자동 장애 조치, 지연·패킷 손실 보정 등 제공.  

### 2) SSE(Security Service Edge, 보안 서비스 엣지)

SASE에서 **보안 부분**을 담당하는 축이며, 대표적으로 다음 기능들을 포함한다.

- SWG (Secure Web Gateway)  
  - 웹/인터넷 트래픽 프록시, URL 필터링, 악성 사이트/콘텐츠 차단, 웹 기반 위협 방지.  
- CASB (Cloud Access Security Broker)  
  - SaaS 사용 가시성, 섀도 IT 탐지, 클라우드 앱 내 데이터 보호·접근 제어.  
- ZTNA (Zero Trust Network Access)  
  - VPN 대신 사용자·디바이스·컨텍스트 기반으로 특정 애플리케이션에만 최소 권한 접근 허용.  
- FWaaS (Firewall as a Service)  
  - 클라우드 기반 L3~L7 방화벽/IPS, DNS 보안 등 네트워크 계층 보안.  

정리하면, **SASE = SD‑WAN + SSE(SWG + CASB + ZTNA + FWaaS …)** 인 구조.  

***

## 동작 방식 (흐름)

1. 사용자가 지사·집·모바일·카페 어디에서든 인터넷/업무 앱에 접속 시도.  
2. 단말/지사 장비/에이전트가 트래픽을 가장 가까운 SASE PoP(클라우드 엣지)로 전송.  
3. SASE 플랫폼이  
   - SD‑WAN 기능으로 최적 경로를 선택하고,  
   - 동시에 SWG/CASB/ZTNA/FWaaS로 보안 정책·제로 트러스트 검증을 수행.  
4. 허용된 트래픽만 SaaS, 인터넷, 기업 데이터센터, 클라우드 VPC/VNet 등 목적지로 전달.  
5. 모든 활동과 위협 이벤트는 중앙 콘솔에서 가시화·분석·정책 관리.  

***

## 왜 쓰는지 (주요 목적)

- 지사·재택·모바일 사용자와 멀티 클라우드 환경에서 **일관된 네트워크·보안 정책** 적용.  
- 온프레미스 방화벽·프록시·VPN·WAN 라우터 등을 따로 관리하던 것을 **클라우드 기반 단일 서비스**로 단순화.  
- 제로 트러스트 원칙(항상 검증, 최소 권한)을 네트워크·보안 전반에 적용해,  
  위치에 상관없이 안전한 애플리케이션 접근을 제공하기 위한 아키텍처로 활용된다.  

***

## SASE vs SSE 간 관계

- SSE = 보안 서비스만 모은 것(SWG, CASB, ZTNA, FWaaS 등).  
- SASE = **SD‑WAN(네트워크) + SSE(보안)** 를 합친 전체 프레임워크.  
- 기존 WAN은 유지하면서 보안만 클라우드로 빼면 SSE 중심,  
  네트워크까지 함께 재설계하면 SASE 도입이라고 이해하면 편하다.
