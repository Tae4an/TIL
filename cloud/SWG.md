# SWG(Secure Web Gateway)란?

## 개념 정의

SWG(Secure Web Gateway, 보안 웹 게이트웨이)는 사용자의 **웹/인터넷 트래픽을 중간에서 프록시처럼 가로채 검사·필터링하여, 악성 사이트·피싱·멀웨어·유해/비업무 사이트 접속을 차단하고 조직의 웹 사용 정책을 강제하는 보안 솔루션**이다.  
사용자와 인터넷 사이에 위치해 URL 필터링, 악성코드 검사, 데이터 유출 방지(DLP), 애플리케이션 제어 등을 수행하는 “웹 출입문 보안 장치/서비스”로 이해하면 됨.  

***

## SWG가 하는 일 한 줄로

“직원이 웹이나 SaaS에 접속할 때 **먼저 SWG를 거치게 해서**, 위험한 사이트·파일·행위를 걸러내고 허용된 웹/클라우드 서비스만 안전하게 쓰게 만드는 인터넷 관문 보안”이라고 보면 된다.  

***

## 핵심 특징

**웹 트래픽 프록시/게이트웨이 역할**

- 사용자는 인터넷에 직접 나가지 않고, SWG를 경유하여 접속.  
- SWG가 HTTP/HTTPS 트래픽을 가로채 URL·도메인·콘텐츠·파일을 검사한 뒤, 정책에 맞으면 목적지 사이트로 연결, 아니면 차단 페이지로 응답.  

**다계층 웹 보안 기능 통합**

- URL 필터링, 안티바이러스/맬웨어 탐지, SSL/TLS 복호화 검사, DLP, 애플리케이션 제어 등 웹 관련 보안 기능을 한 제품/서비스에 통합.  
- 일부 솔루션은 샌드박스, 원격 브라우저 격리(RBI) 같은 고급 기능까지 포함.  

**정책 기반 웹 사용 통제**

- 사용자/그룹/디바이스/위치별로 다른 웹 사용 정책 적용 가능.  
- 업무 시간 SNS 제한, 스트리밍 대역폭 제한, 특정 부서만 특정 SaaS 허용 같은 세밀한 정책 설계 가능.  

***

## 핵심 기능

**1. URL 필터링**

- 요청된 URL을 카테고리/평판 DB와 비교해 악성·유해·비업무 사이트 차단.  
- 도메인·경로·카테고리(성인, 도박, P2P, SNS 등) 기반 허용/차단.  

**2. 악성코드·위협 탐지**

- 웹 페이지·스크립트·파일 다운로드를 검사해 멀웨어, 랜섬웨어, 피싱, 익스플로잇 등을 탐지/차단.  
- 일부는 샌드박스로 의심 파일을 격리 실행해 동적 분석 수행.  

**3. SSL/TLS 복호화 및 검사**

- HTTPS 트래픽을 복호화(SSL inspection)해 암호화된 채널 내부에 숨은 악성코드·데이터 유출을 탐지.  
- 조직 정책에 따라 특정 사이트(금융, 개인용 사이트 등)는 복호화 예외 처리도 가능.  

**4. DLP(Data Loss Prevention) 연계**

- 업로드/포스트/폼 제출 등 웹을 통한 데이터 전송 내용을 검사.  
- 주민번호, 카드번호, 고객 식별 정보, 내부 기밀 키워드 등 민감 정보 외부 유출 시도 탐지·차단.  

**5. 웹·SaaS 애플리케이션 제어**

- 특정 웹/클라우드 앱(예: 개인용 클라우드 저장소, 비인가 SaaS) 사용 차단 또는 읽기 전용 허용.  
- 그림자 IT(Shadow IT) 탐지, 비인가 앱 사용 억제.  

**6. 로깅·리포팅·컴플라이언스**

- 사용자별/그룹별 웹 접속 로그, 차단 이력, 위협 탐지 내역 기록.  
- 규제·감사 대응용 보고서, 대시보드 제공.  

***

## 동작 방식 (흐름)

1. 사용자가 브라우저로 웹 사이트 또는 SaaS에 접속 시도.  
2. 트래픽이 PAC 파일/에이전트/네트워크 리다이렉트 등을 통해 SWG로 우회.  
3. SWG가 URL, 도메인, 요청 헤더, 콘텐츠, 파일, 사용자·기기 정보 등을 정책·위협 DB와 대조.  
4. 허용 정책 및 위협 없음 → 실제 목적지에 연결 후 응답을 사용자에게 전달.  
5. 차단 대상 또는 위협 발견 → 접속 차단 페이지/경고 페이지 표시, 로그 기록.  

***

## SWG vs CASB 관점

- SWG:  
  - “인터넷 전체”를 대상으로 웹 트래픽/사이트 접속을 보호하는 게이트웨이.  
  - URL·웹 트래픽 기반 제어에 강점.  

- CASB(Cloud Access Security Broker):  
  - 특정 **클라우드 서비스(SaaS)** 사용과 그 안의 **데이터·행위**에 초점을 맞춘 통제.  
  - API 연동, 세밀한 SaaS 내 동작 제어(예: 특정 폴더 공유 금지, 외부 공유 탐지 등)에 강점.  

실무에서는 SWG와 CASB를 함께 사용해,  
“웹 전체 보안(SWG) + SaaS 내부 세밀 제어(CASB)” 조합으로 클라우드 시대 웹·SaaS 보안을 구성하는 경우가 많다.  

***

## SSE/SASE와의 관계

- SSE(Security Service Edge)의 핵심 구성 요소 중 하나가 SWG.  
- SSE는 SWG + CASB + ZTNA 등을 클라우드 기반 서비스로 제공해,  
  위치와 상관없이 사용자–웹/SaaS/사내 앱 접근을 보호하는 구조.  
- SASE에서는 SD‑WAN(네트워크) + SSE(보안)를 통합한 더 큰 개념 안에서,  
  SWG가 “인터넷 출구” 보안 역할을 담당한다고 보면 된다.
