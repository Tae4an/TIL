# 헤어피닝(Hairpinning)이란?

## 개념 정의

헤어피닝(Hairpinning)은 **NAT 환경에서 내부 클라이언트가 같은 내부망에 있는 서버를 “공인 IP/도메인”으로 접속하려 할 때, NAT 장비가 그 트래픽을 다시 내부로 꺾어 돌려보내 통신이 가능하도록 만드는 동작**을 말한다.  
이때 패킷이 NAT 장비까지 갔다가 다시 내부로 되돌아가는 경로가 머리핀(hairpin) 모양이라서 Hairpin NAT, NAT Loopback, NAT Reflection이라고도 부른다.  

***

## 예시로 이해하기

- 내부 웹서버: 192.168.0.204  
- 라우터 내부 IP: 192.168.0.1  
- 라우터 공인 IP: 203.0.113.10  
- 도메인: example.com → 203.0.113.10으로 DNS 설정  

상황:  
- 외부 사용자가 `example.com`으로 접속 → 공인 IP 203.0.113.10 → 포트포워딩으로 192.168.0.204에 잘 연결.  
- 내부 PC(192.168.0.44)가 `example.com`으로 접속 → DNS는 역시 203.0.113.10을 반환.  

이때 헤어핀 NAT가 동작하면:  
1. 내부 PC → 203.0.113.10(공인 IP)로 요청 전송.  
2. NAT 장비가 “이 공인 IP는 내 외부 주소고, 실제로는 내부 서버 192.168.0.204로 포워딩해야 하는 트래픽”임을 인지.  
3. 목적지 IP를 192.168.0.204로 변환하고, 소스 IP도 적절히 변환한 뒤(보통 NAT 자신의 내부 IP),  
4. 다시 내부망으로 “꺾어서” 보내 웹서버에 전달.  
5. 응답도 NAT를 거쳐 클라이언트에게 돌아가므로 통신 성립.  

이렇게 해서 **내부·외부 어디서든 동일한 도메인/공인 IP로 동일하게 접속**할 수 있게 된다.  

***

## 왜 쓰는지 (목적)

- 내부 사용자도 외부와 같은 URL/도메인으로 서비스에 접속 가능하게 하기 위해.  
  - 예: 사내·외부 모두 `app.company.com`으로 접속, 내부 전용 주소 따로 기억할 필요 없음.  
- 애플리케이션 설정·링크·콜백 URL 등에 공인 도메인만 쓰고, 내부/외부 환경을 나누지 않고 통일하기 위해.  
- NAT 뒤에 있는 여러 서비스(NAS, 웹서비스 등)에 대해, 내부에서 DDNS/공인 도메인을 그대로 쓰고 싶을 때.  

***

## 동작 관점 정리

- NAT 앞단(내부)에서 공인 IP/도메인으로 나가는 트래픽을  
- NAT 장비가 “자신의 외부 IP + 포트포워딩 규칙”과 매칭해서,  
- 목적지를 내부 서버 IP로 재매핑하고,  
- 응답이 다시 NAT를 통과하도록 소스 주소도 변환해  
- **내부에서 내부로 향하는 NAT 통신**을 완성하는 것.  

***

## 참고: 지원 여부

- 엔터프라이즈급 방화벽/라우터는 대체로 Hairpin NAT/NAT Loopback 기능을 옵션으로 제공.  
- 일부 저가형 공유기는 지원하지 않거나, 별도 설정이 필요할 수 있어,  
  내부에서 도메인으로 접속이 안 될 때 “헤어핀 NAT 지원 여부”를 먼저 확인하는 경우가 많다.
