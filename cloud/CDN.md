# CDN이란?

## 1. 개념 요약

CDN(Content Delivery Network)은 **전 세계에 분산 배치된 엣지 서버 네트워크를 통해 사용자에게 가장 가까운 위치에서 콘텐츠를 빠르게 전달하는 기술**이다.  
쉽게 말해, **"원본 서버 대신 사용자 가까이 있는 캐시 서버가 콘텐츠를 대신 전달해 주는 배달 네트워크"** 라고 보면 된다.

***

## 2. 동작 원리

- **최초 요청 (Cache Miss)**
  - 사용자 → 가장 가까운 엣지 서버(PoP) 접속
  - 엣지 서버에 캐시 없음 → 원본 서버(Origin)에서 콘텐츠 가져옴
  - 가져온 콘텐츠를 엣지 서버에 저장(캐싱)

- **이후 요청 (Cache Hit)**
  - 동일 콘텐츠 요청 시 엣지 서버 캐시에서 즉시 응답
  - 원본 서버까지 도달하지 않아도 됨 → 빠른 응답

- **TTL(Time To Live)**
  - 캐시 유효 기간 설정, 만료 시 원본 서버에서 갱신

***

## 3. 특징

- **저지연·고속 전송**
  - 물리적 거리 단축으로 응답 시간 대폭 감소

- **원본 서버 부하 감소**
  - 대부분의 요청을 엣지에서 처리 → 트래픽 급증에도 서버 안정적 운영

- **고가용성**
  - 특정 엣지 서버 장애 시 다른 PoP으로 자동 전환(Failover)

- **보안 강화**
  - DDoS 방어, WAF 연계, SSL/TLS 암호화, 봇 차단 기능 제공

- **동적 콘텐츠 가속**
  - 정적 파일뿐 아니라 API 응답, 스트리밍도 최적화 경로로 가속

***

## 4. 어디에 쓰이는가

- **웹사이트·이커머스**
  - 이미지·JS·CSS·HTML 정적 자원을 전 세계에 빠르게 제공

- **동영상 스트리밍**
  - Netflix·YouTube처럼 대용량 영상을 사용자 근처에서 스트리밍

- **게임·소프트웨어 배포**
  - 전 세계 동시 패치 배포 시 원본 서버 과부하 방지

- **AWS CloudFront**
  - AWS의 CDN 서비스, S3·ALB·Lambda@Edge 등과 통합

***

## 5. 원본 서버 직접 제공 vs CDN 비교

| 항목 | 원본 서버 직접 | CDN 사용 |
|---|---|---|
| **응답 속도** | 서버 위치에 따라 느림 | 가까운 엣지에서 빠르게 응답 |
| **서버 부하** | 모든 요청 원본 서버로 집중 | 엣지에서 분산 처리 |
| **가용성** | 서버 장애 시 서비스 중단 | 엣지 Failover로 고가용성 |
| **보안** | 직접 DDoS 노출 | 엣지에서 공격 흡수·차단 |
| **비용** | 대역폭 비용 높음 | 원본 트래픽 감소로 절감 가능 |

***

## 6. 한 줄 정리

CDN은 **전 세계 엣지 서버에 콘텐츠를 분산 캐싱해 사용자에게 가장 가까운 위치에서 빠르고 안전하게 전달하는** "글로벌 콘텐츠 배달 네트워크"다.
