# Sysprep이란?

## 1. 개념 요약

Sysprep(System Preparation)은 **Windows 설치본에서 PC 고유 정보를 제거해 동일한 이미지를 여러 컴퓨터에 배포할 수 있도록 준비하는 Microsoft 공식 시스템 준비 도구**다.  
쉽게 말해, **"마스터 PC 1대의 이미지를 찍어 수백 대에 뿌릴 수 있도록 고유 정보를 초기화하는 도구"** 라고 보면 된다.

***

## 2. 핵심 기능

- **일반화(Generalize)**
  - SID(보안 식별자), 컴퓨터 이름, 하드웨어 설정 등 PC 고유 정보 제거
  - 제거된 정보는 이미지 배포 후 새 PC에서 자동 재생성

- **OOBE(Out-Of-Box Experience) 설정**
  - 배포 후 첫 부팅 시 사용자가 지역·계정·이름 설정 과정을 거치도록 구성
  - 최종 사용자 입장에서 새 PC처럼 동작

- **감사 모드(Audit Mode)**
  - 일반화 전 드라이버·앱 추가 설치·테스트 가능
  - 이미지 커스터마이징 단계에서 활용

- **응답 파일(Unattend.xml) 연동**
  - 자동 응답 파일을 통해 무인(Unattended) 배포 자동화 가능

***

## 3. 어디에 쓰이는가

- **기업 PC 대량 배포**
  - 수십~수천 대 PC에 동일한 골든 이미지를 배포할 때 필수
  - MDT(Microsoft Deployment Toolkit), SCCM과 함께 사용

- **가상화·클라우드 이미지 생성**
  - AWS AMI, Azure VM 이미지 생성 전 Sysprep으로 이미지 일반화 필수
  - 동일 이미지로 만든 VM들이 충돌 없이 동작하도록 보장

- **OEM 초기화 이미지**
  - 삼성·LG 등 제조사가 제품 출고 시 초기 설정 화면 구성에 활용

- **복구 이미지 생성**
  - 공장 초기화용 복구 파티션 이미지 생성에 활용

***

## 4. Sysprep 실행 시 제거되는 정보

| 항목 | 설명 |
|---|---|
| **SID** | 윈도우 보안 식별자, 각 PC마다 고유해야 함 |
| **컴퓨터 이름** | 배포 후 새 이름으로 재설정 |
| **드라이버 캐시** | 하드웨어 특화 드라이버 정보 초기화 |
| **이벤트 로그** | 기존 로그 삭제 |
| **사용자 계정** | 관리자 계정 설정 초기화 |

***

## 5. 주요 실행 명령어

```
sysprep /generalize /oobe /shutdown
```
- `/generalize` : 고유 정보 제거 (일반화)
- `/oobe` : 재부팅 후 초기 설정 화면 진입
- `/shutdown` : Sysprep 완료 후 시스템 종료 (이미지 캡처 준비)

***

## 6. 한 줄 정리

Sysprep은 **Windows 이미지의 PC 고유 정보를 제거해 어떤 하드웨어에도 배포 가능한 범용 이미지로 만드는 도구**로, 기업 대량 배포와 클라우드 VM 이미지 생성의 "필수 전처리 단계"다.
