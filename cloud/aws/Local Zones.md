# AWS Local Zones란

## 1. 개념 요약

AWS Local Zones는 **특정 AWS 리전과 물리적으로 멀리 떨어진 대도시·인구 밀집 지역에 AWS 인프라(컴퓨팅·스토리지·네트워크)를 확장 배치한 인프라 유형**이다. 쉽게 말하면, **"리전은 서울에 있지만, 부산·LA·휴스턴 사용자에게도 한 자릿수 밀리초(single-digit ms) 지연 시간을 제공하기 위해 해당 도시에 미니 AWS 거점을 두는 것"** 이라고 보면 된다.

***

## 2. 핵심 특징

- **리전의 논리적 확장**
  - Local Zone은 독립된 리전이 아니라 **부모 리전(Parent Region)에 연결된 AZ 하나처럼 동작**한다. VPC 서브넷을 Local Zone에 생성해 리전 리소스와 동일한 API·도구로 관리한다.

- **극저지연 목표**
  - 최종 사용자 기준 **한 자릿수 밀리초(< 10ms)** 지연 시간을 목표로 설계된다. Direct Connect 연동 시 1~2ms까지 낮출 수 있다.

- **Opt-in 방식**
  - 기본 비활성화 상태이며, AWS 콘솔에서 원하는 Local Zone을 명시적으로 활성화(Opt-in)해야 사용 가능하다.

- **제한된 서비스 지원**
  - 전체 AWS 서비스가 아닌 일부만 지원한다. EC2, EBS, VPC, ALB, ElastiCache, RDS(일부), EMR 등이 대표적이며, 리전보다 서비스 종류가 적다.

***

## 3. 어디에 쓰이는가

- **미디어·엔터테인먼트**: 영상 편집·라이브 프로덕션에서 제작 스튜디오와 클라우드 인스턴스 간 1~2ms 지연 실현 (예: LA Local Zone 활용)
- **게임**: 실시간 멀티플레이어 게임 서버를 플레이어와 가까운 도시에 배치해 입력 지연 최소화
- **금융 트레이딩**: 초저지연이 필수인 거래 시스템을 거래소 인근에 배치
- **AR/VR·미디어 스트리밍**: 고화질·저지연 스트리밍 처리 서버를 사용자 근처에 운영
- **데이터 레지던시 준수**: 특정 국가·도시에 데이터를 보관해야 하는 규제 요구사항 충족

***

## 4. AWS 인프라 유형 비교

| 항목 | 리전(Region) | 가용 영역(AZ) | Local Zone | Wavelength Zone |
|---|---|---|---|---|
| 위치 | 지리적 대권역 (서울, 버지니아 등) | 리전 내 독립 데이터센터 | 대도시 근교 확장 거점 | 통신사 5G 엣지 |
| 주목적 | 범용 클라우드 | 고가용성 | 극저지연 (도시) | 5G 모바일 극저지연 |
| 지연 시간 목표 | 수십~수백 ms | 수 ms (리전 내) | < 10ms | < 10ms (5G 단말) |
| 서비스 폭 | 전체 | 전체 | 일부 제한 | 매우 제한적 |
| 부모 리전 의존 | 독립 | 리전 종속 | 리전 종속 | 리전 종속 |

***

## 5. 한 줄 정리

AWS Local Zones는 **대도시 사용자 근처에 AWS 컴퓨팅·스토리지 인프라를 확장 배치해 한 자릿수 밀리초의 극저지연을 실현하는 리전 확장 인프라로, 지연 시간에 민감한 미디어·게임·금융 워크로드에 적합하다.**
