# Amazon S3 Files란?

### 1. 개념 요약

Amazon S3 Files는 Amazon S3 버킷을 NFS(Network File System) 프로토콜로 마운트해 **AWS 컴퓨팅 리소스에서 로컬 파일 시스템처럼 사용할 수 있게 해주는 S3의 신규 기능**이다.

기존에는 오브젝트 스토리지(S3)의 비용·내구성과 파일 시스템의 대화형 기능 중 하나를 선택해야 했다. S3 Files는 이 트레이드오프를 제거하여, **S3를 조직 전체 데이터의 중앙 허브로 활용**할 수 있게 한다.

쉽게 말해, "S3 버킷을 EC2·ECS·EKS·Lambda에서 **NFS 공유 디렉토리처럼 마운트해 파일 단위로 읽고 쓸 수 있게 해주는 기능**"이라고 보면 된다.

***

### 2. 특징

**NFS v4.1 기반 파일 시스템 마운트**
- EC2·ECS·EKS·Lambda에서 NFS v4.1 이상 프로토콜로 마운트
- 파일 생성·읽기·업데이트·삭제 등 모든 POSIX 파일 작업 지원
- 애플리케이션 코드 변경 없이 기존 파일 I/O 그대로 사용

**S3 네이티브 저장 및 자동 동기화**
- 파일 시스템 변경 사항이 S3 버킷에 **몇 분 내 자동 반영**
- S3 버킷 변경 사항은 **몇 초~1분 내 파일 시스템에 반영**
- S3 버전 관리·수명 주기 정책·복제 등 S3 기능 그대로 활용

**고성능 스토리지 계층 (내부적으로 EFS 활용)**
- 자주 액세스하는 파일 → 고성능 스토리지에 캐싱, **약 1ms 지연 시간**
- 대용량 순차 읽기 파일 → S3에서 직접 서비스, 처리량 극대화
- 바이트 범위 읽기 → 요청된 바이트만 전송하여 비용 최소화
- 지능형 사전 가져오기(Prefetch) 기능으로 접근 패턴 예측

**다중 컴퓨팅 리소스 동시 접근**
- 여러 EC2·컨테이너·Lambda가 **동일 파일 시스템에 동시 접근** 가능
- NFS 수준 개방형 일관성 제공
- 데이터 중복 없이 클러스터 간 공유 액세스

**보안 통합**
- IAM 기반 파일 시스템 및 객체 수준 권한 관리
- 전송 중 TLS 1.3 암호화, 저장 시 SSE-S3 또는 AWS KMS 암호화
- POSIX 권한(UID/GID) 지원
- CloudWatch 성능 모니터링 + CloudTrail 관리 이벤트 로깅

### 3. 어디에 쓰이는가

| 시나리오 | 설명 |
|---|---|
| **ML/AI 학습 파이프라인** | 훈련 데이터셋을 S3에 저장 → 여러 GPU 인스턴스에서 파일 시스템으로 직접 로드 |
| **에이전트형 AI 시스템** | 파일 기반 Python 라이브러리·셸 스크립트로 S3 데이터에 직접 접근 |
| **프로덕션 앱 공유 스토리지** | 여러 컴퓨팅 리소스가 동일 S3 버킷 데이터를 공동 읽기·쓰기 |
| **미디어·콘텐츠 파이프라인** | 영상·이미지 파일을 파일 시스템으로 업로드 → S3 기반 처리 파이프라인 연결 |
| **데이터 레이크 수집** | 앱이 파일로 데이터 생성 → S3 자동 적재 → Athena·EMR 분석 |

### 4. S3 Files vs 관련 서비스 비교

| 항목 | **S3 Files** | **Amazon EFS** | **Amazon FSx for Lustre** | **S3 File Gateway** |
|---|---|---|---|---|
| 프로토콜 | NFS v4.1 | NFS | Lustre | NFS·SMB |
| 저장소 | S3 버킷 | EFS 전용 스토리지 | S3 연계 | S3 버킷 |
| 주 용도 | S3 데이터 파일 시스템 접근 | 범용 공유 파일 시스템 | HPC·GPU 고성능 워크로드 | 온프레미스 → S3 게이트웨이 |
| 위치 | AWS 클라우드 전용 | AWS 클라우드 전용 | AWS 클라우드 전용 | 온프레미스·EC2 |
| S3 직접 연계 | ✅ 네이티브 | ❌ | ✅ (S3 백엔드 옵션) | ✅ |

***

### 5. 한 줄 정리

S3 Files는 **NFS v4.1 프로토콜로 S3 버킷을 AWS 컴퓨팅 리소스에 파일 시스템으로 마운트해, 코드 변경 없이 S3를 고성능 파일 스토리지로 활용할 수 있게 해주는 "오브젝트 스토리지와 파일 시스템 사이의 경계를 허무는 기능"이다.**
