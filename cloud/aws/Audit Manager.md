# AWS Audit Manager란?

## 개념 정의
AWS Audit Manager는 **AWS 사용 환경의 규정 준수(컴플라이언스) 상태를 지속적으로 평가하고, 감사 증적을 자동으로 수집·관리하는 완전 관리형 서비스**다. 내부 감사, 외부 감사, 규제 준수 검증에 필요한 증거를 자동으로 모으고 정리해, 감사 준비 시간과 부담을 대폭 줄여준다.

쉽게 말해 "AWS 환경에서 발생하는 모든 활동·설정·로그를 자동으로 수집해, 감사 기준(ISO, SOC, PCI-DSS, GDPR 등)에 맞춰 리포트를 만들어주는 감사 자동화 플랫폼"이다.

## 핵심 특징

**자동 증적 수집**
- CloudTrail, Config, Security Hub, VPC Flow Logs, CloudWatch Logs 등에서 자동으로 증거 수집
- AWS API 호출, 리소스 변경 이력, 보안 설정 상태 등을 실시간으로 캡처
- 수동으로 로그를 찾아 정리할 필요 없이 자동 집계

**사전 정의된 프레임워크(Framework)**
- AWS가 제공하는 표준 감사 프레임워크: CIS, PCI DSS, GDPR, HIPAA, SOC 2, ISO 27001, NIST 등
- 각 프레임워크에 맞는 통제 항목(Control)과 평가 기준이 사전 구성됨
- 커스텀 프레임워크 생성 가능: 조직 고유의 감사 기준을 직접 정의

**평가(Assessment) 관리**
- 특정 프레임워크를 선택해 평가(Assessment) 생성
- 평가 범위: 전체 계정, 특정 서비스, 특정 리전 등 선택 가능
- 지속적 평가: 실시간으로 증거를 수집하며 규정 준수 상태를 추적

**증거 저장 및 정리**
- 수집된 증거는 S3에 자동 저장(암호화, 접근 통제)
- 증거는 평가 리포트에 자동 연결되어 감사관에게 제출 가능
- 증거 보존 정책 설정 가능

**감사 리포트 생성**
- 평가 결과를 PDF/CSV 등으로 내보내기
- 통제 항목별 준수 여부, 증거 목록, 위험 항목 요약
- 감사관과 공유하거나 내부 리뷰에 활용

**협업 및 위임**
- 감사팀, 보안팀, 개발팀 등 여러 역할에 평가 접근 권한 부여
- 증거 검토, 코멘트 추가, 승인 프로세스 지원

**AWS Organizations 통합**
- 멀티 계정 환경에서 중앙 관리 계정이 모든 계정의 감사 증적 통합 수집
- 조직 전체의 규정 준수 상태를 한눈에 파악

## 주요 활용 사례

**외부 감사 대응**
- ISO, SOC, PCI 등 인증 심사 시 필요한 증거를 자동으로 준비
- 감사관 요청 시 즉시 리포트 및 증거 제출

**내부 감사 및 리스크 관리**
- 분기별/연간 내부 감사 자동화
- 규정 미준수 항목 조기 발견 및 개선

**규정 준수 지속 모니터링**
- 실시간으로 AWS 환경의 규정 준수 상태 추적
- 위반 발생 시 즉시 알림 및 대응

**보안 태세 관리**
- Security Hub, Config와 연동해 보안 설정 오류 자동 탐지
- CIS Benchmark, NIST 등 보안 표준 자동 평가

**고객/파트너 신뢰 확보**
- 규정 준수 증명자료를 고객·파트너에게 제공
- 계약 요구사항(SOC 2, ISO 등) 충족 증명

## 기본 워크플로

### 1. 프레임워크 선택
- AWS 제공 프레임워크(CIS, PCI DSS, GDPR 등) 또는 커스텀 프레임워크 선택
- 평가할 통제 항목(Control) 확인

### 2. 평가(Assessment) 생성
- 평가 이름, 설명, 범위(계정, 서비스, 리전) 설정
- 증거 수집 시작일, 종료일 지정
- 평가 소유자 및 검토자 지정

### 3. 자동 증거 수집
- CloudTrail, Config 등에서 실시간으로 증거 수집
- 각 통제 항목별로 관련 증거 자동 매핑
- S3에 증거 저장(암호화, 버전 관리)

### 4. 검토 및 승인
- 수집된 증거를 검토하고 코멘트 추가
- 필요시 수동 증거(문서, 스크린샷 등) 업로드
- 통제 항목별 준수 여부 확인

### 5. 리포트 생성 및 제출
- 평가 리포트를 PDF/CSV로 내보내기
- 감사관에게 제출하거나 내부 리뷰용으로 활용

## 지원하는 데이터 소스

**AWS 서비스**
- AWS CloudTrail: API 호출 이력
- AWS Config: 리소스 구성 변경 이력
- AWS Security Hub: 보안 파인딩
- Amazon CloudWatch: 로그 및 메트릭
- AWS License Manager: 라이선스 규정 준수
- VPC Flow Logs: 네트워크 트래픽 로그

**수동 증거**
- 정책 문서, 프로세스 문서, 스크린샷 등 직접 업로드

## 비용 구조

- 평가당 과금: 활성 평가 개수에 따라 월 단위 요금
- 증거 수집량에 따른 추가 요금
- S3 스토리지 비용(증거 저장)

## 보안 및 운영 모범 사례

**증거 보호**
- S3 버킷 암호화(KMS), 버전 관리 활성화
- 증거 접근 권한을 최소한으로 제한(IAM 정책)
- CloudTrail로 증거 접근 이력 감사

**멀티 계정 관리**
- AWS Organizations와 통합해 중앙 관리
- 위임 관리자(Delegated Administrator) 계정 설정
- 모든 계정의 증거를 중앙 S3 버킷에 집계

**프레임워크 커스터마이징**
- 조직 고유의 감사 기준을 커스텀 프레임워크로 정의
- 표준 프레임워크를 복사 후 수정해 사용

**자동화 통합**
- EventBridge로 평가 상태 변경 시 알림
- Lambda로 증거 수집 후 자동 처리 워크플로 구성
- Security Hub, Config와 연동해 실시간 준수 상태 모니터링

**정기 평가**
- 분기별/연간 평가를 정기적으로 실행
- 평가 결과를 트렌드 분석해 개선 포인트 도출

## 관련 서비스와의 차이

| 서비스 | 목적 | 차이점 |
|--------|------|--------|
| AWS Config | 리소스 구성 추적·규칙 평가 | 리소스 설정 상태 점검 중심 |
| AWS Security Hub | 보안 파인딩 집계·컴플라이언스 평가 | 보안 위협·취약점 탐지 중심 |
| AWS Artifact | 규정 준수 문서·계약서 다운로드 | AWS 자체 인증서 제공 |
| AWS Audit Manager | 감사 증적 자동 수집·리포트 생성 | 감사 프로세스 자동화 중심 |

Audit Manager는 Config, Security Hub 등의 데이터를 활용해 "감사 관점"에서 증적을 정리·리포팅하는 상위 레이어 서비스다.

## 요약

AWS Audit Manager는 클라우드 환경의 감사 준비 부담을 대폭 줄여주는 자동화 도구로, 증거 수집부터 리포트 생성까지 전 과정을 간소화하며, 지속적인 규정 준수 관리와 감사 대응의 핵심 솔루션이다.
