# AWS Nitro Enclaves란

## 1. 개념 요약

AWS Nitro Enclaves는 **Amazon EC2 인스턴스 안에 고립된(격리된) 별도 실행 환경을 만들어, 매우 민감한 데이터를 안전하게 처리할 수 있도록 해주는 기밀 컴퓨팅 기능**이다.  
쉽게 말해, **EC2 안에 또 하나의 “초보안 전용 작은 VM(샌드박스)”을 만들어 그 안에서만 개인정보·금융·의료 데이터·암호키 같은 민감 데이터를 처리하도록 하는 기술**이다.

***

## 2. 특징

- **강력한 격리(Isolation)**  
  - Nitro 하이퍼바이저가 EC2 인스턴스의 CPU·메모리 일부를 떼어 별도의 Enclave로 분리한다.  
  - Enclave는 네트워크, 디스크, SSH, 콘솔 접근이 모두 차단되어 있고, 상위 EC2나 운영자조차 Enclave 메모리 내용을 직접 볼 수 없다.

- **제한된 통신 방식 (vsock)**  
  - Enclave는 외부 네트워크를 직접 쓰지 않고, **로컬 가상 소켓(vsock)** 으로만 상위 EC2 인스턴스와 통신한다.  
  - 상위 EC2가 네트워크·KMS·외부 시스템과 통신하고, 민감한 연산만 Enclave에 요청하는 구조다.

- **원격 증명(Attestation)**  
  - Enclave가 어떤 코드/이미지(EIF)로 부팅되었는지 암호학적으로 증명해, 신뢰할 수 있는 코드에서만 암호키·데이터를 사용하도록 할 수 있다.  
  - 이 증명 결과를 기반으로 KMS 등의 키 사용을 허용/거부하는 정책을 구성할 수 있다.

- **Nitro 기반 기밀 컴퓨팅**  
  - Nitro 전용 하드웨어·펌웨어가 보안 경계를 강제해, AWS 운영자·하이퍼바이저·다른 테넌트로부터 워크로드를 보호하는 “기밀 컴퓨팅” 구현에 사용된다.

***

## 3. 어디에 쓰이는가

- **암호키·서명 연산 보호**  
  - 블록체인 지갑 키, 코드 서명 키, 인증서 발급용 키 등을 Enclave 내부에만 두고, 서명·복호화 같은 연산도 Enclave 안에서만 수행해 키 유출 위험을 최소화.

- **개인정보(PII)·의료·금융 데이터 처리**  
  - 신용카드 정보, 주민등록번호, 의료 기록, 거래 내역 등 높은 규제 수준의 데이터를 별도의 격리 환경에서 처리해 공격 표면을 크게 줄인다.

- **보안 민감 CI/CD 작업**  
  - 빌드 아티팩트 서명, 토큰 발급, 시크릿 생성·암호화 같은 작업을 Nitro Enclaves 기반 CI/CD 파이프라인에서 수행해, 빌드 서버가 침해돼도 핵심 비밀은 보호되도록 한다.

- **여러 기관 간 안전한 합산 연산**  
  - 여러 주체의 암호화된 데이터를 모아 Enclave 내부에서만 복호화·집계·연산하고, 결과만 내보내는 패턴(예: 기관 간 통계 합산, 공동 리스크 분석)에도 활용 가능하다.

***

## 4. 동작 구조 (간단 흐름)

1. **상위 EC2 인스턴스 준비**  
   - Nitro 기반 인스턴스에서 애플리케이션과 Nitro CLI를 설치한다.  

2. **Enclave 이미지(EIF) 생성**  
   - 민감 데이터 처리를 담당할 애플리케이션을 작은 리눅스 이미지 + 바이너리 형태로 패키징해 Enclave Image File(EIF)을 만든다.  

3. **Enclave 실행**  
   - Nitro CLI를 사용해 EC2 vCPU·메모리 일부를 Enclave에 할당하고 기동한다.  

4. **통신 & KMS 연동**  
   - 상위 EC2가 vsock으로 Enclave에 요청을 보내고, Enclave는 원격 증명을 통해 KMS 등의 키 사용 권한을 받아 민감 데이터 복호화·서명·연산을 수행한다.  

5. **결과 반환**  
   - Enclave는 처리 결과(서명값, 계산 결과 등)만 상위 EC2에 돌려주고, 원본 데이터·키는 Enclave 밖으로 나오지 않게 설계한다.

***

## 5. 장점 정리

| 항목 | 내용 |
|---|---|
| 보안 격리 | 네트워크·디스크·사용자 접근 차단, 메모리·CPU 수준 격리 |
| 공격 표면 축소 | 애플리케이션 중 민감 처리 부분만 Enclave로 분리해 보호 |
| 규제·컴플라이언스 | 금융, 의료, 공공 등 고규제 산업의 기밀 컴퓨팅 요구 충족에 도움 |
| 신뢰 증명 | Enclave 코드·이미지 무결성을 원격 증명(Attestation) 가능 |

***

## 6. 한 줄 정리

AWS Nitro Enclaves는 **EC2 인스턴스 내부에 완전히 격리된 보안 실행 환경을 만들어, 암호키·PII·금융·의료 데이터 같은 민감 정보를 안전하게 처리할 수 있게 해주는 AWS의 기밀 컴퓨팅 기능**이다.
