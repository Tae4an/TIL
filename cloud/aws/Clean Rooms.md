# AWS Clean Rooms란

## 1. 개념 요약

AWS Clean Rooms는 **서로 다른 회사·파트너가 각자의 원본 데이터를 상대방에게 직접 공개하거나 복사하지 않고도, 공동 데이터셋을 분석해 인사이트를 얻을 수 있도록 하는 안전한 데이터 협업 서비스**다. 쉽게 말하면, **"A사와 B사가 각자의 민감 데이터를 보호한 채로, 같은 분석 공간(클린룸)에서 함께 쿼리를 실행해 결과만 공유하는 서비스"** 라고 보면 된다.

***

## 2. 핵심 동작 원리

각 참여자는 자신의 AWS 계정에 데이터를 그대로 보관하고, Clean Rooms 협업 공간에서 분석 규칙이 허용하는 쿼리만 실행할 수 있다. 쿼리 결과는 원본 레코드 단위가 아닌 **집계(Aggregation) 결과만 반환**되어, 개별 데이터 노출 없이 인사이트를 얻는 구조다.

협업에는 세 역할이 존재한다.

- **협업 생성자(Creator)**: 클린룸을 만들고 분석 규칙·참여자를 초대
- **데이터 기여자(Contributor)**: 자신의 데이터셋을 제공하되, 원본 접근권은 허용하지 않음
- **쿼리 분석자(Analyst)**: 승인된 규칙 내에서 SQL·PySpark 쿼리를 실행해 결과를 수령

***

## 3. 주요 기능

- **분석 규칙(Analysis Rules)**
  - 허용된 컬럼·집계 방식·출력 제약을 사전 정의한다.
  - 규칙에 맞지 않는 쿼리(개별 레코드 조회, 금지 컬럼 참조 등)는 자동 거부된다.

- **차등 프라이버시(Differential Privacy)**
  - 쿼리 결과에 통계적 노이즈를 추가해, 역추적으로도 개인 데이터를 식별할 수 없게 만든다.

- **암호화 컴퓨팅(Cryptographic Computing)**
  - 데이터를 암호화된 상태로 처리해 쿼리 실행 중에도 원본 노출을 차단한다.

- **레코드 매칭**
  - 서로 다른 식별자(이메일, 쿠키, 광고 ID 등)를 가진 두 회사의 고객 레코드를 안전하게 매칭·연결한다.

- **ML 모델 협업(Clean Rooms ML)**
  - 원시 데이터나 커스텀 모델을 공유하지 않고도 파트너 데이터로 ML 모델을 학습·추론할 수 있다.

- **멀티 클라우드·데이터 소스 지원**
  - Snowflake에 저장된 파트너 데이터셋도 Zero-ETL 방식으로 협업에 참여 가능하다.

***

## 4. 주요 사용 사례

- **광고 효과 측정**
  - 광고주(구매 데이터)와 미디어사(노출 데이터)가 원본 공유 없이 광고 캠페인의 전환율·ROI를 공동 분석

- **리테일 미디어 네트워크**
  - 유통사와 브랜드사가 구매 데이터를 결합해 타겟 광고 오디언스 세그먼트를 생성

- **금융 사기 탐지**
  - 여러 금융기관이 고객 데이터를 공유하지 않고 공통 사기 패턴을 공동 식별

- **헬스케어 R&D**
  - 병원·제약사가 환자 데이터 보호 규정 준수하며 임상 데이터를 공동 분석

- **서드파티 쿠키 대체**
  - 쿠키 없는 환경에서 광고 타겟팅·성과 측정을 위한 1st-party 데이터 협업

***

## 5. 유사 서비스 비교

| 항목 | AWS Clean Rooms | AWS DataZone | AWS Glue |
|---|---|---|---|
| 핵심 목적 | 외부 파트너와 데이터 협업 (원본 비공개) | 내부 데이터 자산 거버넌스·공유 | 데이터 ETL·카탈로그 |
| 데이터 소유 | 각 참여사가 자기 계정에 보유 | 단일 조직 내 관리 | 단일 조직 내 처리 |
| 원본 노출 여부 | 차단 (집계 결과만 반환) | 승인 후 접근 가능 | 직접 접근 |
| 주요 대상 | B2B 데이터 협업, 광고·금융·헬스케어 | 데이터 분석가·비즈니스 팀 | 데이터 엔지니어 |

***

## 6. 한 줄 정리

AWS Clean Rooms는 **회사 간 원본 데이터를 공개·복사하지 않고도 공동 분석이 가능한 안전한 데이터 협업 서비스로, 분석 규칙·차등 프라이버시·암호화 컴퓨팅으로 데이터 주권을 보호하면서 광고·금융·헬스케어 등 다양한 산업의 파트너십 인사이트를 실현한다.**
