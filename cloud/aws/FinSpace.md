# AWS FinSpace란

## 1. 개념 요약

Amazon FinSpace는 **금융 서비스 산업(FSI)을 위해 특별히 설계된 데이터 관리 및 분석 완전관리형 서비스**다.  
금융권 분석가·데이터 과학자가 **페타바이트 규모의 재무 데이터를 찾고 준비하는 데 걸리는 시간을 몇 달에서 몇 분으로 단축**시키는 것이 핵심 목적이다.  
현재는 **Managed kdb Insights** 기반의 자본 시장 실시간·고속 데이터 처리에 특화된 형태로 운영되고 있다.

***

## 2. 특징

- **금융 도메인 특화 데이터 관리**
  - 금융 서비스 산업 전용 분류 체계로 데이터를 저장·구성한다.
  - "S&P500", "CAC40", "사모펀드" 같은 금융 비즈니스 용어로 데이터를 직접 검색할 수 있다.

- **Managed kdb Insights 지원**
  - 자본 시장에서 널리 쓰이는 **kdb+ (시계열 데이터베이스) 기반 고속 분석 환경**을 AWS에서 완전관리형으로 제공한다.
  - 티커 플랜트(실시간 시세), 실시간 클러스터, 과거 데이터 클러스터, 게이트웨이를 구성하고 Q 코드 그대로 마이그레이션 가능하다.

- **인프라 프로비저닝 자동화**
  - kdb Insights 인프라 구성·통합·보안 작업을 자동화해 수분 만에 신규 환경을 구성할 수 있다.
  - FinSpace 콘솔 또는 AWS API를 통해 프로그래밍 방식으로 환경 관리가 가능하다.

- **데이터 접근·보안·감사 통합**
  - 규제 요건에 맞는 데이터 접근 제어, 감사 로그, 권한 관리를 내장한다.
  - IAM 정책과 통합해 세밀한 액세스 제어가 가능하다.

***

## 3. 어디에 쓰이는가

- **자본 시장 실시간 분석**
  - 주식·채권·파생상품 시세를 수십억 건 실시간 처리하고, 과거 데이터와 함께 분석하는 HFT(고빈도 매매)·퀀트 분석 환경 구축.

- **위험 관리 (Risk Management)**
  - 포트폴리오·리스크 계산 워크로드를 kdb+ 기반으로 고속 처리.

- **규제·컴플라이언스 데이터 관리**
  - 금융 규제 보고용 데이터를 분류·보관·감사할 수 있도록 관리.

- **내부+외부 데이터 통합 분석**
  - 사내 포트폴리오·보험계리·리스크 시스템 데이터와 증권거래소 제3자 데이터 피드를 통합해 분석.

***

## 4. FinSpace vs 일반 데이터 분석 서비스 비교

| 항목 | Amazon Redshift / Athena | Amazon FinSpace |
|---|---|---|
| 대상 산업 | 범용 | 금융 서비스(FSI) 특화 |
| 데이터 유형 | 정형·반정형 범용 | 금융 시계열·시세 데이터 중심 |
| 주요 엔진 | SQL 기반 | kdb+ (시계열 고속 처리) |
| 도메인 분류 체계 | 없음 | 금융 비즈니스 용어 기반 카탈로그 |
| 규제·감사 | 직접 구현 필요 | 금융 규제 요건 내장 |

***

## 5. 참고 (서비스 현황)

- Amazon FinSpace의 **데이터 관리 기능(Data Management)은 2024년 7월 서비스 종료**됐다.
- 현재는 **Managed kdb Insights** 기능 중심으로 서비스가 유지되고 있으며, 자본 시장 kdb+ 워크로드 전용으로 포지셔닝되어 있다.
- 향후 추가 서비스 변경 여부는 AWS 공식 공지 확인이 필요하다.

***

## 6. 한 줄 정리

AWS FinSpace는 **금융 서비스 산업 전용 데이터 관리·분석 서비스로, 현재는 자본 시장 고속 시계열 분석을 위한 Managed kdb Insights에 특화된 "AWS의 금융 도메인 전용 분석 플랫폼"** 이다.
