# AWS Network Firewall이란?

## 개념 정의

AWS Network Firewall은 **VPC에 적용하는 AWS의 완전관리형 네트워크 방화벽 서비스**다.  
VPC를 드나드는 트래픽을 검사해서 IP, 포트, 프로토콜, 도메인, 패턴 기준으로 차단하거나 허용할 수 있고, 침입 탐지·방지 기능도 함께 제공한다.

***

## 핵심 특징

- **관리형 서비스**라서 방화벽 장비를 직접 설치·운영할 필요가 없다.  
- **상태 저장 검사(stateful inspection)** 와 **무상태 검사(stateless inspection)** 를 함께 지원한다.  
- **Suricata 호환 규칙**을 사용해 세밀한 위협 탐지와 차단이 가능하다.  
- **AWS 관리형 규칙 그룹**을 이용하면 알려진 악성 IP, 도메인, C2 통신 같은 위협을 빠르게 막을 수 있다.  
- 트래픽이 늘어나면 **자동 확장**되도록 설계되어 있다.  

***

## 무엇을 막는가

- 알려진 악성 IP와 도메인 접속  
- 비인가된 아웃바운드 통신  
- 특정 포트/프로토콜 기반의 위험 트래픽  
- 시그니처 기반 공격 패턴  
- VPC 안팎으로 오가는 트래픽 중 정책 위반 흐름  

***

## 어디에 두는가

보통 **VPC 경계**에 두고, 서브넷 라우팅을 통해 트래픽이 반드시 AWS Network Firewall을 지나가게 만든다.  
그래서 인터넷 게이트웨이, NAT 게이트웨이, VPN, Direct Connect로 오가는 트래픽을 중앙에서 통제하는 데 많이 쓴다.

***

## 왜 쓰는가

- VPC 트래픽을 중앙에서 일관되게 통제하려고  
- 온프레미스 방화벽과 비슷한 네트워크 보안을 AWS 안에서 구현하려고  
- SaaS, 인터넷, 외부 API로 나가는 아웃바운드 통신을 세밀하게 관리하려고  
- 규제 대응과 감사 로그 확보를 쉽게 하려고  

***

## 한 줄 요약

AWS Network Firewall은 **AWS VPC용 관리형 네트워크 방화벽으로, VPC를 드나드는 트래픽을 상태 기반으로 검사하고 차단하는 클라우드 방화벽 서비스**다.
