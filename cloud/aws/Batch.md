# AWS Batch란?

## 개념 정의

AWS Batch는 대규모 배치 작업(일괄 처리 작업)을 효율적으로 실행하도록 돕는 완전관리형 배치 컴퓨팅 서비스다. 사용자가 Docker 컨테이너로 작업을 정의하면, AWS Batch가 작업 우선순위·의존성·리소스 요구사항에 따라 자동으로 EC2/Fargate/Spot 인스턴스를 프로비저닝·스케줄링·확장·종료하여, 수천~수만 개 작업을 동시에 실행하고 관리해 준다.

쉽게 말해, "데이터 분석·ML 학습·렌더링처럼 오래 걸리는 배치 작업을 수백~수천 개 던지면, 알아서 서버 띄우고 병렬 실행하고 끝나면 정리해 주는 자동 배치 작업 관리 서비스"다.

## 핵심 기능

- **완전관리형 스케줄링**  
  작업 큐(Job Queue)에 제출된 작업을 우선순위·의존성·리소스 요구사항에 따라 자동 스케줄링하고, 컴퓨팅 리소스를 동적으로 프로비저닝한다.

- **자동 리소스 프로비저닝·확장**  
  작업량에 맞춰 EC2, Fargate, Spot 인스턴스를 자동으로 생성·확장·축소·종료하며, 사용자는 인프라 관리 불필요하다.

- **작업 의존성 관리**  
  작업 A가 끝난 후 작업 B를 실행하는 식으로, 복잡한 워크플로를 DAG(Directed Acyclic Graph) 형태로 정의하고 자동 실행할 수 있다.

- **우선순위 기반 큐**  
  여러 Job Queue를 생성하고 우선순위를 설정해, 중요한 작업을 먼저 실행하도록 제어할 수 있다.

- **Spot 인스턴스 통합**  
  EC2 Spot을 사용해 최대 90% 비용 절감이 가능하며, Spot 중단 시 자동으로 작업을 재시도한다.

- **다양한 컴퓨팅 환경**  
  EC2(온디맨드/Spot), Fargate, EKS 등 다양한 컴퓨팅 환경을 지원하며, CPU/GPU/메모리 요구사항에 맞게 선택할 수 있다.

## 주요 구성 요소

### Job Definition (작업 정의)

- Docker 컨테이너 이미지, vCPU, 메모리, IAM 역할, 환경 변수 등을 정의한 작업 템플릿
- 작업 실행에 필요한 모든 설정을 포함

### Job Queue (작업 큐)

- 제출된 작업이 대기하는 큐
- 우선순위와 연결된 Compute Environment를 지정
- 여러 큐를 만들어 우선순위 기반 스케줄링 가능

### Compute Environment (컴퓨팅 환경)

- 작업을 실행할 컴퓨팅 리소스(EC2, Fargate, Spot)를 정의
- Managed(AWS가 자동 관리) 또는 Unmanaged(사용자가 직접 관리) 타입 선택
- 인스턴스 타입, 최대 vCPU 수, VPC/보안 그룹 등을 설정

### Job (작업)

- 실제 실행할 배치 작업 단위
- Job Definition을 기반으로 Job Queue에 제출
- 단일 작업 또는 배열 작업(Array Job, 수백~수천 개 병렬 실행) 가능

## 작동 방식

1. **Job Definition 생성**: Docker 이미지, vCPU, 메모리 등 작업 스펙 정의
2. **Compute Environment 생성**: EC2/Fargate/Spot 환경 설정
3. **Job Queue 생성**: Compute Environment와 연결하고 우선순위 설정
4. **Job 제출**: Job Queue에 작업 제출
5. **스케줄링**: AWS Batch가 작업 우선순위·리소스 요구사항·의존성에 따라 스케줄링
6. **리소스 프로비저닝**: 필요한 만큼 EC2/Fargate 인스턴스 자동 생성
7. **작업 실행**: 컨테이너에서 작업 실행
8. **종료**: 작업 완료 후 리소스 자동 종료

## 주요 활용 사례

- **데이터 분석·ETL**  
  매일 수백 GB의 로그 데이터를 S3에서 가져와 변환·집계하고 Redshift/Athena로 로드하는 배치 작업을 자동 실행한다.

- **ML 모델 학습·추론**  
  수천 개의 하이퍼파라미터 조합으로 ML 모델을 병렬 학습하거나, 대량 이미지·영상에 대한 추론 작업을 배치로 실행한다.

- **금융 서비스 분석**  
  하루 거래 내역을 분석하고, 리스크 계산, 포트폴리오 시뮬레이션 등을 매일 밤 배치 작업으로 수행한다.

- **미디어 렌더링·인코딩**  
  영화·광고 영상의 렌더링, 대량 이미지 리사이징, 영상 트랜스코딩 작업을 병렬로 처리한다.

- **과학 시뮬레이션**  
  약물 설계, 유전체 분석, 기후 모델링, 자율주행 시뮬레이션 등 계산 집약적 작업을 대규모로 실행한다.

- **테스트·CI/CD**  
  대량의 단위 테스트, 통합 테스트, 보안 스캔 작업을 병렬로 실행해 개발 속도를 높인다.

## 비용

- 컴퓨팅 리소스(EC2, Fargate) 사용량에 대해서만 과금
- AWS Batch 서비스 자체는 무료
- Spot 인스턴스 사용 시 최대 90% 비용 절감 가능
