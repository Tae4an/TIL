# AWS Resource Explorer란?

## 개념 정의

AWS Resource Explorer는 여러 AWS 리전과 수십 개 서비스에 분산된 리소스를 **검색 엔진처럼 빠르게 찾을 수 있게** 해주는 리소스 검색·탐색 서비스다. 리소스 이름, 태그, ID, 타입 등으로 검색하면, 전체 리전에 걸친 EC2, S3, DynamoDB, Lambda 등 수백 가지 AWS 리소스를 한 번에 조회할 수 있고, Organizations 통합으로 **멀티 계정 검색**도 지원한다.

쉽게 말해, "Google처럼 AWS 리소스를 검색하는 도구로, 어느 리전 어느 계정에 있는지 모를 때 키워드만 치면 찾아주는 통합 검색 서비스"다.

## 핵심 기능

- **크로스 리전 검색**  
  한 곳에서 검색하면 모든 리전의 리소스를 한 번에 조회할 수 있다.

- **멀티 계정 검색**  
  AWS Organizations와 통합하여, 조직 전체 또는 특정 OU(조직 단위)의 모든 계정 리소스를 중앙에서 검색할 수 있다.

- **자유 텍스트 및 필터 검색**  
  키워드(이름, 태그, ID)를 입력하거나, 리소스 타입·리전·태그·계정 등으로 필터링하여 원하는 리소스만 추출할 수 있다.

- **인덱스 기반 빠른 검색**  
  Local Index(리전별) + Aggregator Index(전체 리전 통합)를 자동 생성하여, 대량 리소스도 빠르게 검색할 수 있다.

- **IAM 기반 자동 활성화**  
  IAM 권한만 있으면 자동으로 사용 가능하며, 일부 리소스는 인덱스 설정 없이도 즉시 검색된다.

- **Unified Search 통합**  
  AWS 콘솔 상단 검색창(Unified Search)에서도 Resource Explorer를 통해 리소스를 찾을 수 있다.

## 주요 구성 요소

### Index (인덱스)

**Local Index (로컬 인덱스)**  
- 특정 리전 내 리소스만 포함
- 리전별로 하나씩 생성

**Aggregator Index (집계 인덱스)**  
- 모든 리전의 Local Index를 복제하여 통합
- 계정당 하나만 생성 가능
- 크로스 리전 검색을 가능하게 함

### View (뷰)

**Default View (기본 뷰)**  
- 필터 없이 모든 리소스 반환

**Custom View (사용자 정의 뷰)**  
- 특정 리소스 타입, 태그, 리전 등으로 필터링한 뷰

**Multi-Account View (멀티 계정 뷰)**  
- Organizations 전체 또는 특정 OU의 계정 리소스를 검색하는 뷰
- Delegated Admin 계정에서 생성

## 주요 활용 사례

- **리소스 위치 파악**  
  "prod-server-01"이라는 EC2 인스턴스가 어느 리전 어느 계정에 있는지 모를 때, 한 번에 검색하여 위치를 빠르게 파악한다.

- **태그 누락 검출**  
  "Environment" 태그가 없는 리소스를 검색하여, 태그 정책 준수 여부를 확인하고 누락된 태그를 추가한다.

- **멀티 리전·멀티 계정 관리**  
  수십 개 계정과 리전에 분산된 Lambda, S3, DynamoDB 등을 한 곳에서 검색하여, 전체 인프라 현황을 파악하고 리소스 정리 작업을 수행한다.

- **리소스 정리·비용 절감**  
  사용하지 않는 EBS 볼륨, 오래된 스냅샷, 미사용 Elastic IP 등을 검색하여 삭제하고 비용을 절감한다.

- **인시던트 대응**  
  장애 발생 시 관련 리소스(EC2, RDS, ELB 등)를 빠르게 검색하여, 문제 리소스를 즉시 식별하고 조치한다.

- **감사·컴플라이언스**  
  특정 태그("Compliance=PCI")가 있는 리소스만 검색하여, 규정 준수 대상 리소스를 일괄 확인한다.

## 비용

AWS Resource Explorer는 **무료**다. 인덱스 생성, 검색, 뷰 생성 모두 무료로 제공된다.
