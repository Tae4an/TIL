# AWS CloudSearch란

## 1. 개념 요약

Amazon CloudSearch는 **웹사이트·애플리케이션에 검색 기능을 쉽게 구축·운영할 수 있도록 AWS가 인프라 전체를 관리해주는 완전관리형 검색 서비스**다.  
검색 서버 설치·튜닝·스케일링·장애 복구를 AWS가 자동으로 처리하므로, **개발자가 검색 인프라 운영 없이 클릭 몇 번으로 고성능 검색 엔진을 사용할 수 있는 서비스**라고 보면 된다.  
내부적으로는 **Apache Solr (Lucene 기반)** 검색 엔진을 사용한다.

***

## 2. 특징

- **완전관리형 (Fully Managed)**
  - 서버 프로비저닝, 소프트웨어 패치, 노드 모니터링·복구, 데이터 파티셔닝을 AWS가 자동으로 처리한다.
  - 개발자는 검색 도메인 구성과 데이터 업로드·쿼리에만 집중하면 된다.

- **자동 확장**
  - 트래픽 증가에 따라 검색 인스턴스를 자동으로 수평·수직 확장하고, 트래픽이 줄면 다시 축소한다.

- **다양한 검색 기능 내장**
  - 자유 텍스트 검색, 패싯 검색, 지형 정보 검색(거리 기반 필터링), 강조 표시(Highlight), 자동 완성(Suggestions), 커스터마이즈 가능한 관련성 랭킹 등을 지원한다.

- **34개 언어 지원**
  - 한국어 포함 34개 언어에 대한 언어별 텍스트 처리(형태소 분석, 불용어 처리 등)를 기본 지원한다.

- **세 가지 API로 구성**
  - 구성 서비스: 검색 도메인(스키마·인덱스) 생성·설정
  - 문서 서비스: 데이터 업로드·색인(JSON/XML 배치)
  - 검색 서비스: 검색 쿼리 실행·결과 반환

- **IAM 통합 보안**
  - IAM 정책으로 도메인별 접근 제어가 가능하다.

***

## 3. 어디에 쓰이는가

- **웹사이트·쇼핑몰 검색**
  - 상품·게시글·문서 데이터를 색인하고, 사용자가 입력한 키워드로 빠르게 검색 결과를 반환.

- **애플리케이션 내 검색 기능**
  - 사내 문서 검색, 고객 지원 FAQ 검색, 뉴스·미디어 콘텐츠 검색 등 범용적인 검색 기능 구현.

- **지형 정보 검색**
  - 현재 위치 기준 가까운 매장·식당·서비스를 거리 순으로 검색하는 위치 기반 서비스.

- **스타트업·소규모 팀의 빠른 검색 도입**
  - 검색 전문가나 별도 인프라 없이 빠르게 검색 기능을 프로덕션에 배포할 때 유용.

***

## 4. CloudSearch vs OpenSearch Service 비교

| 항목 | Amazon CloudSearch | Amazon OpenSearch Service |
|---|---|---|
| 엔진 | Apache Solr (Lucene 기반) | OpenSearch/Elasticsearch |
| 관리 편의성 | 매우 쉬움, 설정 최소화 | 더 많은 설정·튜닝 가능 |
| 기능 확장성 | 제한적 | 풍부한 플러그인·기능 |
| 로그 분석 | 부적합 | ELK 스택 대체로 강점 |
| 벡터 검색 | 미지원 | 최근 버전에서 지원 |
| 버전 업데이트 | 느림 | 활발한 업데이트 |
| 주요 적합 상황 | 단순 검색 빠른 도입 | 복잡한 검색·분석·로그 파이프라인 |

***

## 5. 참고 (서비스 현황)

CloudSearch는 오래된 서비스로 **버전 업데이트가 거의 이루어지지 않고 있으며**, AWS는 신규 검색 워크로드에 대해 **Amazon OpenSearch Service를 권장**하는 추세다. 새로 검색 기능을 구축하는 경우 OpenSearch Service나 Amazon Kendra(AI 검색) 도입을 먼저 검토하는 것이 일반적이다.

***

## 6. 한 줄 정리

Amazon CloudSearch는 **Apache Solr 기반으로 서버 관리 없이 웹사이트·앱에 검색 기능을 빠르게 구축할 수 있는 AWS의 완전관리형 검색 서비스**로, 단순한 검색 요구사항에 적합하지만 신규 구축 시에는 OpenSearch Service 검토가 권장된다.
