# AWS Reachability Analyzer란?

## 1. 개념 요약

AWS Reachability Analyzer는 **VPC 내 두 리소스 간 네트워크 연결 가능 여부를 실제 트래픽 전송 없이 정적 구성만으로 분석하는 진단 도구**다.  
쉽게 말해, **"패킷을 실제로 보내지 않고도 네트워크 경로가 뚫려 있는지 확인해주는 AWS의 네트워크 X-Ray"** 라고 보면 된다.

***

## 2. 특징

- **정적 구성 분석**
  - 실제 트래픽 없이 라우팅 테이블, 보안 그룹, NACL 등 구성 정보만 분석
  - 패킷 캡처나 테스트 트래픽 없이 즉시 결과 확인

- **홉별 경로 상세 제공**
  - 연결 가능하면 소스 → 대상까지의 홉별 경로 시각화
  - 연결 불가하면 **어떤 구성 요소가 차단하는지** 정확히 식별

- **다양한 리소스 지원**
  - EC2, ENI, IGW, Transit Gateway, VPC Peering, VPC Endpoint 등 대부분의 네트워크 리소스 간 분석 가능

- **멀티 계정 지원**
  - AWS Organizations와 연동해 **다른 계정 간 리소스 연결성도 분석** 가능

***

## 3. 어디에 쓰이는가

- **네트워크 트러블슈팅**
  - EC2 → RDS 연결이 안 될 때 보안 그룹, NACL, 라우팅 중 어디가 막혔는지 즉시 확인

- **보안 설정 검증**
  - 의도치 않은 경로가 열려 있는지, 의도한 차단이 제대로 되어 있는지 검증

- **인프라 변경 전 사전 검증**
  - 라우팅·보안 그룹 변경 전 영향도 미리 확인

- **자동화 파이프라인 통합**
  - IaC 배포 후 네트워크 연결성 자동 검증에 활용

***

## 4. 유사 서비스와의 차이

| 항목 | Reachability Analyzer | Network Access Analyzer | VPC Flow Logs |
|---|---|---|---|
| **분석 방식** | 정적 구성 분석 | 의도치 않은 접근 탐지 | 실제 트래픽 로그 |
| **주요 목적** | 특정 경로 연결 여부 | 보안 위반 경로 감사 | 트래픽 흐름 모니터링 |
| **트래픽 필요** | 불필요 | 불필요 | 필요 |
| **결과** | 경로 가능/불가 + 차단 지점 | 비의도적 접근 경로 목록 | 실제 흐름 통계 |

***

## 5. 한 줄 정리

AWS Reachability Analyzer는 **실제 트래픽 없이 VPC 네트워크 구성을 분석해 연결 가능 여부와 차단 지점을 홉별로 알려주는** "AWS 네트워크 연결 진단 도구"다.
