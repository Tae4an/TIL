# AWS Organizations란?

## 개념 정의
AWS Organizations는 AWS에서 제공하는 **멀티 계정(계층형 계정) 통합 관리 플랫폼**이다. 여러 AWS 계정(부서, 프로젝트, 환경별 등)을 하나의 조직 트리 구조로 묶어, 중앙에서 효율적으로 관리·정책 적용·비용 통제·보안·거버넌스를 일괄 처리할 수 있도록 설계된 서비스다.

쉽게 말하면 “여러 AWS 계정을 한 번에 통제·정렬·정책 적용하는 중앙 관리 도구”로, 대기업·대규모 프로젝트·MSP 등에서 필수적인 멀티 계정 운영의 표준이다.

## 핵심 특징

**계층형 조직 구조**
- 루트(Root) 계정을 중심으로 Organizational Unit(OU) 계층 생성
- OU마다 서브OU와 계정 반복 배치 가능 (트리 구조)
- 부서, 프로젝트, 용도(Dev/Prod), 계열사별 등 유연하게 계정 분리·구성

**중앙집중 정책 관리**
- 서비스 제어 정책(SCP)으로 조직·OU·계정별 AWS 사용 범위 제한(상위 제한 정책)
- 태그 정책(Tag Policy), 백업 정책, AI Opt-Out 정책 등 다양한 거버넌스 자동 적용

**비용·결제 통합 관리**
- 모든 계정의 요금을 마스터 계정에서 통합 청구(Consolidated Billing)
- AWS Cost Explorer, 예산, 비용 할당 태그 등으로 전체 Cost·할당 분석 가능

**일괄 자동화 및 보안 통제**
- 신규 계정 생성·초대·관리 자동화(템플릿/스크립트)
- CloudTrail, Config 등 조직 단위 일괄 로깅/감사 구성
- IAM과 연계해 중앙 권한·보안 정책 강제 적용

**확장성 및 MSP 지원**
- 수십~수천 개 계정 규모 확장, 조직경계·MSP·파트너 관리 유용
- AWS Control Tower, SSO 등 멀티 계정 기반 서비스와 긴밀 연동

## 주요 구성 요소

- **Root Account**: 조직 전체를 대표하는 최고 책임 계정(결제/SCP/OU/계정 관리)
- **Organizational Unit(OU)**: 계정 분류/트리 구조, OU별 정책/권한/비용 통제 가능
- **Member Account**: 실제 리소스 운영 및 정책 적용 대상 계정
- **SCP**: OU/계정에 적용되는 최대 권한 상한(명시된 것 외 부분은 모두 차단)
- **통합 결제(Consolidated Billing)**: 모든 계정 비용을 Master 계정에서 통합 관리

## 기존 개별 계정 운영 vs AWS Organizations 비교

|       | 개별 계정 운영                 | AWS Organizations           |
|-------|-------------------------------|-----------------------------|
| 계정관리 | 수동, 계정별 별도 관리           | 중앙집중, 자동/일괄 관리        |
| 정책     | 각 계정별 별도 설정            | 조직 단위 정책 일괄/강제 적용   |
| 비용     | 계정별 별도 청구               | 통합 청구, 비용 분석 최적화    |
| 확장성   | 계정 수 증가시 복잡도↑         | 수백~수천 계정도 간편히 관리   |
| 거버넌스 | 비일관성 발생 가능             | 정책·로깅·보안 일관적 적용     |

## 주요 기능

- OU 트리 구조 설계 및 계정(OU) 추가/초대/관리 자동화
- 서비스 제어 정책(SCP)으로 계정별·조직별 권한 설계(최대 허용 범위 지정)
- 통합 결제 및 비용 분석 태그/정책 적용
- 태그 정책, 백업 정책 등 조직적 자원 통제 및 자동 준수
- 자동화된 신규 계정 생성/초대/구성 템플릿 활용(AWS Control Tower와 연동)
- 조직 전체 공통 CloudTrail/Config/보안체계 자동화 구성

## 주요 활용 사례

- 기업/대학/공공기관/프로젝트별로 AWS 계정 분리 관리
- 보안/개발/운영/Ops 등 역할별 계정 생성으로 도메인별 통제
- 대규모 멀티 계정 아키텍처에서 보안·거버넌스·비용 통합 관리
- MSP·파트너(외부) 운영, 하위 조직별 자체 계정 운용 허용 구조 구성

## 기본 워크플로

### 1. 조직 생성 및 OU 구성
- Root에서 AWS Organizations 시작 > Organizational Unit 생성
- 신규 Member 계정 생성 또는 기존 계정 초대/이동

### 2. 서비스 제어 정책(SCP) 작성/적용
- OU별로 최대 허용 권한(예: 리소스 생성/삭제 제한) 정책 대입

### 3. 통합 청구·비용 관리 활성화
- Billing에서 콘솔 Cost Explorer, 예산, 할당 태그 관리

### 4. 조직 전체 보안 및 로깅 구성
- CloudTrail/Config/GuardDuty 등 일괄 활성화, 리소스 정책 OU별 통제

### 5. 신규 계정 일괄 생성 자동화
- Control Tower, 자동화 스크립트(IaC/CLI)로 신규 계정 일괄 구축·정책 즉시 적용

## 비용 구조

- AWS Organizations 기본 기능 무료(정책, 계정 생성, OU 관리 등)
- 리소스/서비스 자체는 각 계정별로 과금, 통합 결제/분석 무료

## 보안 모범 사례

- Root 계정 MFA 필수, 최소 권한 원칙 적용
- OU·계정·조직 정책(SCP)으로 세밀한 제어·불필요 권한 차단
- 공통 로깅/감사/보안 체계 일괄 활성화
- 비용 정책, 태깅 전략 조직 전체 통일 적용
- 신규 계정 생성/초대 자동화로 인적 오류·편차 방지

AWS Organizations는 대규모/복합 조직의 클라우드 환경 거버넌스와 운영/보안/비용 효율화를 위한 필수 서비스다.
