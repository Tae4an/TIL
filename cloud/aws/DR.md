# AWS DR(Disaster Recovery, 재해복구)이란?

## 개념 정의
AWS DR은 **서비스나 데이터에 심각한 장애·재해(자연재해, 인프라 오류, 보안 사고 등)가 발생해도 빠르게 정상 복구하여 비즈니스 연속성을 확보하는 아키텍처 및 운영 전략**이다. 클라우드 및 온프레미스 환경의 데이터를 안전하게 백업·복제해, 예기치 않은 문제 시 최소한의 다운타임·데이터 손실로 서비스를 복원할 수 있도록 설계한다.

쉽게 말하면 “AWS에서 장애·재해/사고 시 빠르고 신뢰성 있게 원상/대체 복구하는 전략과 기술의 집합”이다.

## 핵심 특징

**탄력성(Resilience)·가용성(Avalibility) 중심**
- 여러 리전, AZ(가용역역)에 데이터와 서비스를 분산·복제
- 장애/재해 발생 시 신속한 자동/수동 Failover 및 복구 지원

**자동화된 백업, 복제, 스냅샷**
- S3, DynamoDB, Aurora, RDS, EBS 등 AWS 자체 서비스는 자동/수동 백업, 스냅샷, 교차 리전 복제 기능
- 정기 백업/스케줄, 보존 정책으로 데이터 무결성 유지

**복구 시나리오 다양성**
- 백업/복제된 데이터를 빠르게 새로운 리소스에 복원(Recovery)
- 장애 시 예비 인프라, 리전, 계정, 온프레미스 리소스로 서비스 전환

**DR 패턴(모드)별 설계**
- 백업·복원(Backup & Restore): 최소비용, 복구 느림, 주로 저가용성 서비스
- Pilot Light: 핵심 서비스만 최소로 예비 배포, 장애 시 빠른 전체 리소스 배포
- Warm Standby: 소규모 실시간 동기화 인프라 운영, 장애 시 확장
- Multi-Site(Active-Active): 두 곳 이상 리전·AZ에 동기화 서비스, 장애 즉시 트래픽 우회

## 주요 구성 요소

- S3 / Glacier: 객체 데이터 및 아카이브 데이터 백업·복제·복원, 교차 리전 복제
- EBS / RDS / DynamoDB: 자동/수동 스냅샷, 교차 AZ/리전 복제·복원
- EC2 / ECS / Lambda 등: AMI, 컨테이너 이미지, 코드/설정의 복제·백업
- Route53: DNS Failover로 장애 발생 시 자동 트래픽 우회
- AWS Backup: 클라우드·온프레미스 데이터 일괄 백업·보존·복구 자동화

## 주요 DR 설계 방법

**Backup & Restore(백업&복원)**
- 서비스 데이터 정기 백업만 유지, 장애 시 신속 배포·복원

**Pilot Light(파일럿 라이트)**
- 중요한 서버·DB·구성 요소만 최소 예비로 배포, 평상시 최소비용  
- 장애 시 전체 서비스/인프라 신속 배포·확장

**Warm Standby(웜 스탠바이)**
- 실제 프로덕션의 축소판을 예비 인프라로 운영(트래픽 낮음)
- 장애 시 규모 확장하며 신속 서비스 전환

**Multi-Site Active-Active**
- 두 곳 이상 리전, AZ에 실시간 동기화·부하 분산
- 장애 발생 시 자동 트래픽 분산·장애 노드 제거

## 주요 활용 사례

- 주요 서비스 및 데이터 교차 리전/교차 AZ 자동 복제(무정지 복구 구현)
- 금융/공공/의료 등 SLA가 높은 중요 서비스의 다중 DR 시나리오 적용
- 온프레미스-클라우드 하이브리드 DR 설계(Backup, Multi-Site 등)
- AWS Backup/CloudEndure DR 등 도구 활용한 자동 복구/백업 정책 
- 주기적 DR 훈련 및 복구 시나리오 실제 테스트

## 기본 워크플로 및 설계 예시

### 1. 자동 백업·스냅샷·복제 정책 시행
- S3 버킷 교차 리전 복제 활성화
- RDS/Aurora/DynamoDB Multi-AZ/리전 복제 및 자동 스냅샷
- EC2/EBS/AMI·EFS 정기 백업 및 보존 설정

### 2. 장애 탐지 및 복구 자동화
- Route53 DNS Failover, 헬스체크 자동화
- 장애/재해 발생 시 Lambda/CloudFormation/Terraform 등으로 자동 복구/배포 트리거

### 3. DR 시나리오 운영 및 검증
- Pilot/Warm/Multi-Site DR 구성 및 복구 워크플로 짜기
- 실제 장애 복구 훈련(테스트) 및 복원 시간 확인

## 비용 구조

- 백업/복제/스냅샷 스토리지 및 복구 대상 리소스에 따른 요금
- 복제된 리전의 데이터 전송/저장, 예비 인프라 운영비까지 합산 고려

## 보안 및 운영 모범 사례

- 교차 리전·교차 AZ 복제 및 백업으로 단일 장애지점 완전 제거
- 백업 암호화, 접근 제어, 정책기반 장기 보존 설계, 보안감사 기록
- DR 복구 시나리오 주기적 테스트로 신뢰성 검증
- 정기적인 백업 정책 점검 및 자동화로 운영 효율성과 복구 신뢰도 극대화

AWS DR(Disaster Recovery)은 비즈니스 연속성과 운영 안정성을 유지하는 핵심 전략으로, 무정지·신속 복구 아키텍처와 자동화·효율화를 기반으로 실질적인 재해복구 능력을 클라우드에서 실현한다.
