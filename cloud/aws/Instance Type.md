# AWS Instance Type란?

## 1. 개념 요약

AWS EC2 Instance Type은 **EC2 인스턴스가 가지는 vCPU·메모리·네트워크·스토리지 성능 조합을 정의한 “서버 사양 템플릿”** 이다.  
`t3.micro`, `m6i.large`처럼 이름으로 패밀리(용도), 세대, 크기(스펙)를 표현하고, 워크로드에 맞게 골라 쓰는 구조다.

***

## 2. 이름 규칙 (구조 이해)

예: `m5.2xlarge`

- `m` : 인스턴스 패밀리(용도) – 여기서는 **범용(General Purpose)** 
- `5` : 세대(Generation) – 숫자가 클수록 최신 세대
- `2xlarge` : 크기(Size) – vCPU·메모리 양을 의미 (`large`, `xlarge`, `2xlarge` 등)

추가 접미사 예시

- `a` : AMD 기반 (`m6a`, `c6a` 등)  
- `g` : Graviton(ARM) 기반 (`m7g`, `c7g` 등)  
- `n` : 네트워크/ENI 특화 (`c5n` 등)

***

## 3. 인스턴스 패밀리별 용도·차이

### 3-1. 범용 (General Purpose) – `t`, `m`

- 대표 패밀리: `t3`, `t4g`, `m5`, `m6i`, `m7g` 등
- 특징  
  - vCPU·메모리·네트워크가 **균형 잡힌** 타입.  
  - 대부분의 웹/앱 서버, 소규모 DB, 개발·테스트, 마이크로서비스 등에 적합.  
- T 시리즈 (`t3`, `t4g` 등)
  - **버스트 가능(기본은 낮지만 필요 시 잠깐 고성능)** CPU 크레딧 기반.  
  - 항상 CPU를 많이 쓰지 않는 웹 서버, 작은 API 서버 등에 비용 효율적.  
- M 시리즈 (`m5`, `m6i`, `m7g` 등)
  - 더 안정적이고 균형 잡힌 범용형.  
  - 대부분의 엔터프라이즈 일반 워크로드에 기본 선택지.  

### 3-2. 컴퓨팅 최적화 (Compute Optimized) – `c`

- 대표 패밀리: `c5`, `c6i`, `c7g` 등
- 특징  
  - **vCPU 대비 메모리 비율이 낮고 CPU 비율이 높은** 구조.  
  - 고성능 웹 서버, 배치 처리, 미디어 트랜스코딩, 고성능 게임 서버, 광고 서버, 과학 계산, ML 추론 등 CPU 집약 워크로드에 적합.

### 3-3. 메모리 최적화 (Memory Optimized) – `r`, `x`, `u` 등

- 대표 패밀리: `r5`, `r6i`, `x1e`, `x2idn` 등
- 특징  
  - **메모리(GB) 대비 vCPU 수 비율이 아주 높은** 구조.  
  - 인메모리 DB(Redis, SAP HANA), 대규모 캐시, 실시간 빅데이터 분석, 고메모리 애플리케이션에 적합.

### 3-4. 스토리지 최적화 (Storage Optimized) – `i`, `d`, `h` 등

- 대표 패밀리: `i3`, `i4i`, `d3`, `h1` 등
- 특징  
  - **고 IOPS / 고 처리량 로컬 NVMe/SSD/HDD** 스토리지가 포함.  
  - NoSQL DB, Elasticsearch, 분산 파일시스템, 데이터 웨어하우스, 로그/분석 워크로드에 적합.

### 3-5. 가속 컴퓨팅 (Accelerated Computing) – `p`, `g`, `f`, `trn` 등

- 대표 패밀리 
  - `p*` : 일반 GPU – 딥러닝 학습, HPC
  - `g*` : 그래픽/GPU – VDI, 게임 스트리밍, 인퍼런스 등  
  - `f*` : FPGA – 특수 하드웨어 가속  
- 특징  
  - GPU·FPGA 같은 **하드웨어 가속기**가 포함되어 ML, 그래픽, 비디오 처리, 과학 시뮬레이션 등에 최적.

### 3-6. 기타 (HPC 최적화 등)

- HPC 전용: `hpc6a`, `hpc7g` 등 – 초고성능 네트워크/CPU 조합, MPI/HPC 워크로드용.

***

## 4. 패밀리별 스펙 차이 감 잡기

CPU:RAM 비율 예시 (세대별 상이하지만 대략적 감각)

- General Purpose(M 시리즈): 대략 **1 vCPU : 4GB RAM** 수준  
- Compute Optimized(C 시리즈): RAM 비율 더 적음(1 : 2 GB 근처 등)  
- Memory Optimized(R 시리즈): RAM 비율 더 큼(1 : 8GB 이상 등)  

즉,  
- CPU만 많이 필요하면 C,  
- 메모리가 훨씬 중요하면 R/X,  
- 둘 다 적당히면 M/T를 고르는 식으로 선택한다.  

***

## 5. 인스턴스 타입 선택 팁 (요약표)

| 요구사항 / 워크로드 | 추천 패밀리 | 이유 |
|---------------------|-------------|------|
| 일반 웹/앱, 중소 규모 DB, 개발·테스트 | `t`, `m` | 균형형, 비용 효율적
| CPU 집약(배치, 트랜스코딩, 게임 서버, 광고 서버, 과학 계산) | `c` | 높은 vCPU 비율
| 메모리 집약(인메모리 DB, 캐시, 실시간 분석) | `r`, `x` | 대용량 RAM
| 고 IOPS/고 처리량 스토리지(ES, NoSQL, 로그 분석) | `i`, `d` | 로컬 NVMe/SSD/HDD 튜닝
| GPU/가속기(딥러닝, 그래픽, 영상 처리) | `p`, `g`, `f` | 하드웨어 가속 포함

***

## 6. 한 줄 정리

AWS Instance Type은 **EC2의 “CPU·메모리·네트워크·스토리지 조합을 캡슐화한 이름”이고**,  
`t/m`=범용, `c`=CPU, `r/x`=메모리, `i/d`=스토리지, `p/g/f`=GPU·가속기처럼 **패밀리 이름만 봐도 어떤 워크로드에 맞는지 감이 오도록 설계된 타입 체계**다.
