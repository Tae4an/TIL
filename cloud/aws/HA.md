# AWS HA(High Availability)란?

## 개념 정의
AWS의 HA(High Availability, 고가용성)는 **서비스 또는 애플리케이션이 장애·장애 상황에서도 지속적으로 동작하도록 설계하는 AWS 인프라 아키텍처의 핵심 원칙**이다. 시스템 중단이나 데이터 손실 없이 언제나 사용자가 서비스에 안정적으로 접근할 수 있도록 하는 설계를 의미한다.

쉽게 말해 “서버, 스토리지, 네트워크, 데이터베이스 등 모든 클라우드 구성요소가 장애 발생에도 자동 복구/대체되어 서비스 가용성(업타임)을 극대화하는 AWS 설계 철학”이다.

## 핵심 특징

**다중 가용영역(AZ)·리전 아키텍처**
- AWS 리전(region)·가용영역(AZ)별로 인프라를 분산시켜 단일 장애지점(SPOF) 제거
- AZ 간 자동 failover, 데이터 복제, 트래픽 분산, DNS 자동 전환

**자동 확장 및 복구**
- 오토스케일링(Auto Scaling) 그룹으로 인스턴스 장애/트래픽 급증 시 자동 인스턴스 증가·복구
- ELB(Elastic Load Balancing)로 여러 서버에 트래픽 균등 분배, 헬스체크 후 이상 노드 자동 우회

**데이터 내구성·복제**
- S3, RDS, DynamoDB 등은 기본적으로 다중 AZ/Region 복제 및 자동 백업, 데이터 손실 최소화
- RDS Multi-AZ, Aurora 글로벌 클러스터, S3 크로스 리전 복제 등 내구성 강화 옵션

**무중단 장애 전환(Failover) 및 유지보수**
- 장애 발생 시 자동으로 예비 인스턴스·리전·DB에 트래픽 전환
- 점검 시 무중단 롤링 업데이트, 배포 등 지원

**상태 모니터링 및 알림**
- CloudWatch, Health Dashboard 등으로 상태 지속 모니터링 및 이상·알림 자동 통합  
- 장애 징후 감지 시, 자동 복구/Noti/Failover 트리거

## 주요 설계 방식

**1. 멀티 AZ 배포**
- EC2, RDS 등 인스턴스나 DB를 여러 AZ에 분산  
- 장애 발생 시 즉시 예비 인스턴스(AZ)로 자동 전환

**2. ELB+오토스케일링**
- ELB가 트래픽 감시 및 분배, 오토스케일링 그룹이 인스턴스 자동 증가/복구

**3. Multi-Region 아키텍처**
- 중요 서비스 및 데이터 글로벌 다중 리전에 분산, 지역 재해에도 서비스 지속(Active-Active, Active-Passive)

**4. 서버리스 및 매니지드 서비스 활용**
- Lambda, DynamoDB, Aurora Serverless, ECS/Fargate 등 AWS에서 내장적 HA 제공

**5. 무중단 배포(Blue/Green, Rolling, Canary 등)**
- 배포 과정 오류·실패에도 Downtime 없이 서비스 지속

## 주요 활용 사례

- EC2 웹·앱서버를 여러 AZ에 분산, ELB로 부하분산 및 장애시 즉시 우회
- RDS, Aurora, DynamoDB의 Multi-AZ/Region 복제와 자동 장애조치
- S3/Glacier 내구성·복제 및 자동 Failover
- 기업 미션 크리티컬(금융, 공공, 헬스케어 등) 서비스의 99.99% 이상의 업타임 구현
- 서버리스 서비스의 SLA 99.95% 이상 실현

## 기본 워크플로/설계

### 1. 멀티 AZ 인프라 배포
- 리소스 생성 시 반드시 2개 이상 AZ에 배포 옵션 선택/구성

### 2. 오토스케일링 그룹+ELB 설정
- Auto Scaling 그룹 생성 → ELB 연결 → 헬스체크·최소/최대 인스턴스 지정

### 3. RDS/Aurora Multi-AZ 활성화
- DB 생성 시 Multi-AZ 옵션 선택 또는 리소스 확장

### 4. 데이터 복제·백업
- S3, DynamoDB, RDS 등에서 교차 AZ/Region 복제 및 백업 정책 활성화

### 5. 상태 감지·자동화 대응
- CloudWatch, Lambda 등으로 상태 모니터링 및 장애 발생 시 이벤트 트리거
- Route53 DNS Failover, AutoRecovery EC2 등 장치별 자동화 대응

## 비용 구조

- 이중화/복제/예비 인프라 확충으로 단독 인프라보다 비용 증가(최소 2배 이상)
- ELB, Auto Scaling, 데이터 복제, 크로스 리전 트래픽, 백업 등 별도 요금 발생
- SLA 기반의 고가용성 옵션은 추가 요금 반영

## 보안 및 운영 모범 사례

- AZ/Region 별 리소스 분산 및 지리적 리스크 완화
- 트래픽·데이터·백업·인증 모두 다중화, SPoF 완전 제거
- 주기적 장애 테스트, DR 훈련, 상태 모니터링 자동화
- IaC 기반 표준 아키텍처 관리 및 배포, 리소스 태그/정책으로 사고 예방

AWS HA는 비즈니스 연속성과 서비스 신뢰성을 극대화하는 현대 클라우드 인프라의 핵심 성공 전략이다.
