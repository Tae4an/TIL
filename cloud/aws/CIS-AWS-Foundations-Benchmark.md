# CIS AWS Foundations Benchmark란?

## 개념 정의
CIS AWS Foundations Benchmark는 **클라우드 환경의 보안 모범 사례와 기준을 제공하는 국제 표준 가이드**로, CIS(Center for Internet Security, 인터넷 보안 센터)가 AWS(아마존 웹 서비스) 사용자들이 클라우드 보안 구조를 “최소 권장 기준(Baseline)”에 맞게 구조화·운영할 수 있도록 만든 체크리스트이자 권고안이다.

쉽게 말해 “AWS 환경을 처음부터 안전하게 만들기 위한 보안/구성 체크리스트+자동화 기준서”이다.

## 목적 및 활용

- AWS 계정/서비스/리소스의 기본 보안 구성을 검증·강화하는 기준 제공
- 보안 감사, 규정 준수(컴플라이언스), 관리·운영 정책 수립에 바로 활용
- Security Hub, Config 등 AWS 관리 도구에서 자동 진단 및 평가 리포트의 근거로 사용

## 주요 구조/항목

**1. 계정 및 인증 관리**
   - Root 계정 보안(사용 차단, MFA 적용)
   - IAM 사용자의 최소 권한 원칙 적용 및 강력 정책
   - 액세스 키, 콘솔 로그인 모니터링·비활성화

**2. 로깅 및 모니터링**
   - CloudTrail 전 계정/리전 활성화 및 로그 집합(집중 저장)
   - S3/Buckets의 버전관리, 로그 무결성 강화
   - AWS Config, CloudWatch, VPC 흐름 로그 등 보안 로그 필수 활성화

**3. 네트워크 관리**
   - 기본 VPC 보안 그룹 규칙 Harden(과도한 허용 정책 금지)
   - NACL, 보안 그룹, 라우팅 테이블의 보안 정책 정비

**4. 데이터 보호**
   - S3/DB/EBS 등 저장 데이터 암호화(기본 활성화)
   - KMS(키관리/암호화키) 정책 최적화
   - S3 퍼블릭 접근 차단

**5. 감사 및 경고**
   - 이상 상황, 미준수 항목은 자동 알림·대응 프로세스 적용
   - Security Hub, EventBridge, SNS 등으로 즉각 통보 및 리미디에이션 연결

## 실제 운영에서의 적용 방법

- Security Hub 등에서 “CIS AWS Foundations Benchmark” 평가 활성화
- 자동 스캔 통해 미준수 항목 발견 → 대응 우선순위 및 시정 조치
- 기업 내부 규정·지침(ISO, ISMS, NIST 등)과 연계하여 보안 프레임워크 기반으로 활용
- 감사 리포트, 월별 취약 항목 추적·대응에 표준 레퍼런스로 사용

## 장점

- 국제적으로 인정된 AWS 보안 준수의 “기본 표준”
- 신규/기존 AWS 환경 모두 즉시 적용 가능한 실무 중심 체크리스트
- 자동화 도구 및 AWS 네이티브 서비스와 연동해 지속적인 점검·감사 가능

## 참고사항

- CIS Benchmark는 버전(예: v1.4, v2.0 등)별로 항목·세부 기준이 다를 수 있으니 최근 버전을 기준으로 적용 및 검토
- 반드시 벤치마크 모든 항목을 강제로 적용하기보다, 조직/서비스별 위험도·비용·운영 편의성을 고려해 “우선순위” 기준으로 점진적 적용/강화를 권장

CIS AWS Foundations Benchmark는 AWS 보안을 체계적으로 관리하고, 글로벌 표준 수준에서 위험요인을 꾸준히 점검할 수 있게 해주는 필수 체크리스트/가이드다.
