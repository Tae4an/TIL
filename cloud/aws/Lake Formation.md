# AWS Lake Formation란?

## 개념 정의
AWS Lake Formation은 S3 위에 데이터 레이크를 빠르게 구축하고 중앙에서 권한·카탈로그·거버넌스를 관리하는 완전 관리형 서비스로, Glue Data Catalog를 활용해 S3·RDS 등 데이터 소스를 통합 카탈로그화하고 테이블·컬럼 단위 세밀한 권한 제어를 제공한다.

쉽게 말해 "S3에 흩어진 데이터를 중앙에서 카탈로그화하고, 누가 어떤 테이블·컬럼을 볼 수 있는지 관리해주는 데이터 레이크 거버넌스 서비스"다.

## 핵심 특징

### 데이터 레이크 구축 자동화
S3 버킷이나 RDS 위치만 지정하면 크롤링·카탈로그 등록·포맷 변환(Parquet/ORC)을 자동으로 처리해 데이터 레이크를 만든다. Glue ETL을 통합해 데이터 변환도 지원한다.

### 중앙 메타데이터 카탈로그
Glue Data Catalog를 중앙 카탈로그로 사용해 Athena, Redshift Spectrum, EMR, Glue Job 등이 공통 메타데이터를 공유한다. 스키마 추론·파티셔닝 자동화도 제공한다.

### 세밀한 권한 관리
IAM 역할 기반으로 데이터베이스·테이블·컬럼 단위 권한을 정의하고, 쿼리 실행 시 Lake Formation이 권한을 강제 적용한다. LF-Tags로 태그 기반 정책도 가능하다.

### 데이터 공유·협업
다른 AWS 계정·리전으로 데이터를 복사 없이 공유하며, 데이터 계보(Lineage)를 추적해 데이터 출처와 변환 이력을 확인할 수 있다.

## 주요 구성 요소

### 1. 데이터 카탈로그
Glue Data Catalog를 확장해 데이터 레이크 전용으로 사용하며, 데이터베이스·테이블·컬럼 메타데이터를 중앙 관리한다.

### 2. 권한 모델
- **데이터 레이크 권한**: 데이터베이스·테이블·셀 수준 권한 설정
- **LF-Tags**: 키-값 태그로 권한 정책 적용 (예: "PII=High" 태그 가진 컬럼 제한)
- **권한 상속**: 데이터베이스 권한이 테이블로 상속

### 3. 데이터 로드
- **Register**: 기존 S3 데이터 등록
- **Crawl**: Glue Crawler로 자동 카탈로그화
- **Ingest**: RDS 등 외부 소스에서 S3로 데이터 복제

### 4. 분석 통합
Athena(서버리스 쿼리), Redshift Spectrum(분석용), EMR(Spark), Glue ETL 등과 완전 통합된다.

## 활용 사례

### 데이터 레이크 중앙 거버넌스
여러 팀이 S3에 데이터를 쌓아도 Lake Formation에서 권한·카탈로그를 중앙 통제해 데이터 실루와 규제 준수를 보장한다.

### 멀티 계정 데이터 공유
본사 계정에서 개발·운영 계정으로 데이터를 안전하게 공유하며, 권한만 부여해 데이터 복제 비용을 절감한다.

### PII·민감 데이터 관리
LF-Tags로 "PII=true" 컬럼을 특정 역할만 접근 가능하게 제한하고, 감사 로그로 컴플라이언스를 증명한다.

## 요약
AWS Lake Formation은 S3 데이터 레이크 위에 중앙 Glue Data Catalog와 테이블·컬럼 단위 권한 관리, LF-Tags 기반 정책, 계정 간 데이터 공유, 데이터 계보 추적을 제공하는 거버넌스 서비스로, Athena·Redshift Spectrum·EMR 등 분석 서비스와 통합되어 데이터 엔지니어가 인프라 대신 거버넌스에 집중할 수 있게 한다.
