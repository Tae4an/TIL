# IAM Access Analyzer란?

## 개념 정의  
IAM Access Analyzer는 AWS IAM(Identity and Access Management) 서비스의 내장 기능으로, **AWS 리소스(예: S3, IAM, KMS, SNS, SQS 등)가 외부(타 계정, 조직 밖, 퍼블릭)에서 접근 가능하도록 설정된 경우를 자동 탐지·분석하는 보안 진단 도구**다.  
즉, “내 AWS 리소스가 의도치 않게 외부에 공개되어 있거나, 잘못된 정책 때문에 타 계정에서 접근 가능한 상태를 자동으로 알려주는 보호 기능”이다.

## 핵심 특징

- **외부 접근 탐지**  
  - 정책(리소스 정책, ACL, 버킷 정책 등)을 분석해 조직 내/외부 또는 인터넷에서 접근이 가능한 항목을 자동 식별
  - 퍼블릭(Everyone), 외부 AWS 계정, 다른 조직, 외부 서비스(예: CloudFront, Lambda 등)까지 타겟 분석

- **자동 분석 및 알림**  
  - Access Analyzer 활성화 시, 리소스 정책 변경 사항을 실시간 감시
  - 분석 결과는 Finding(경고, 탐지 내역)으로 기록되고, 콘솔·API·CloudWatch Events 등으로 알림 표시

- **적용 대상 리소스**  
  - S3 버킷, IAM 역할, KMS 키, SQS 큐, SNS 토픽, Secrets Manager, ECR 리포지터리 등 AWS의 정책 기반 액세스 제어가 있는 리소스에 적용 가능
  - 조직/계정 단위 및 여러 리전에 동시에 활성화 가능

- **정책 시뮬레이션/권고**  
  - 탐지된 외부 접근 가능 리소스에 대해 “권장 조치(Recommended Action)” 제시
  - 필요한 경우 정책 자동 수정을 위한 코드/페이지 바로가기 제공
  - 신규 정책 변경 전에 “시뮬레이션” 모드로 미리 효과를 검증

- **Automated Reasoning 기반**  
  - AWS 자체 수학적 검증 엔진으로, 정책의 실제 효과(Who, What, Where)를 자동으로 해석
  - 복잡한 정책도 빠르게 경계·위험 항목 검증 가능

## 주요 활용 사례

- S3 버킷, KMS 키 등 민감 리소스의 의도치 않은 퍼블릭 노출 여부 점검  
- 신규 프로젝트/배포/팀의 권한 정책 변경 시 사전 검사 및 실시간 감시  
- 조직-wide 보안 진단, 여러 계정·리전에 걸쳐 의심스러운 외부 접근 자동 탐지  
- 컴플라이언스 대응(ISO, SOC, NIST 등) 및 감사 준비시 자동화 도구로 활용  
- CloudWatch Events와 연계한 위험 이벤트 자동 대응 워크플로 구성

## 기대 효과 및 운영 팁

- 보안 사고(데이터 유출, 잘못된 권한 공개) 사전 예방  
- 정책 관리 복잡도 감소, 빠른 진단과 수정  
- 최소 권한 원칙 및 클라우드 보안 거버넌스 강화  
- 콘솔/CLI/API/CloudWatch 연동으로 DevSecOps 파이프라인에 손쉽게 연결  
- 대규모 멀티 계정·멀티 리전 환경에서도 효율적 보안 감시 실현

IAM Access Analyzer는 AWS 환경에서 "내가 모르는 사이 공개된 리소스"를 찾고, 실시간으로 퍼블릭/외부 접근 위험을 해결하는 데 매우 강력한 자동화 보안 도구다.
