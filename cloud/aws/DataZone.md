# AWS DataZone이란
## 1. 개념 요약
Amazon DataZone은 **AWS·온프레미스·서드파티 등 여러 소스에 분산된 데이터를 카탈로그화·검색·공유·거버넌스하는 완전관리형 데이터 관리 서비스**다. 쉽게 말하면, **"조직 전체의 데이터 자산을 하나의 포털에서 찾고, 접근 요청하고, 안전하게 공유할 수 있게 해주는 데이터 마켓플레이스·거버넌스 허브"** 라고 보면 된다.

***
## 2. 핵심 구성 요소
- **도메인(Domain)**
  - DataZone의 최상위 단위로, 조직 또는 사업부 단위로 데이터 자산·사용자·프로젝트를 묶는 논리적 경계다.
  - 도메인 하위에 비즈니스 단위(Domain Unit)를 두어 팀·부서별로 세분화 관리할 수 있다.

- **비즈니스 데이터 카탈로그(Business Data Catalog)**
  - S3, Redshift, Glue, RDS 등의 데이터 자산을 수집·색인해 비즈니스 컨텍스트(설명, 소유자, 태그)와 함께 카탈로그화한다.
  - 데이터 엔지니어·분석가·비즈니스 사용자 모두 검색 포털에서 필요한 데이터를 쉽게 찾을 수 있다.

- **프로젝트(Project)**
  - 공통 목적을 가진 사람·데이터·도구의 협업 그룹이다.
  - 프로젝트 멤버는 데이터를 카탈로그에 게시(Publish)하거나, 다른 프로젝트의 데이터를 구독 신청(Subscribe)할 수 있다.

- **구독 워크플로(Subscription Workflow)**
  - 데이터 소비자가 원하는 자산에 접근 요청을 보내면, 데이터 소유자가 승인·거부하는 거버넌스 승인 흐름이 내장되어 있다.
  - 승인되면 DataZone이 자동으로 Lake Formation·Redshift 등의 권한을 부여한다.

- **데이터 계보(Data Lineage)**
  - 데이터가 어디서 왔고, 어떻게 변환·이동했는지를 시각화해준다.
  - OpenLineage 표준을 지원하며, 데이터 오류 발생 시 근본 원인 추적에 활용된다.

***
## 3. 주요 연동 서비스
DataZone은 AWS 분석 스택과 깊게 통합되어 있다.

- **데이터 소스**: Amazon S3, Redshift, Athena, Glue Data Catalog, RDS
- **권한 관리**: AWS Lake Formation, IAM
- **쿼리·분석**: Amazon Athena, Redshift Query Editor
- **메타데이터 자동 수집**: AWS Glue Crawler 연동으로 스키마 자동 수집

***
## 4. 어디에 쓰이는가
- **데이터 민주화**: 데이터 팀이 아닌 비즈니스 사용자도 포털에서 직접 필요한 데이터를 검색·요청해 인사이트를 도출.
- **데이터 거버넌스·컴플라이언스**: 누가 어떤 데이터에 접근했는지 감사 로그를 남기고, 민감 데이터 접근을 승인 워크플로로 통제.
- **멀티 계정 데이터 공유**: 여러 AWS 계정에 분산된 데이터를 하나의 DataZone 도메인에서 통합 관리·공유.
- **데이터 마켓플레이스 구축**: 조직 내 부서 간 데이터를 사고파는(구독) 내부 데이터 마켓 운영 (LG 에너지솔루션 사례 등).

***
## 5. 유사 서비스 비교
| 항목 | Amazon DataZone | AWS Glue Data Catalog | AWS Lake Formation |
|---|---|---|---|
| 주요 목적 | 비즈니스 데이터 거버넌스·공유 | 기술 메타데이터 저장소 | 데이터 레이크 접근 제어 |
| 사용자 대상 | 비즈니스·분석·엔지니어 통합 | 주로 엔지니어·개발자 | 주로 데이터 엔지니어·관리자 |
| 승인 워크플로 | 내장 | 없음 | 없음 |
| 데이터 검색 포털 | 비즈니스 카탈로그 포털 제공 | 없음 | 없음 |
| 데이터 계보 | 지원 | 제한적 | 없음 |

***
## 6. 한 줄 정리
Amazon DataZone은 **조직 전체의 데이터 자산을 카탈로그·검색·구독 워크플로·계보 추적으로 통합 관리하는 데이터 거버넌스 서비스로, "데이터 찾기·요청·승인·접근"을 하나의 포털에서 해결해 데이터 민주화와 컴플라이언스를 동시에 실현한다.**
