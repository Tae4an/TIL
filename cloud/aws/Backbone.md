# AWS Backbone이란?

## 1. 개념 요약

AWS Backbone(또는 AWS Global Network Backbone)은 전 세계 AWS 리전, 가용 영역(AZ), 엣지 로케이션을 서로 연결하는 AWS의 전용 글로벌 초고속 광 네트워크를 말한다.  
쉽게 말해, “AWS가 직접 운영하는 전 세계용 전용 인터넷망(통신사급 백본망)”이라고 보면 된다.  

***

## 2. 특징

- 글로벌 전용 광 네트워크  
  - 수많은 리전·AZ·엣지(CloudFront, Global Accelerator 등)를 잇는 대규모 사설 광 네트워크.  
  - 100G/400G급 링크로 구성된 캐리어급(티어1 ISP 수준) 백본 구조.  

- 고성능·저지연  
  - 리전 간 트래픽이 일반 인터넷이 아니라 AWS 사설망을 타기 때문에 지연·지터가 낮고, 경로 품질이 예측 가능하다.  

- 높은 가용성과 복원력  
  - 다중 경로, 중복 링크, 자동 우회가 기본 설계라 광케이블 절단 등 장애 시에도 빠르게 우회한다.  

- 강력한 암호화  
  - 시설을 나가는 트래픽은 여러 계층(L1/L2/L4 등)에서 암호화되어 이동한다.  

***

## 3. 어디에 쓰이는가

- 리전 간 통신  
  - Cross‑Region VPC Peering, Aurora Global Database, S3 크로스 리전 복제 등 리전 간 서비스 통신이 이 백본을 활용한다.  

- 엣지 서비스  
  - CloudFront, Route 53, Global Accelerator 등은 사용자 트래픽을 가장 가까운 엣지에서 받아, AWS Backbone을 통해 최적 리전으로 전달해 성능을 높인다.  

- 하이브리드/글로벌 WAN  
  - Direct Connect, Site‑to‑Site VPN, Cloud WAN, Transit Gateway와 조합해 온프레미스–지사–AWS 간 글로벌 네트워크를 AWS Backbone 위에서 설계할 수 있다.  

***

## 4. Public Internet과의 차이

| 항목        | Public Internet                          | AWS Backbone                                |
|-------------|------------------------------------------|---------------------------------------------|
| 경로        | 여러 ISP·라우터를 거치는 공개망         | AWS가 소유·운영하는 전용 글로벌망          |
| 지연/지터   | 지역·시간·혼잡도에 따라 변동 큼         | 더 일관되고 낮은 지연·지터                 |
| 안정성      | 혼잡·라우팅 변경 영향 큼                | 엔지니어링된 고가용·다중 경로 구조         |
| 보안        | 애플리케이션 수준 암호화에 의존         | 사설망 + 물리/링크/전송 계층 암호화 조합   |

일반 인터넷보다 안정적이고 예측 가능한 경로를 제공해야 할 때(글로벌 서비스, 실시간 서비스, 금융 등) AWS 백본 경로(Global Accelerator, 리전 간 통신 등)를 의도적으로 사용하는 경우가 많다.  

***

## 5. 한 줄 정리

AWS Backbone은 전 세계 AWS 인프라를 잇는 AWS 전용 글로벌 네트워크로, 리전·엣지·온프레미스를 고성능·고안정성·고보안으로 연결하는 “AWS의 자체 글로벌 백본망”이라고 이해하면 된다.
