# Fortinet
Fortinet(포티넷)은 네트워크 경계부터 내부·클라우드까지 **방화벽 + UTM + SD‑WAN + 로그/분석**을 통합 제공하는 보안 벤더다. 대표 제품은 차세대 방화벽/UTM 장비인 **FortiGate**다.  

***

## Fortinet이 하는 일 한 줄로  

“네트워크 입구마다 FortiGate 같은 장비를 세워서 트래픽을 통제하고(방화벽·VPN), 각종 공격을 막고(IPS/AV/웹 필터링), 지점·클라우드까지 하나의 보안 패브릭으로 묶어 관리하게 해주는 보안 플랫폼 회사”로 보면 된다.  

***

## UTM/NGFW 관점에서 FortiGate 역할  

- **UTM(통합 위협 관리) + NGFW**  
  - 기존 방화벽, IPS, 안티바이러스, 웹 필터링, VPN을 하나의 장비에서 제공하는 형태가 UTM이고, FortiGate는 이 기능을 모두 포함하는 NGFW이자 UTM 장비다.  
  - 필요한 기능(IPS, AV, 웹 필터 등)을 선택적으로 켜서 “순수 방화벽” 또는 “풀 UTM”으로 동작시킬 수 있다.  

- **주요 보안 기능**  
  - L3/L4 방화벽, NAT  
  - IPS/IDS(침입 탐지·방지)  
  - 안티바이러스·안티멀웨어  
  - 웹 필터링(카테고리 기반 URL 차단, DNS 필터)  
  - 애플리케이션 제어(L7 식별, SaaS 제어)  
  - IPsec/SSL VPN, 사용자·그룹 기반 정책  
  - SD‑WAN 기능을 통한 지사–본사–클라우드 연결 최적화  

- **성능 특징**  
  - Fortinet 전용 ASIC(SPU, FortiASIC)을 사용하여 UTM 기능을 켠 상태에서도 높은 처리량을 제공하는 것을 강점으로 내세운다.  

***

## Fortinet 장비가 하는 일 정리  

### 1) FortiGate (NGFW / UTM / SD‑WAN)

- 네트워크 경계에서 트래픽을 허용/차단하고,  
- 애플리케이션·사용자 단위로 세밀하게 정책을 적용하며,  
- 동시에 IPS/AV/웹 필터링·DNS 필터·애플리케이션 제어 등으로 위협을 탐지·차단한다.  
- WAN 구간에서는 SD‑WAN 기능으로 여러 회선을 묶어 품질·비용을 최적화한다.  

### 2) FortiAnalyzer (로그·분석·리포트)

- FortiGate 등 Fortinet 장비의 로그를 중앙에 모아서 저장·검색·시각화·리포트한다.  
- 트래픽/위협 가시성, 규제 준수 보고서, SOC/NOC 운영에 활용된다.  

### 3) FortiManager (정책·구성 중앙 관리)

- 여러 대의 FortiGate에 대한 정책·설정·펌웨어를 중앙에서 일괄 관리한다.  
- 지사 수십·수백 개 환경에서 통일된 정책 배포·변경에 사용된다.  

### 4) FortiSIEM (통합 SIEM)

- Fortinet 제품뿐 아니라 타사 장비 로그까지 수집·연동해  
  실시간으로 이벤트 상관분석, 공격 탐지, 인시던트 대응에 활용되는 SIEM 플랫폼이다.  

***

## Fortinet Security Fabric 관점  

- **Security Fabric**는 FortiGate를 중심으로, FortiSwitch, FortiAP, FortiClient(EDR), FortiMail, FortiWeb(WAF) 등 Fortinet 솔루션을 하나의 통합 프레임워크로 묶은 개념이다.  
- 각 장비가 위협 정보를 공유하고, 중앙에서 통합 관리·분석·자동화(정책 적용, 차단, 알림 등)를 수행할 수 있도록 설계되어 있다.  

***

## 왜 쓰는지 (사용 목적)  

- 여러 개의 포인트 보안 솔루션(방화벽, IPS, AV, 웹 필터 등)을 따로 관리하지 않고 **하나의 플랫폼으로 통합**하려는 조직  
- 본사–지사–클라우드–원격근무 환경에 걸쳐 **일관된 보안 정책과 가시성**을 확보하려는 경우  
- 고성능 UTM/NGFW로 **트래픽 증가**와 **복잡한 애플리케이션(특히 SaaS)**을 제어하고자 하는 엔터프라이즈 환경에서 많이 사용된다.
