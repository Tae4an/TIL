# ADC
ADC(Application Delivery Controller)는 웹/업무 애플리케이션의 **부하 분산 + 성능 최적화 + 보안**을 한 번에 담당하는 L4/L7 로드밸런서 계열 네트워크 장비다.

## ADC가 하는 일 한 줄로

“여러 서버 앞에 서서 트래픽을 똑똑하게 나눠주고(로드 밸런싱), 속도를 최적화하고(캐싱·압축·SSL 오프로드), 애플리케이션 보안까지 챙겨주는 장비”라고 보면 된다.

## L4 스위치와 뭐가 다른지

| 항목 | L4 스위치 | ADC |
|------|-----------|-----|
| 작동 계층 | OSI 4계층 (TCP/UDP) | 주로 7계층 (HTTP/HTTPS 등) |
| 기준 정보 | IP, 포트 | URI, 헤더, 쿠키, 호스트명, 콘텐츠 |
| 주요 기능 | 기본 로드밸런싱, 단순 페일오버 | 고급 L7 로드밸런싱, SSL 오프로드, 캐싱, 압축, 콘텐츠 스위칭, WAF 옵션 등 |
| 적용 관점 | “패킷 단위 분산” | “애플리케이션 단위 딜리버리·보안·최적화” |

대부분의 상용 ADC는 L4 기능을 포함해서 L4+L7를 같이 제공한다.

## 대표 기능

- **로드밸런싱**
  - 라운드 로빈, 최소 연결 수, 응답 시간 기반 등 다양한 알고리즘으로 서버에 트래픽 분산
  - 서버 헬스체크 후 장애 서버는 자동 제외, 다른 서버로 우회
- **애플리케이션 최적화**
  - SSL/TLS 오프로드(서버 대신 암복호화 수행)
  - TCP 멀티플렉싱, Keep-Alive 최적화
  - 캐싱(정적 콘텐츠), 압축(CSS·JS·이미지 등)
- **컨텐츠 기반 라우팅(L7 스위칭)**
  - URL 경로, 호스트명, 쿠키, 헤더에 따라 다른 서버/서비스로 라우팅
- **보안 기능**
  - 기본 DDoS 완화(속도 제한, 연결 제한 등)
  - 일부 제품은 WAF, 인증 연동(SSO/SAML), IP 평판 기반 차단 등 통합

## 왜 쓰는지 (사용 목적)

- 트래픽 증가 시 서버 여러 대로 부하 분산해서 **고가용성** 확보
- 모바일·WAN 환경에서도 **응답 속도 개선**
- 장애 서버 자동 우회로 **무중단 서비스**에 가까운 운영
- 애플리케이션 앞단에서 인증/보안 정책을 통합 적용

## 대표 벤더·제품 예시

- F5 BIG-IP
- Citrix NetScaler(현재 NetScaler 브랜드)
- A10 Networks Thunder 시리즈
- 클라우드 쪽에서는 AWS ALB/NLB, GCP/LB 등이 ADC 역할을 하는 매니지드 서비스에 가깝다.
