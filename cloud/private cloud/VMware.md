# VMware란?

## 개념 정의

VMware는 **가상화(Virtualization) 소프트웨어** 분야의 선두 기업으로, 물리적 서버 하드웨어를 소프트웨어로 추상화하여 하나의 물리 서버에서 여러 개의 독립적인 가상 머신(VM)을 실행할 수 있게 해주는 기술을 제공한다. 2023년 반도체 기업 Broadcom이 약 690억 달러(약 89조원)에 인수했으며, 현재 Broadcom의 인프라 소프트웨어 사업부로 운영되고 있다.

쉽게 말해, "하나의 물리 서버를 여러 개의 독립적인 컴퓨터처럼 나눠 쓸 수 있게 해주는 가상화 소프트웨어 플랫폼"이다.

## 핵심 개념: 가상화와 하이퍼바이저

### 가상화 (Virtualization)
물리 서버의 CPU, 메모리, 스토리지, 네트워크를 소프트웨어로 추상화하여, 하나의 물리 서버 위에 여러 개의 가상 머신을 독립적으로 실행하는 기술이다. 각 VM은 자체 OS와 애플리케이션을 실행하며, 서로 완전히 격리된다.

### 하이퍼바이저 (Hypervisor)
물리 하드웨어와 가상 머신 사이에 위치하는 핵심 소프트웨어 계층으로, VM들이 하드웨어 리소스를 공유·할당받을 수 있게 중재한다.

- **Type 1 하이퍼바이저 (Bare-Metal)**: 물리 하드웨어에 직접 설치 → VMware ESXi
- **Type 2 하이퍼바이저 (Hosted)**: 기존 OS 위에 설치 → VMware Workstation, VMware Fusion

## 주요 제품군

### vSphere (서버 가상화의 핵심)
- VMware의 핵심 서버 가상화 플랫폼
- **ESXi**: Type 1 하이퍼바이저, 물리 서버에 직접 설치되어 VM을 실행하는 엔진
- **vCenter Server**: 여러 ESXi 호스트와 VM을 중앙에서 통합 관리하는 관리 플랫폼
- 기업 데이터센터에서 가장 널리 쓰이는 서버 가상화 솔루션

### vMotion
- 실행 중인 VM을 **무중단**으로 다른 물리 서버로 실시간 이동하는 기능
- 서버 유지보수, 부하 분산, 장애 대응 시 서비스 중단 없이 VM 이전 가능

### vSAN (스토리지 가상화)
- 여러 서버의 로컬 디스크를 소프트웨어로 묶어 공유 스토리지 풀 생성
- 별도의 SAN 스토리지 장비 없이 소프트웨어 정의 스토리지(SDS) 구현

### NSX (네트워크 가상화)
- 물리 네트워크 장비 없이 소프트웨어로 네트워크를 가상화
- VLAN, 방화벽, 로드밸런서, VPN 등을 소프트웨어로 정의·관리
- 마이크로세그멘테이션으로 VM 간 트래픽을 세밀하게 제어

### VMware Cloud Foundation (VCF)
- vSphere + vSAN + NSX를 하나의 패키지로 묶은 풀스택 프라이빗 클라우드 플랫폼
- Broadcom 인수 후 핵심 제품으로 집중 육성 중
- 온프레미스 데이터센터를 퍼블릭 클라우드 수준으로 자동화·현대화

### VMware Workstation / Fusion
- 개발자·테스터가 개인 PC에서 사용하는 Type 2 하이퍼바이저
- Windows에서 Linux, Linux에서 Windows 등 여러 OS를 동시에 실행
- 개발·테스트 환경 구성에 많이 활용

## 주요 기능

- **HA (High Availability)**: 물리 서버 장애 시 VM을 다른 서버에 자동 재기동
- **DRS (Distributed Resource Scheduler)**: 클러스터 내 부하를 자동 분산, VM을 최적 호스트로 자동 이동
- **스냅샷(Snapshot)**: 특정 시점의 VM 상태 저장, 언제든지 해당 시점으로 롤백 가능
- **VM 암호화**: VM 단위 데이터 암호화로 보안 강화
- **vMotion**: 실행 중인 VM 무중단 마이그레이션

## 서버 가상화의 장점

- **서버 통합**: 물리 서버 수를 줄여 하드웨어·전력·공간 비용 70~80% 절감
- **리소스 효율화**: 유휴 서버 리소스를 다른 VM이 공유 사용
- **빠른 프로비저닝**: 새 VM 생성이 수분 내 완료 (물리 서버 도입 대비 수주~수개월 단축)
- **재해 복구**: 스냅샷, 복제, HA로 빠른 복구 가능
- **환경 격리**: 개발·스테이징·운영 환경을 동일 서버에서 완전히 격리

## Broadcom 인수 이후 변화

2023년 11월 Broadcom이 VMware를 인수한 이후 큰 변화가 있었다.

- **영구 라이선스 폐지**: 기존 영구 라이선스 판매 종료, 구독(Subscription) 방식으로 전환
- **번들 패키지 중심 개편**: VCF(VMware Cloud Foundation) 중심의 풀스택 패키지 판매로 재편
- **가격 인상**: 구독 전환과 함께 대규모 라이선스 가격 인상으로 기업 고객 반발
- **파트너 생태계 축소**: 리셀러·MSP 파트너 정책 변경으로 채널 생태계 변화
- **Carbon Black·EUC 사업 매각**: 보안(Carbon Black)과 엔드유저 컴퓨팅(EUC) 사업을 분리·매각

이로 인해 일부 기업들이 VMware 대안으로 Proxmox, Nutanix, Microsoft Hyper-V, OpenStack 등을 검토하는 움직임이 생기고 있다.

## VMware와 AWS의 관계

- **VMware Cloud on AWS**: VMware의 vSphere 환경을 AWS 데이터센터에 그대로 올려 실행하는 서비스로, 온프레미스 VMware 워크로드를 코드 변경 없이 AWS로 마이그레이션 가능
- **AWS Outposts**: AWS 하드웨어를 온프레미스에 설치하는 방식으로, VMware 환경을 AWS로 점진 전환 시 병행 활용 가능
- **App2Container**: 온프레미스 VMware에서 실행 중인 Java/.NET 앱을 컨테이너화하여 AWS ECS/EKS로 이전

## 정리

VMware는 하이퍼바이저(ESXi)를 핵심으로 서버·스토리지·네트워크 가상화를 제공하는 기업용 인프라 소프트웨어 플랫폼으로, vSphere·vSAN·NSX·VCF로 구성된 풀스택 프라이빗 클라우드 솔루션을 제공하며, HA·DRS·vMotion·스냅샷으로 고가용성과 운영 효율화를 구현하고, Broadcom 인수 이후 영구 라이선스 폐지·구독 전환·가격 인상으로 기업 고객의 대안 검토가 늘고 있으며, AWS와의 연계를 통해 온프레미스 VMware 환경을 클라우드로 점진 전환하는 하이브리드 마이그레이션 경로로도 활용된다.
