# NFS란?

## 1. 개념 요약

NFS(Network File System)는 **네트워크를 통해 원격 서버의 파일 시스템을 로컬 디스크처럼 마운트해 사용할 수 있는 분산 파일 시스템 프로토콜**이다.  
1984년 Sun Microsystems가 개발했으며, **"다른 서버의 디렉터리를 내 로컬 폴더처럼 읽고 쓸 수 있게 해주는 네트워크 파일 공유 표준"** 으로 Unix·Linux 환경에서 가장 널리 쓰인다.

***

## 2. 특징

- **투명한 원격 접근**
  - 클라이언트 입장에서 원격 디렉터리가 로컬 파일 시스템과 동일하게 보임
  - 애플리케이션이 NFS 여부를 인식하지 않아도 됨

- **클라이언트-서버 구조**
  - NFS 서버: 디렉터리를 `export`(공유)해 네트워크에 제공
  - NFS 클라이언트: 공유 디렉터리를 `mount`해 로컬처럼 사용

- **RPC 기반 통신**
  - 내부적으로 RPC(원격 프로시저 호출) 사용
  - NFSv4 기준 TCP 2049 포트 단일 사용

- **다중 클라이언트 동시 접근**
  - 여러 서버·클라이언트가 동일한 NFS 공유에 동시 접근 가능

- **권한 제어**
  - `rw`(읽기/쓰기), `ro`(읽기 전용), `root_squash` 등 세밀한 접근 제어

***

## 3. 어디에 쓰이는가

- **공유 스토리지**
  - 여러 웹·앱 서버가 동일한 파일(이미지·로그·설정)을 공유할 때

- **Kubernetes Persistent Volume**
  - K8s에서 PV(Persistent Volume) 백엔드로 NFS 사용해 Pod 간 공유 스토리지 제공

- **AWS EFS**
  - Amazon EFS(Elastic File System)는 NFS 프로토콜 기반의 완전관리형 서비스

- **HPC·AI 학습 환경**
  - 대용량 학습 데이터를 여러 GPU 서버가 NFS로 공유해 접근

- **백업·아카이빙**
  - 중앙 NFS 서버에 여러 호스트 데이터를 집중 백업

***

## 4. NFS 버전 비교

| 항목 | NFSv3 | NFSv4 |
|---|---|---|
| **상태 관리** | Stateless(무상태) | Stateful(상태 유지) |
| **포트** | 여러 포트(111, 2049 등) | 단일 포트(2049) |
| **방화벽 통과** | 어려움 | 단순(포트 1개) |
| **보안** | AUTH_SYS 기반 | Kerberos 인증 지원 |
| **성능** | 양호 | 개선된 캐싱·위임 |
| **WAN 지원** | 비권장 | 지원 |

***

## 5. 한 줄 정리

NFS는 **네트워크를 통해 원격 서버의 파일 시스템을 로컬처럼 마운트해 공유**하는 표준 프로토콜로, Kubernetes PV·AWS EFS·HPC 환경까지 광범위하게 쓰이는 "네트워크 파일 공유의 사실상 표준"이다.
