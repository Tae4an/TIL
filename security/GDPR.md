# GDPR란?

## 개념 정의
GDPR(General Data Protection Regulation, 일반 데이터 보호 규정)은 유럽연합(EU)이 2016년 제정하고 2018년 5월 25일부터 시행한 개인정보 보호 및 프라이버시 관련 법률로, EU 시민·거주자의 개인 데이터를 수집·처리·저장하는 모든 조직에 적용되며 위반 시 전 세계 연간 매출의 최대 4% 또는 2,000만 유로 중 높은 금액의 과징금을 부과하는 세계에서 가장 강력한 데이터 보호 법규다.

쉽게 말해 "EU 시민의 개인정보를 다루는 모든 기업(위치 무관)이 반드시 지켜야 하는 글로벌 개인정보 보호 법으로, 개인이 자신의 데이터를 통제할 권리를 보장하고 위반 시 천문학적 벌금을 부과한다."

## 핵심 특징

### 역외 적용
EU 외부 기업도 EU 시민에게 상품·서비스 제공하거나 행동을 모니터링하면 적용된다. 예를 들어 한국 쇼핑몰이 유럽 고객에게 배송하면 GDPR 준수 의무가 발생한다.

### 강력한 처벌
최대 전 세계 연간 매출의 4% 또는 €2,000만 중 높은 금액의 과징금이 부과되며, Amazon €7억 4,600만, Meta €12억, Google €5,000만 등 실제 거액 과징금 사례가 다수 존재한다.

### 데이터 주체 권리 강화
정보 접근권, 수정권, 삭제권(잊힐 권리), 처리 제한권, 데이터 이동권, 반대권, 자동화 의사결정 거부권 등 8가지 권리를 보장한다.

## 7대 원칙
1. **합법성·공정성·투명성**: 법적 근거 필요, 명확한 고지
2. **목적 제한**: 수집 목적만 사용
3. **데이터 최소화**: 필요한 것만 수집
4. **정확성**: 정확하고 최신 상태 유지
5. **보관 제한**: 목적 달성 후 삭제
6. **무결성·기밀성**: 암호화·접근 제어
7. **책임성**: 준수 입증 의무

## 6가지 법적 근거
개인 데이터 처리를 위해서는 최소 1개의 법적 근거가 필요하다.

1. **동의**: 자발적·구체적·명시적 동의 (사전 체크 금지)
2. **계약**: 계약 이행 필수 데이터
3. **법적 의무**: 법률이 요구 (세법 등)
4. **생명 보호**: 응급 상황
5. **공익**: 공공 기관 업무
6. **정당한 이익**: 사기 탐지 등

## 핵심 의무

### 동의 관리
명확하고 구체적인 동의를 받아야 하며, 사전 체크 금지, 동의 철회는 동의만큼 쉬워야 한다.

### 개인정보 침해 통지
침해 발생 시 72시간 이내 감독기구 통지, 고위험 침해 시 데이터 주체에게도 통지해야 한다.

### DPO (데이터 보호 책임자)
공공 기관, 대규모 민감 정보 처리, 대규모 모니터링 시 DPO 지정이 의무다.

### 보안 조치
저장·전송 시 암호화, 접근 제어, Privacy by Design(설계 단계부터 프라이버시 고려)이 필요하다.

### 필수 문서
개인정보 처리방침, 처리 활동 기록(250명 이상 기업), 데이터 보호 영향 평가(고위험 처리 시)를 문서화해야 한다.

### 제3국 데이터 이전
적정성 결정 국가(한국, 일본 등), 표준 계약 조항(SCC), 또는 명시적 동의가 필요하다.

## 실무 체크리스트
- 데이터 매핑: 수집·저장 데이터 문서화
- 법적 근거 지정: 처리 활동별 6가지 근거 중 선택
- 개인정보 처리방침 업데이트
- 동의 메커니즘 점검: 사전 체크 제거
- 데이터 주체 권리 대응 절차: 1개월 내 응답
- 침해 대응 계획: 72시간 통지 절차
- 정기 교육 및 벤더 계약 검토

## 요약
GDPR은 EU 시민 데이터를 다루는 전 세계 모든 기업에 적용되며, 7대 원칙과 6가지 법적 근거 준수, 명시적 동의 관리, 72시간 침해 통지, 데이터 주체 8가지 권리 보장, 암호화·접근 제어, DPO 지정(조건부), 개인정보 처리방침·처리 활동 기록·DPIA 문서화가 핵심이고, 위반 시 최대 연간 매출 4% 또는 €2,000만 과징금이 부과된다.
