# ISMS-P란?

## 개념 정의
ISMS-P(Information Security Management System - Personal Information Protection, 정보보호 및 개인정보보호 관리체계 인증)는 기업·기관이 정보보호와 개인정보보호를 위한 일련의 조치와 활동을 체계적으로 수립·운영하고 있음을 한국인터넷진흥원(KISA) 또는 인증기관이 객관적으로 평가·증명하는 대한민국의 국가 인증 제도다.

쉽게 말해 "회사가 고객 개인정보와 중요 정보를 안전하게 지키기 위한 보안 관리체계를 제대로 갖추고 운영하고 있다는 국가 공인 인증"이다.

## 핵심 특징

**정보보호 + 개인정보보호 통합 관리**
- 기존 ISMS(정보보호 관리체계)와 PIMS(개인정보보호 관리체계)를 2018년 통합
- 한 번의 인증으로 정보보호와 개인정보보호를 모두 증명
- 중복 심사 부담 제거

**법적 의무 인증**
- 일정 규모 이상 정보통신서비스 제공자는 의무 인증
- 전년도 3개월간 평균 일일 이용자 100만 명 이상
- 정보통신서비스 부문 전년도 매출액 100억 원 이상
- 클라우드 컴퓨팅 서비스 제공자 (일부)
- 집적정보통신시설 사업자

**리스크 기반 접근(Risk-Based Approach)**
- PDCA(Plan-Do-Check-Act) 사이클 기반
- 위험 평가 → 보호대책 수립 → 실행 → 점검 → 개선
- 조직별 맞춤형 보안 체계 구축

**지속적 관리체계**
- 일회성 심사가 아닌 지속적 운영·개선
- 최초 인증 후 3년마다 갱신 심사
- 매년 사후 관리 심사 (유지 심사)

**국가 공인 인증**
- 과학기술정보통신부 + 개인정보보호위원회 공동 주관
- KISA(한국인터넷진흥원), FSI(금융보안원) 인증 수행
- 법적 효력 및 대외 신뢰도 높음

## ISMS vs ISMS-P 차이

| 구분 | ISMS | ISMS-P |
|------|------|--------|
| 정식 명칭 | 정보보호 관리체계 인증 | 정보보호 및 개인정보보호 관리체계 인증 |
| 범위 | 정보보호만 | 정보보호 + 개인정보보호 |
| 인증 기준 | 80개 항목 (1영역 + 2영역) | 101개 항목 (1영역 + 2영역 + 3영역) |
| 적용 대상 | 개인정보 미처리 조직 | 개인정보 처리 조직 |
| 예시 | 내부 시스템만 운영 | 고객 정보 수집·처리 서비스 |

## 인증 기준 체계 (101개 항목)

### 1. 관리체계 수립 및 운영 (16개 항목)

**1.1 관리체계 기반 마련 (4개)**
- 경영진의 참여 및 의지 표명
- 정보보호·개인정보보호 정책 수립
- 범위 설정 (인증 범위 명확화)
- 조직 구성 (CISO, 보안 조직)

**1.2 위험 관리 (4개)**
- 정보자산 식별
- 위험 평가 (위협·취약점 분석)
- 보호대책 선정
- 위험 처리 (수용·감소·회피·전가)

**1.3 관리체계 운영 (4개)**
- 보호대책 이행 (기술적·물리적·관리적 통제)
- 사고 예방 및 대응
- 재해 복구 (Business Continuity Plan)
- 공급망 보안 관리

**1.4 관리체계 점검 및 개선 (4개)**
- 법적 요구사항 준수 점검
- 관리체계 점검 (내부 심사)
- 경영진 검토 (Management Review)
- 지속적 개선

### 2. 보호대책 요구사항 (64개 항목)

**2.1 정책·조직·자산 관리 (6개)**
- 정보보호 정책 수립·배포
- 조직 역할·책임 명확화
- 정보자산 분류 및 관리

**2.2 인적 보안 (8개)**
- 주요 직무자 지정·관리
- 보안 서약서
- 정기 보안 교육·훈련
- 퇴직자 보안 (접근 권한 회수)

**2.3  외부자 보안 (5개)**
- 외부자 계약 시 보안 요구사항 명시
- 외부자 접근 통제
- 외부자 보안 이행 점검

**2.4 물리적 보안 (9개)**
- 보호구역 지정 (출입 통제)
- 물리적 접근 통제 (출입 카드, CCTV)
- 사무실·전산실 보안
- 장비 반입·반출 통제

**2.5 인증 및 권한 관리 (7개)**
- 사용자 식별 및 인증
- 비밀번호 관리 (복잡도, 주기적 변경)
- 접근 권한 부여·변경·말소
- 특권 계정 관리

**2.6 접근 통제 (9개)**
- 네트워크 접근 통제 (방화벽)
- 정보시스템 접근 통제
- 응용프로그램 접근 통제
- 데이터베이스 접근 통제
- 무선 네트워크 접근 통제

**2.7 암호화 (3개)**
- 암호 정책 수립
- 암호키 관리
- 암호화 적용 (중요정보, 개인정보)

**2.8 정보시스템 도입·개발 보안 (6개)**
- 보안 요구사항 정의
- 보안 설계·구현·테스트
- 시험·운영 환경 분리
- 소스코드 보안 점검

**2.9 시스템·서비스 운영 관리 (11개)**
- 시스템 보안 설정
- 패치 관리
- 악성코드 대응
- 로그 및 접속 기록 관리
- 백업 및 복구
- 용량 및 성능 관리
- 원격 접근 통제

### 3. 개인정보 처리 단계별 요구사항 (21개 항목)

**3.1 개인정보 수집 시 보호조치 (7개)**
- 개인정보 수집·이용 목적 명시
- 동의 획득
- 최소 수집 원칙
- 민감정보·고유식별정보 처리 제한
- 아동 개인정보 보호

**3.2 개인정보 보유·이용 시 보호조치 (5개)**
- 개인정보 암호화
- 개인정보 접근 통제
- 개인정보 처리 시스템 접근 로그 관리
- 개인정보 재식별 방지 (비식별화)

**3.3 개인정보 제공 시 보호조치 (4개)**
- 개인정보 제공 시 동의
- 개인정보 목적 외 이용·제공 제한
- 개인정보 제3자 제공 관리
- 개인정보 처리 위탁 관리

**3.4 개인정보 파기 시 보호조치 (2개)**
- 개인정보 파기 (보유 기간 만료 시)
- 파기 방법 (복구 불가능하게)

**3.5 정보주체 권리 보호 (3개)**
- 개인정보 열람·정정·삭제 요구권 보장
- 개인정보 처리 정지 요구권
- 개인정보 유출 통지 및 신고

## 인증 대상

### 의무 인증 대상

**정보통신서비스 제공자 (정보통신망법)**
- 전년도 3개월간 평균 일일 이용자 100만 명 이상
- 정보통신서비스 부문 전년도 매출액 100억 원 이상
- 예: 포털, SNS, 쇼핑몰, 게임, OTT

**집적정보통신시설 사업자 (IDC)**
- 연간 매출액 100억 원 이상 또는 일평균 이용자 100만 명 이상 정보통신서비스 제공자의 정보통신시스템이 집적

**클라우드 컴퓨팅 서비스 제공자**
- 일부 조건 충족 시 의무 (클라우드컴퓨팅법)

### 자발적 인증 대상

**스타트업·중소기업**
- 보안 수준 향상·대외 신뢰도 제고
- 공공기관 입찰·제안 시 우대

**B2B·B2G 기업**
- 고객사 요구 (보안 요구사항)
- 공공기관 납품 시 필수

## 인증 절차

### 1. 사전 준비 (3~6개월)

**관리체계 구축**
- 정보보호·개인정보보호 정책 수립
- 위험 평가 수행
- 보호대책 이행 (암호화, 접근 통제, 로그 관리 등)
- 내부 심사 실시

**인증 범위 확정**
- 조직: 전사 or 특정 사업부
- 물리적 위치: 본사, 지사, IDC
- 정보자산: 서버, DB, 네트워크, 애플리케이션
- 개인정보처리시스템 및 취급자

### 2. 신청 (인증기관)

**신청 서류 제출**
- 인증 신청서
- 관리체계 운영 명세서
- 사업자 등록증

**계약 및 수수료 납부**
- 인증 심사 수수료 산정 (조직 규모·범위 기준)
- 계약 체결 및 납부

### 3. 심사 (2~3개월)

**문서 심사**
- 정책·절차·매뉴얼 검토
- 위험 평가 결과·보호대책 검토

**현장 심사**
- 인증 심사원 방문 (보통 3~5일)
- 담당자 인터뷰
- 시스템·물리적 보안 점검
- 증적 자료 확인

**결함 보고서**
- 미흡 사항 지적 (Critical, Major, Minor)
- 조직이 보완 조치

**보완 조치 내역서 제출**
- 지적 사항 개선 증빙

### 4. 인증서 발급

**인증위원회 심의**
- 최초·갱신 심사는 인증위원회 의결
- 유지 심사는 인증기관 자체 의결

**인증서 발급**
- 유효 기간: 3년
- 인증 범위·인증 번호 명시

### 5. 사후 관리

**연간 유지 심사**
- 인증 후 매년 1회 유지 심사
- 관리체계 지속 운영 점검

**3년 갱신 심사**
- 3년 후 전체 재심사
- 갱신 심사 통과 시 3년 연장

## 간편 인증 제도 (2024년 도입)

**대상**
- 정보통신서비스 매출 300억 원 미만 중소기업
- 중견기업 중 일부 조건 충족 시

**특징**
- 101개 → 80개 필수 통제 항목으로 축소
- 심사 기간·비용 절감 (기존 대비 30% 이상)
- 중소기업 부담 경감

**제한**
- 의무 인증 대상 중 일부만 해당
- 대기업은 기존 인증 유지

## 인증 효과

**법적 의무 이행**
- 정보통신망법·개인정보보호법 준수
- 과태료·과징금 회피

**보안 사고 예방**
- 체계적 위험 관리로 해킹·랜섬웨어·개인정보 유출 방지
- 사고 대응 체계 구축

**대외 신뢰도 향상**
- 고객·파트너 신뢰 확보
- 브랜드 이미지 제고

**입찰·제안 우대**
- 공공기관 입찰 시 가점
- 민간 기업도 보안 요구사항으로 요구

**보험료 할인**
- 사이버 보험료 할인 혜택

**조직 내부 보안 문화 정착**
- 임직원 보안 인식 제고
- 보안 사고 대응 능력 향상

## 인증 비용

**심사 수수료**
- 조직 규모·인증 범위에 따라 차등
- 소규모: 약 500만~1,000만 원
- 중규모: 약 1,000만~3,000만 원
- 대규모: 3,000만 원 이상

**컨설팅 비용 (선택)**
- 외부 컨설팅 업체 활용 시
- 약 2,000만~5,000만 원 (규모별 차등)

**내부 인력 투입**
- 전담 인력 3~6개월 투입
- 기회 비용 포함

**보호대책 구축 비용**
- 방화벽, IDS/IPS, SIEM, 백업 솔루션 등
- 기존 인프라에 따라 변동

## 주요 준비 사항

### 기술적 보호대책

**접근 통제**
- 방화벽, IPS/IDS
- 네트워크 세그멘테이션
- 계정·권한 관리 시스템

**암호화**
- 개인정보·중요정보 암호화 (DB, 파일, 전송)
- SSL/TLS 인증서
- 암호키 관리 체계

**로그 관리**
- 접속 로그 수집·보관 (최소 6개월~1년)
- 로그 무결성 보호
- 로그 정기 분석

**악성코드 대응**
- 백신·EDR 솔루션
- 정기 업데이트

**백업·복구**
- 정기 백업 (일·주·월)
- 복구 테스트
- 백업 매체 안전 보관

### 관리적 보호대책

**정책·절차 문서화**
- 정보보호 정책
- 개인정보 처리 방침
- 보안 관련 절차서·지침서

**위험 관리**
- 정보자산 목록
- 위험 평가 보고서
- 보호대책 이행 계획

**교육·훈련**
- 전 직원 연 1회 이상 보안 교육
- 개인정보 취급자 별도 교육
- 교육 이수 기록 관리

**내부 심사**
- 연 1회 이상 자체 점검
- 미흡 사항 개선

### 물리적 보호대책

**출입 통제**
- 출입 카드, 생체 인증
- 출입 로그 기록

**CCTV**
- 주요 구역 감시
- 녹화 영상 보관

**서버실 보안**
- 별도 구획·잠금 장치
- 환경 관리 (온도·습도·화재 감지)

## 주요 점검 항목 예시

**정책 수립**
- 정보보호 정책서 및 개인정보 처리 방침이 최신 법령 반영했는가?
- 경영진이 승인했는가?

**위험 평가**
- 정보자산을 식별하고 분류했는가?
- 위협·취약점 분석을 했는가?
- 위험 처리 계획이 있는가?

**암호화**
- 개인정보가 암호화되어 저장되는가? (DB, 파일)
- 네트워크 전송 시 암호화하는가? (HTTPS, VPN)

**접근 통제**
- 비밀번호 정책이 있는가? (복잡도, 주기)
- 불필요한 계정을 정기적으로 삭제하는가?
- 최소 권한 원칙을 적용하는가?

**로그 관리**
- 접속 로그를 수집·보관하는가?
- 로그를 정기적으로 분석하는가?

**개인정보 파기**
- 보유 기간 만료 시 자동 파기하는가?
- 파기 기록을 남기는가?

## 요약

ISMS-P(정보보호 및 개인정보보호 관리체계 인증)는 기업·기관이 정보보호와 개인정보보호를 체계적으로 수립·운영하고 있음을 KISA·금융보안원이 객관적으로 평가·증명하는 대한민국 국가 인증 제도다. 101개 인증 기준(관리체계 16개, 보호대책 64개, 개인정보 처리 21개)을 통해 위험 관리, 암호화, 접근 통제, 로그 관리, 개인정보 수집·파기 등을 점검하며, 일정 규모 이상 정보통신서비스 제공자는 법적 의무 인증 대상이다. 3년 유효 기간·매년 유지 심사를 통해 지속적 보안 수준 유지를 요구하며, 법적 의무 이행, 보안 사고 예방, 대외 신뢰도 향상, 입찰 우대 등의 효과를 제공한다.
