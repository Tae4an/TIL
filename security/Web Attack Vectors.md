# Web Attack Vectors
웹 공격 벡터는 **공격자가 웹 애플리케이션·사용자·서버를 공격하기 위해 “지나가는 경로/수단”** 을 의미한다.  
아래는 실무에서 가장 자주 보는 주요 웹 공격 벡터들이다.

***
## 1. 인젝션 계열 (Injection)
- SQL Injection  
  - 입력값을 통해 SQL 문을 깨뜨려, 데이터 조회·변조·삭제, 계정 탈취까지 하는 공격.  
- 명령어/OS Injection, LDAP/NoSQL Injection 등  
  - 시스템 명령어나 디렉터리/쿼리 언어에 악의적 데이터를 주입해 의도치 않은 명령 실행을 유도.

핵심 포인트: 입력값 검증/바인딩(Prepared Statement), ORM 사용, 입력값 이스케이프.

***
## 2. XSS (Cross-Site Scripting)
- 개념  
  - 공격자가 악성 스크립트(JS)를 페이지에 주입하고, 다른 사용자의 브라우저에서 그 스크립트가 실행되게 하는 공격.  
- 종류  
  - Stored XSS: 게시글/댓글 등에 악성 스크립트를 저장해, 페이지 열 때마다 실행.  
  - Reflected XSS: URL 파라미터 등에 스크립트를 넣어, 응답에 그대로 반사되게 함.  
  - DOM-based XSS: 클라이언트 측 JS가 DOM을 조작하는 과정의 취약점을 악용.

핵심 포인트: 출력 인코딩, 입력 검증, CSP(Content Security Policy), 라이브러리 기준의 안전한 템플릿 사용.

***
## 3. CSRF (Cross-Site Request Forgery)
- 개념  
  - 사용자가 로그인해 있는 사이트로, 공격자가 “위조된 요청”을 보내게 만들어 사용자의 권한으로 의도치 않은 작업을 실행시키는 공격.  
  - 예: 로그인된 상태에서 악성 페이지 접속 → 자동으로 송금/비밀번호 변경 요청 전송.  
- 특징  
  - 브라우저가 자동으로 쿠키를 붙여 보내는 특성을 악용.

핵심 포인트: CSRF 토큰, SameSite 쿠키, Referer/Origin 검증.

***
## 4. 인증·세션 관련 공격
- 세션 하이재킹(Session Hijacking)  
  - 세션 ID를 탈취해 사용자인 척 가장. (XSS, 네트워크 스니핑, 취약한 쿠키 설정 등으로 세션 탈취)  
- 세션 고정(Session Fixation)  
  - 미리 정한 세션 ID를 사용자에게 사용하게 만든 뒤, 그 세션으로 로그인된 상태를 공격자가 재사용.  
- 취약한 인증 / 약한 비밀번호  
  - 브루트포스, Credential Stuffing(유출된 ID/비번 재사용), 2FA 미사용 등.

핵심 포인트: 안전한 세션 관리(HTTPOnly/Secure, 재발급), 강한 패스워드 정책, MFA, 로그인 보호(락아웃, CAPTCHA 등).

***
## 5. 파일 업로드 / 경로 조작
- 악성 파일 업로드  
  - 웹셸, 악성 스크립트, 실행 가능한 코드 등을 업로드해 서버에서 실행되도록 유도.  
- 디렉터리 트래버설(Path Traversal)  
  - `../` 등을 이용해 의도치 않은 디렉터리/파일에 접근.

핵심 포인트: 확장자/콘텐츠 타입 검증, 업로드 디렉터리 실행 금지, 파일명 난수화, 경로 정규화.

***
## 6. 서버 측 요청 위조 (SSRF)
- 개념  
  - 서버가 외부/내부 리소스에 HTTP 요청을 보내는 기능을 악용해,  
    공격자가 서버로 하여금 내부망·메타데이터 엔드포인트 등에 요청을 보내게 하는 공격.  
- 예  
  - URL 입력 기능이 있는 곳에 `http://169.254.169.254/latest/meta-data/` 등 내부 주소 입력.

핵심 포인트: 아웃바운드 요청 도메인/주소 화이트리스트, 내부 주소 차단, 프록시 계층 검증.

***
## 7. 설정/구성 취약점
- 디폴트 계정·디폴트 패스워드 방치  
- 디버그 페이지 노출, 상세 에러 메시지 노출  
- 불필요한 포트/서비스 오픈, 관리자 인터페이스 노출  
- CSP, 보안 헤더 미설정(HSTS, X-Frame-Options 등)

공격 벡터 관점에서는 “구성 실수로 열려 있는 각종 관리/디버그 엔드포인트, 불필요한 노출 포인트”들이 직접적인 공격 경로가 된다.

***
## 8. 클라이언트 측 벡터 (브라우저/사용자)
- Drive-by Download  
  - 취약한 브라우저·플러그인을 노려, 페이지 방문만으로 악성코드 내려받게 함.  
- 피싱 / 가짜 로그인 페이지  
  - 진짜 서비스처럼 보이는 가짜 사이트로 유도해 자격증명 탈취.  
- 약한/없는 TLS  
  - HTTPS 미사용·오류 등으로 중간자 공격(MiTM)에 취약.

***
## 9. 요약: Web Attack Vectors를 볼 때 포인트
- “입력이 어디로 흘러가는지”  
  - DB 쿼리, HTML 출력, 파일 시스템, 외부 요청 등으로 연결되는 모든 경로가 잠재적 벡터.  
- “누가/무엇이 신뢰되는지”  
  - 쿠키, 세션, 토큰, 내부 IP, 관리자 기능 등 신뢰하고 있는 요소가 공격자에 의해 가장될 수 있는지.  
- “어디까지 노출되어 있는지”  
  - 관리 콘솔, 디버그 인터페이스, 메타데이터/내부 API, 클라우드 스토리지 URL 등.
