# PCI DSS란?

## 개념 정의
PCI DSS(Payment Card Industry Data Security Standard, 지불 카드 산업 데이터 보안 표준)는 신용카드·체크카드 소유자의 카드 정보를 저장·처리·전송하는 모든 기업·조직이 카드 정보를 안전하게 보호하기 위해 준수해야 하는 국제 정보보호 표준이다. Visa, Mastercard, American Express, Discover, JCB 등 주요 카드 브랜드사들이 설립한 PCI SSC(Payment Card Industry Security Standards Council, 지불 카드 산업 보안 표준 위원회)가 제정·관리한다.

쉽게 말해 "신용카드 정보를 다루는 모든 기업이 카드 정보 유출·도용을 막기 위해 반드시 지켜야 하는 글로벌 보안 표준"이다.

## 핵심 특징

**글로벌 표준**
- 전 세계 5대 카드 브랜드사 공동 제정
- 국가·지역 무관하게 동일한 기준 적용
- 카드 결제를 받는 모든 기업에 적용

**카드 소유자 데이터(CHD) 보호**
- PAN(Primary Account Number, 카드 번호)
- 카드 소유자 이름
- 만료일
- 서비스 코드
- 민감한 인증 데이터(CAV2/CVC2/CVV2, PIN)

**법적 의무 + 계약상 의무**
- 카드 브랜드사와의 가맹 계약 조건
- 미준수 시 카드사로부터 벌금·과징금 부과
- 카드 결제 처리 권한 상실 가능

**리스크 기반 접근**
- 거래량·규모에 따라 4단계 레벨로 구분
- 레벨별 검증 방법 차등 적용
- 중소기업부터 대기업까지 모두 대응 가능

**연간 검증 의무**
- 매년 또는 분기마다 준수 검증
- 외부 심사 또는 자체 평가
- 지속적 보안 관리 요구

## PCI DSS 적용 대상

### 모든 카드 정보 취급 조직

**가맹점(Merchants)**
- 오프라인 매장 (POS 시스템)
- 온라인 쇼핑몰 (전자상거래)
- 모바일 결제 앱
- 정기 결제·구독 서비스

**서비스 제공자(Service Providers)**
- PG사(결제 대행사)
- 호스팅 업체
- 클라우드 서비스 제공자
- 콜센터 (전화 주문 처리)

**카드 발급사·매입사(Issuers & Acquirers)**
- 은행·금융기관
- 카드사 자체

**적용 기준**
- 카드 정보를 저장·처리·전송하는 모든 기업
- 거래 규모·건수 무관 (연 1건이라도 적용)

## 준수 레벨 (Merchant Level)

연간 카드 거래량에 따라 4단계로 구분되며, 레벨별로 검증 방법이 다르다.

| 레벨 | 연간 거래량 | 검증 방법 |
|------|------------|----------|
| Level 1 | 600만 건 이상 | 외부 QSA 심사 + 분기별 네트워크 스캔 (필수) |
| Level 2 | 100만~600만 건 | 자체 평가(SAQ) + 분기별 네트워크 스캔 |
| Level 3 | 2만~100만 건 | 자체 평가(SAQ) + 분기별 네트워크 스캔 |
| Level 4 | 2만 건 미만 | 자체 평가(SAQ) + 분기별 네트워크 스캔 (선택) |

**QSA (Qualified Security Assessor)**
- PCI SSC 공인 외부 보안 심사 전문가
- Level 1은 필수, 다른 레벨은 선택

**SAQ (Self-Assessment Questionnaire)**
- 자체 평가 설문지
- 기업이 직접 체크리스트 점검

**ASV (Approved Scanning Vendor)**
- PCI SSC 공인 취약점 스캔 업체
- 분기별 외부 네트워크 취약점 스캔

## PCI DSS 요구사항 (12개 항목)

PCI DSS v4.0.1 기준 12개 필수 요구사항을 6개 통제 목표로 그룹화한다.

### 통제 목표 1: 안전한 네트워크 및 시스템 구축·유지

**요구사항 1: 네트워크 보안 통제 설치·유지**
- 방화벽·라우터 설치 및 구성
- 카드 데이터 환경(CDE)과 비CDE 네트워크 분리
- DMZ 구축 (인터넷과 내부망 격리)
- 인바운드·아웃바운드 트래픽 제한
- 최소 6개월마다 방화벽 규칙 검토

**요구사항 2: 시스템 구성 요소에 안전한 구성 적용**
- 벤더 기본 비밀번호 변경 (admin/admin 금지)
- 불필요한 서비스·프로토콜·포트 비활성화
- 보안 설정 표준화 (하드닝 가이드)
- 암호화 프로토콜만 사용 (TLS 1.2 이상)

### 통제 목표 2: 카드 소유자 데이터 보호

**요구사항 3: 저장된 계정 데이터 보호**
- 카드 데이터 최소 보관 (꼭 필요한 것만)
- PAN(카드 번호) 저장 시 암호화 또는 마스킹
- CAV2/CVC2/CVV2, PIN 저장 절대 금지
- 암호화 키 안전 관리
- 불필요한 데이터 정기 삭제

**요구사항 4: 공개 네트워크 전송 시 강력한 암호화 적용**
- 인터넷 전송 시 TLS/SSL 암호화 (HTTPS)
- 무선 네트워크 WPA2/WPA3 암호화
- 이메일로 카드 정보 전송 금지
- 종단 간 암호화(End-to-End Encryption)

### 통제 목표 3: 취약점 관리 프로그램 유지

**요구사항 5: 악성 소프트웨어로부터 시스템·네트워크 보호**
- 안티바이러스·안티멜웨어 설치
- 실시간 보호 활성화
- 정기 업데이트 및 전체 스캔
- 로그 모니터링

**요구사항 6: 안전한 시스템·소프트웨어 개발·유지**
- 보안 패치 정기 적용
- 웹 애플리케이션 보안 취약점 제거 (OWASP Top 10)
- 개발 환경과 운영 환경 분리
- 소스 코드 보안 점검
- 변경 관리 절차 (Change Control)

### 통제 목표 4: 강력한 접근 통제 조치 구현

**요구사항 7: 업무상 필요에 따라 시스템·카드 데이터 접근 제한**
- 최소 권한 원칙 (Least Privilege)
- 역할 기반 접근 제어 (RBAC)
- 카드 데이터 접근 권한 최소화
- 권한 승인·변경·회수 절차

**요구사항 8: 사용자 식별 및 인증**
- 고유 사용자 ID 부여 (공용 계정 금지)
- 강력한 비밀번호 정책 (최소 8자, 복잡도, 주기적 변경)
- 다중 인증(MFA) 적용 (특히 원격 접속)
- 비활성 계정 비활성화

**요구사항 9: 카드 데이터에 대한 물리적 접근 제한**
- 서버실·데이터센터 출입 통제
- 카드 리더기·POS 단말기 물리적 보호
- 방문자 출입 기록 관리
- 미디어·백업 안전 보관
- 장비 폐기 시 데이터 완전 삭제
- CCTV 및 전자 접근 제어
- 출입 기록 최소 90일 보관

### 통제 목표 5: 네트워크 정기 모니터링·테스트

**요구사항 10: 시스템·카드 데이터 접근 로깅·모니터링**
- 모든 카드 데이터 접근 기록
- 로그 최소 1년 보관 (최근 3개월은 즉시 분석 가능)
- 시스템 시간 동기화 (NTP)
- 로그 무결성 보호
- 이상 징후 자동 알림

**요구사항 11: 보안 시스템·프로세스 정기 테스트**
- 분기별 외부 취약점 스캔 (ASV)
- 분기별 무선 분석기 스캔 (불법 AP 탐지)
- 분기별 내부 취약점 스캔
- 연간 침투 테스트 (애플리케이션·네트워크)
- IDS/IPS 운영
- 파일 무결성 모니터링 (FIM) - 주 1회 파일 비교

### 통제 목표 6: 정보보호 정책 수립·유지

**요구사항 12: 조직 정책 및 프로그램으로 정보보호 지원**
- 전사 보안 정책 수립
- 연 1회 공식 위험 평가
- 직원 보안 인식 교육
- 신규 채용자 배경 조사
- 제3자·협력사 보안 계약
- 사고 대응 계획 수립·테스트
- 기술 사용 정책 수립
- 정기 보안 점검

## 검증 방법

### 1. 자체 평가 (SAQ - Self-Assessment Questionnaire)

**대상**
- Level 2~4 기업 (선택적으로 Level 1도 가능)

**SAQ 유형**
- SAQ A: 전체 아웃소싱 (카드 정보 직접 안 다룸)
- SAQ A-EP: 전자상거래 (결제 페이지만 아웃소싱)
- SAQ B: POS 단말기만 사용 (온라인 없음)
- SAQ C: 웹 기반 결제 단말기
- SAQ D: 모든 카드 데이터 환경 (가장 포괄적)

**절차**
1. 해당 SAQ 설문지 다운로드
2. 질문에 따라 체크 (예/아니오/해당 없음)
3. 미흡 사항 개선
4. AOC(Attestation of Compliance, 준수 확인서) 작성
5. 카드사에 제출

### 2. 외부 심사 (QSA - Qualified Security Assessor)

**대상**
- Level 1 기업 필수
- Level 2~4 자발적 선택 가능

**절차**
1. PCI SSC 공인 QSA 선정
2. 현장 심사 (문서·인터뷰·시스템 점검)
3. ROC(Report on Compliance, 준수 보고서) 작성
4. 미흡 사항 보완
5. AOC 발급
6. 카드사에 제출

### 3. 분기별 네트워크 스캔 (ASV)

**대상**
- 모든 레벨 (Level 4는 선택)

**절차**
1. PCI SSC 공인 ASV 선정
2. 외부 네트워크 취약점 스캔
3. 탐지된 취약점 패치
4. 재스캔 (Pass 받을 때까지)
5. 스캔 보고서 카드사 제출

## 준수 비용

**Level 1 (대기업)**
- QSA 심사 비용: 5,000만~2억 원
- ASV 스캔: 500만~1,000만 원/년
- 보안 솔루션 구축: 수억~수십억 원

**Level 2~4 (중소기업)**
- SAQ 자체 평가: 무료 (내부 인력 투입)
- 컨설팅 비용: 1,000만~5,000만 원 (선택)
- ASV 스캔: 300만~700만 원/년
- 보안 솔루션: 수천만~수억 원

## 미준수 시 페널티

**카드사 벌금**
- 월 5,000~100,000달러
- 위반 정도·거래량에 따라 차등

**카드 처리 권한 상실**
- 카드 결제 불가 → 사업 중단

**평판 손실**
- 고객 신뢰 상실
- 브랜드 이미지 훼손

**법적 책임**
- 데이터 유출 시 손해배상 소송
- 규제 기관 제재

## 카드 데이터 환경(CDE)

**정의**
- 카드 소유자 데이터를 저장·처리·전송하는 시스템·네트워크·인프라

**범위**
- 결제 애플리케이션
- 데이터베이스
- 웹 서버
- 네트워크 장비
- POS 단말기

**CDE 최소화 전략**
- 카드 정보를 최대한 저장하지 않음
- 토큰화(Tokenization) 활용
- 아웃소싱 (PG사에 결제 위임)
- 네트워크 세그멘테이션 (CDE 격리)

## 주요 준비 사항

**네트워크 보안**
- 방화벽·IDS/IPS 구축
- 네트워크 세그멘테이션
- 무선 네트워크 암호화

**데이터 보호**
- 카드 번호 암호화 (AES-256)
- 데이터 마스킹
- 안전한 키 관리 (HSM)

**접근 제어**
- MFA 구현
- 최소 권한 원칙
- 계정 관리 프로세스

**로그 관리**
- 중앙 집중식 로그 수집
- SIEM 구축
- 로그 무결성 보호

**취약점 관리**
- 정기 패치 관리
- 분기별 취약점 스캔
- 연간 침투 테스트

**물리적 보안**
- 출입 통제 (카드·생체)
- CCTV 설치
- 출입 기록 관리

**정책·절차**
- 정보보호 정책 문서화
- 사고 대응 계획
- 직원 보안 교육 프로그램

## 모범 사례

**범위 최소화**
- 카드 정보 저장 최소화
- 토큰화·암호화로 민감 데이터 보호
- 결제 처리를 PG사에 아웃소싱

**자동화**
- 패치 관리 자동화
- 취약점 스캔 자동화
- 로그 분석 자동화

**정기 점검**
- 분기별 내부 점검
- 연간 외부 심사
- 지속적 모니터링

**직원 교육**
- 연 1회 이상 보안 교육
- 피싱 시뮬레이션
- 보안 인식 캠페인

**제3자 관리**
- 협력사·벤더 보안 평가
- 계약서에 보안 조항 명시
- 정기 점검

## 요약

PCI DSS는 신용카드 정보를 저장·처리·전송하는 모든 기업이 준수해야 하는 글로벌 카드 정보 보호 표준으로, 5대 카드 브랜드사가 공동 제정한 12개 요구사항(네트워크 보안, 시스템 구성, 데이터 암호화, 악성코드 방어, 접근 통제, 로그 관리, 취약점 테스트, 정보보호 정책 등)을 통해 카드 데이터 환경을 보호한다. 연간 거래량에 따라 4단계 레벨로 구분되며, Level 1은 외부 QSA 심사가 필수이고 나머지는 자체 평가(SAQ)와 분기별 ASV 스캔을 수행한다. 미준수 시 벌금·카드 처리 권한 상실·평판 손실 등 심각한 페널티가 부과되며, 준수를 위해서는 방화벽·암호화·MFA·로그 관리·취약점 스캔·물리적 보안·정책 수립 등 기술적·관리적·물리적 보호대책을 모두 구현해야 한다.
